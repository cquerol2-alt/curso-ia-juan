🧮 Por qué embeddings son la pieza fundamental que tienes que entender

La búsqueda por substring del agente de la clase 03 tiene un problema obvio: si tú tienes un apunte que habla de "caching en Anthropic" y preguntas "cómo reutilizo el contexto entre llamadas", no encuentra nada. No hay una sola palabra en común.

Los embeddings resuelven exactamente eso. Son la representación numérica que te permite buscar por significado, no por palabras. Sin embeddings no hay RAG. Sin entender bien embeddings vas a copiar snippets y tu sistema va a fallar en producción sin que sepas por qué.

Esta clase no monta LangChain encima. Trabajas con la API de OpenAI directamente (text-embedding-3-small) y ChromaDB como vector store local. Al final tendrás una versión semántica del search_docs de la clase 03 que encuentra lo relevante aunque no compartan una palabra.

🧠 Conceptos clave

1. Qué es un embedding

Un embedding es un vector de números reales (típicamente 384–1536 dimensiones) que representa el significado de un trozo de texto. Dos textos que hablan de lo mismo → sus vectores caen cerca en el espacio. Dos textos no relacionados → lejos.

  • "caching en Anthropic" → [0.12, -0.34, 0.88, ..., 0.02] (1536 floats)
  • "reutilizar contexto entre llamadas" → [0.10, -0.30, 0.91, ..., 0.05] ← parecido
  • "receta de paella" → [0.91, 0.45, -0.22, ..., -0.60] ← lejos

2. Distancia coseno (la única que necesitas recordar)

La métrica estándar para comparar embeddings es la similaridad coseno: mide el ángulo entre dos vectores. Va de −1 (opuestos) a 1 (idénticos). 

import numpy as np

def cos_sim(a, b):
    a, b = np.array(a), np.array(b)
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))
OpenAI te devuelve vectores ya normalizados (norm=1). Eso significa que el producto escalar (np.dot) ya ES la similaridad coseno. Puedes ahorrarte el denominador.

3. El modelo: text-embedding-3-small

  • Dimensiones: 1536 (reducibles a 256–512 con el parámetro dimensions sin perder casi calidad)
  • Contexto máximo: 8 191 tokens por documento
  • Coste: $0.02 por millón de tokens → indexar 1 000 páginas cuesta ~$0.01
  • Por qué este y no Anthropic: Anthropic no ofrece API de embeddings a día de hoy. Usa OpenAI o Voyage AI. Es normal mezclar proveedores en stack de producción

4. Vector store: ChromaDB

Un vector store es una base de datos optimizada para buscar "los k vectores más cercanos a este otro" (nearest neighbor search) de forma barata — no recorre los N documentos uno a uno, usa estructuras como HNSW.

  • ChromaDB: el más simple. pip install chromadb y listo. Dos modos:
    • Client() en memoria (se pierde al cerrar) — para prototipos
    • PersistentClient(path="./chroma") en disco — lo que usarás tú
  • Alternativas: FAISS (más rápido, más manual), Qdrant/Weaviate (cliente-servidor, escalable), Pinecone (managed)
  • Regla 2026: si tu corpus cabe en una máquina (< 1M documentos), ChromaDB sobra. No pagues Pinecone por gusto

Paso 2 — Tu primer embedding

import os, numpy as np
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
oai = OpenAI()

def embed(texts: list[str]) -> list[list[float]]:
    resp = oai.embeddings.create(
        model="text-embedding-3-small",
        input=texts
    )
    return [d.embedding for d in resp.data]

frases = [
    "prompt caching reutiliza el contexto entre llamadas",
    "cómo reutilizar contexto entre llamadas al modelo",
    "la paella lleva azafrán, pollo y conejo",
]
V = embed(frases)
print(f"dim = {len(V[0])}")            # 1536
print(f"sim(0,1) = {np.dot(V[0], V[1]):.3f}")   # alto (~0.7–0.8)
print(f"sim(0,2) = {np.dot(V[0], V[2]):.3f}")   # bajo (~0.1–0.2)
Observa: frases 0 y 1 no comparten ni una palabra relevante y salen con similaridad alta. Eso es lo que hace que embeddings sean superpoderosos para búsqueda.

Paso 3 — Indexar documentos en ChromaDB

import chromadb
from pathlib import Path

# Persistent en disco — se guarda en ./chroma
chroma = chromadb.PersistentClient(path="./chroma")

# embedding_function=None → le pasamos nosotros los embeddings ya calculados
col = chroma.get_or_create_collection(
    name="apuntes",
    metadata={"hnsw:space": "cosine"}
)

def indexar(carpeta="apuntes", chunk_chars=800):
    textos, ids, metas = [], [], []
    for p in Path(carpeta).rglob("*.md"):
        raw = p.read_text(encoding="utf-8")
        # chunking simple: trozos de ~800 chars
        for i in range(0, len(raw), chunk_chars):
            trozo = raw[i:i + chunk_chars]
            if trozo.strip():
                textos.append(trozo)
                ids.append(f"{p.name}:{i}")
                metas.append({"archivo": p.name, "offset": i})

    # embeddings en batches de 100
    for k in range(0, len(textos), 100):
        batch = textos[k:k+100]
        vecs = embed(batch)
        col.add(
            documents=batch,
            embeddings=vecs,
            ids=ids[k:k+100],
            metadatas=metas[k:k+100]
        )
    print(f"Indexados {len(textos)} chunks")

indexar()

Paso 4 — Consultar por similaridad semántica

def buscar(pregunta: str, k: int = 3):
    q_vec = embed([pregunta])[0]
    res = col.query(query_embeddings=[q_vec], n_results=k)
    # res tiene: documents, metadatas, distances (todos anidados en listas)
    for doc, meta, dist in zip(res["documents"][0], res["metadatas"][0], res["distances"][0]):
        print(f"\n[{meta['archivo']}:{meta['offset']}]  dist={dist:.3f}")
        print(doc[:200], "…")

buscar("¿cómo reutilizo el contexto entre llamadas?")
Ojo al signo: ChromaDB devuelve distancias (menos es mejor), no similaridades. Con hnsw:space=cosine, distance = 1 − similarity.

Paso 5 — search_docs_semantic como tool para el agente

Sustituye el search_docs de la clase 03 por esta versión semántica:

def search_docs_semantic(query: str, max_results: int = 3) -> str:
    q_vec = embed([query])[0]
    res = col.query(query_embeddings=[q_vec], n_results=max_results)
    out = []
    for doc, meta, dist in zip(res["documents"][0], res["metadatas"][0], res["distances"][0]):
        out.append(f"[{meta['archivo']}:{meta['offset']}  sim={1-dist:.2f}]\n{doc}")
    return "\n\n".join(out) or "Sin resultados."

🎓 Resumen y siguientes pasos

  • Un embedding es un vector denso que captura significado; dos textos parecidos → vectores cercanos.
  • La métrica estándar es similaridad coseno; con vectores normalizados (como los de OpenAI) = producto escalar.
  • text-embedding-3-small (1536 dim, $0.02/M tokens) es el default hoy.
  • ChromaDB es el vector store más simple; suficiente para < 1M documentos. Usa PersistentClient.
  • Swap del search_docs por uno semántico → tu agente ya hace RAG real.
