📖 Los 4 pasos del RAG clásico

  1. Chunk — partes tu corpus (PDF, apuntes) en trozos de ~400-1600 caracteres con solape. El tamaño importa mucho: muy corto pierde contexto, muy largo mete ruido.
  2. Embed — cada chunk pasa por text-embedding-3-small (OpenAI) y queda como vector de 1536 dimensiones. Lo guardas en ChromaDB.
  3. Retrieve — cuando llega la pregunta, la embedees y pides a Chroma los top-K chunks más parecidos (similitud coseno).
  4. Rerank — pasas los top-K (ej: K=20) por un reranker (modelo más potente que decide cuáles de los 20 son realmente relevantes). Te quedas con los top-N (ej: N=3) para el prompt final.
Reranker = filtro de precisión. El retriever es rápido pero ruidoso (mete resultados tangenciales). El reranker es lento pero preciso. Combinar ambos (retrieval amplio + rerank estricto) es la técnica estándar de RAG moderno — lo que Anthropic llama contextual retrieval.

💡 Por qué el chunk size cambia todo

Chunk sizeProsContrasBuen uso
200-400 charsRetrieval muy preciso, poco ruidoFragmenta el contexto; frases sueltas sin sentidoFAQs, Q&A cortos
800 charsBalance estándar, funciona bien en apuntesPuede partir un razonamiento por la mitadDefault recomendado
1600-2000 charsConserva contexto largo, razonamientos completosRetrieval más ruidoso, más tokens al LLMPapers, manuales técnicos

Hoy harás el experimento tú: con el mismo corpus, misma query, compararás los 3 tamaños y verás cuál da mejores respuestas. Este es el tipo de decisión que te hace senior.

Paso 2 · Chunker con solape

Python 
from pathlib import Path

def chunk_text(text: str, size: int = 800, overlap: int = 100) -> list[str]:
    """Parte en chunks con solape. Nada de LangChain."""
    chunks = []
    start = 0
    while start < len(text):
        end = start + size
        chunks.append(text[start:end])
        start = end - overlap
    return chunks

def load_corpus(dir_path: str) -> list[dict]:
    """Carga todos los .txt como {id, source, text}."""
    out = []
    for p in Path(dir_path).glob("**/*.txt"):
        raw = p.read_text(encoding="utf-8")
        for i, ch in enumerate(chunk_text(raw)):
            out.append({"id": f"{p.stem}_{i}", "source": p.name, "text": ch})
    return out

chunks = load_corpus("./apuntes")
print(f"Total chunks: {len(chunks)}")

Paso 3 · Indexar en Chroma

Python 
import os, chromadb
from openai import OpenAI
from dotenv import load_dotenv
load_dotenv()

oai = OpenAI()
chroma = chromadb.PersistentClient(path="./chroma")
col = chroma.get_or_create_collection(
    name="apuntes_rag",
    metadata={"hnsw:space": "cosine"},
)

def embed(texts: list[str]) -> list[list[float]]:
    resp = oai.embeddings.create(model="text-embedding-3-small", input=texts)
    return [d.embedding for d in resp.data]

# Batch de 100 para no romper el límite de la API
BATCH = 100
for i in range(0, len(chunks), BATCH):
    part = chunks[i:i+BATCH]
    col.add(
        ids=[c["id"] for c in part],
        documents=[c["text"] for c in part],
        embeddings=embed([c["text"] for c in part]),
        metadatas=[{"source": c["source"]} for c in part],
    )
print(f"Indexados: {col.count()} chunks")

Paso 4 · Retrieve + Rerank + Answer

Python 
from anthropic import Anthropic
client = Anthropic()
MODEL = "claude-sonnet-4-6"

def retrieve(query: str, k: int = 20) -> list[dict]:
    q_vec = embed([query])[0]
    res = col.query(query_embeddings=[q_vec], n_results=k)
    return [
        {"text": d, "source": m["source"], "distance": dist}
        for d, m, dist in zip(res["documents"][0], res["metadatas"][0], res["distances"][0])
    ]

RERANK_PROMPT = """Eres un reranker. Puntúa de 0 a 10 cuán relevante es cada fragmento para la pregunta.
Devuelve solo JSON: [{{"i": 0, "score": 8}}, ...]

Pregunta: {q}

Fragmentos:
{frags}"""

def rerank(query: str, candidates: list[dict], n: int = 3) -> list[dict]:
    import json
    frags = "\n".join([f"[{i}] {c['text'][:300]}" for i, c in enumerate(candidates)])
    resp = client.messages.create(
        model="claude-haiku-4-5-20251001",  # haiku = barato y rápido para rerank
        max_tokens=1024,
        messages=[{"role": "user", "content": RERANK_PROMPT.format(q=query, frags=frags)}],
    )
    scores = json.loads(resp.content[0].text)
    ranked = sorted(scores, key=lambda s: s["score"], reverse=True)[:n]
    return [candidates[s["i"]] for s in ranked]

ANSWER_PROMPT = """Responde usando SOLO los fragmentos dados. Cita la fuente entre corchetes [fuente].
Si la respuesta no está en los fragmentos, di "no lo sé".

Pregunta: {q}

Fragmentos:
{ctx}"""

def answer(query: str) -> str:
    candidates = retrieve(query, k=20)
    top = rerank(query, candidates, n=3)
    ctx = "\n---\n".join([f"[{c['source']}] {c['text']}" for c in top])
    resp = client.messages.create(
        model=MODEL,
        max_tokens=1024,
        messages=[{"role": "user", "content": ANSWER_PROMPT.format(q=query, ctx=ctx)}],
    )
    return resp.content[0].text

print(answer("¿Qué dicen mis apuntes sobre context engineering?"))

⚠️ Trampas típicas

  • Embedear query con otro modelo distinto al del corpus — el espacio vectorial no coincide. Usa SIEMPRE el mismo text-embedding-3-small para ambos.
  • Chunks sin solape — si un concepto clave está justo en el borde, lo pierdes. Siempre overlap ≥ 10% del size.
  • k demasiado bajo — con k=3 sin rerank, el tercer resultado puede ser basura pero lo metes en el prompt. Mejor k=20 y rerank a 3.
  • Rerank con el modelo grande — caro. Usa Haiku para rerank, Sonnet para la respuesta final.
