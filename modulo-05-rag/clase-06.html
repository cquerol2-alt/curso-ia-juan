📖 Cuándo gana cada arquitectura (resumen teórico)

ArquitecturaGana cuandoPierde cuandoCoste típico por query
Context stuffing Corpus estable <150K tokens, pocas queries al día, calidad máxima Corpus enorme o cambia mucho; muchos usuarios $0.05 - $0.30 (mayoría cacheado)
Agentic search Info dispersa en muchos sitios (web, docs, APIs), query ambigua que necesita exploración Latencia crítica, coste estricto $0.08 - $0.50 (varias vueltas)
RAG clásico Corpus grande pero bien estructurado, muchas queries simultáneas, coste bajo Preguntas que requieren razonamiento cruzado entre muchas secciones $0.01 - $0.05 (muy barato)
La sabiduría convencional dice "siempre RAG". Esto está obsoleto. Con ventanas de 200K-1M tokens y prompt caching, stuffing ha vuelto a ser competitivo — y a veces mejor.

Paso 2 · Cada arquitectura como función

Python — architectures.py 
import time
from anthropic import Anthropic
client = Anthropic()
MODEL = "claude-sonnet-4-6"

# --- STUFFING (clase 02) ---
APUNTES_COMPLETOS = open("apuntes_concat.txt").read()  # tu corpus en un solo string

def arq_stuffing(pregunta: str) -> dict:
    t0 = time.time()
    resp = client.messages.create(
        model=MODEL, max_tokens=512,
        system=[{
            "type": "text",
            "text": "Eres un asistente que responde usando SOLO los apuntes dados.\n\n" + APUNTES_COMPLETOS,
            "cache_control": {"type": "ephemeral"}
        }],
        messages=[{"role": "user", "content": pregunta}],
    )
    return {
        "respuesta": resp.content[0].text,
        "ms": int((time.time() - t0) * 1000),
        "input_tokens": resp.usage.input_tokens,
        "cached": resp.usage.cache_read_input_tokens,
        "output_tokens": resp.usage.output_tokens,
    }

# --- AGENTIC SEARCH (clase 03) ---
from mi_agent import run_agent  # el agente con tools de clase 03

def arq_agentic(pregunta: str) -> dict:
    t0 = time.time()
    respuesta, usage = run_agent(pregunta, return_usage=True)
    return {
        "respuesta": respuesta,
        "ms": int((time.time() - t0) * 1000),
        **usage,
    }

# --- RAG CLÁSICO (clase 05) ---
from mi_rag import answer as rag_answer

def arq_rag(pregunta: str) -> dict:
    t0 = time.time()
    respuesta = rag_answer(pregunta)
    return {"respuesta": respuesta, "ms": int((time.time() - t0) * 1000)}

Paso 3 · Lanzar benchmark y calcular coste

Python — run_benchmark.py 
PRICES = {  # USD por 1M tokens, Claude Sonnet 4.6
    "input": 3.00,
    "cached_read": 0.30,   # 10× más barato
    "cache_write": 3.75,   # 1.25× el input
    "output": 15.00,
}

def coste_usd(result: dict) -> float:
    non_cached = result.get("input_tokens", 0) - result.get("cached", 0)
    return (
        non_cached * PRICES["input"] +
        result.get("cached", 0) * PRICES["cached_read"] +
        result.get("output_tokens", 0) * PRICES["output"]
    ) / 1_000_000

resultados = []
for item in DATASET:
    for arq_name, arq_fn in [("stuffing", arq_stuffing), ("agentic", arq_agentic), ("rag", arq_rag)]:
        r = arq_fn(item["pregunta"])
        resultados.append({
            "arq": arq_name,
            "pregunta": item["pregunta"],
            "respuesta": r["respuesta"],
            "esperada": item["esperada"],
            "ms": r["ms"],
            "coste": coste_usd(r),
        })

import csv
with open("benchmark.csv", "w", encoding="utf-8", newline="") as f:
    w = csv.DictWriter(f, fieldnames=resultados[0].keys())
    w.writeheader(); w.writerows(resultados)

Paso 4 · Evaluar calidad (LLM-as-judge ligero)

Python — judge.py 
JUDGE_PROMPT = """Eres juez imparcial. Evalúa si la respuesta del modelo es correcta comparada con la respuesta esperada.
Devuelve JSON: {"correct": true/false, "razon": "..."}

Pregunta: {q}
Respuesta esperada: {exp}
Respuesta del modelo: {ans}"""

def juzgar(q, esp, ans) -> dict:
    import json
    resp = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=256,
        messages=[{"role": "user", "content": JUDGE_PROMPT.format(q=q, exp=esp, ans=ans)}],
    )
    return json.loads(resp.content[0].text)

Paso 5 · Tabla resumen

Python — summary.py 
import pandas as pd
df = pd.read_csv("benchmark_judged.csv")

resumen = df.groupby("arq").agg(
    accuracy=("correct", "mean"),
    coste_medio=("coste", "mean"),
    latencia_ms=("ms", "median"),
).round(3)
print(resumen)

📊 Cómo leer los resultados

  • Stuffing suele ganar en accuracy (ve todo el contexto de golpe) pero con latencia media.
  • RAG suele ser imbatible en coste y latencia, pero pierde accuracy en preguntas cross-seccionales.
  • Agentic suele quedar en medio: buena accuracy, latencia peor, coste peor — pero brilla cuando necesitas razonamiento dinámico.

El número final no te dice "la mejor". Te dice para tu caso y para tu corpus, cuál balance de coste/latencia/calidad prefieres. Esa es la decisión que justificarás en el ejercicio 2.
