Mind Map — Clase 04

Vector Stores: ChromaDB y FAISS

Bases de datos optimizadas para búsqueda semántica

🗄️
¿Qué es un Vector Store?
+
BD optimizada para vectores, no para SQL tradicional
Búsqueda por similitud (semantic search) en lugar de coincidencia exacta
Indispensable para RAG: almacena embeddings + metadata, devuelve chunks relevantes rápidamente
Operación clave: similarity_search(query, k=4) — busca los 4 vectores más cercanos
📍
ChromaDB
+
Open source, muy fácil de usar, perfecto para prototipado
Chroma.from_documents(documents, embedding_model) — crea el store en memoria
persist_directory="/ruta" — guarda el índice en disco para reutilizarlo
Metadata nativa: guarda y filtra por metadatos fácilmente
⚡
FAISS
+
De Meta/Facebook, ultra rápido con millones de vectores. Librería Python pura, sin API.
faiss.IndexFlatIP(d) — creas el índice con la dimensión d (1536 para text-embedding-3-small) y le haces add(vectors).
IndexFlatL2 — búsqueda exacta (más lento pero preciso). IndexIVFFlat / IndexHNSW — búsqueda aproximada (rápida, millones).
Mejor para: escala grande (producción) cuando necesitas máxima velocidad. Para el curso: ChromaDB es suficiente.
🔍
Operaciones clave
+
collection.query(query_embeddings=[...], n_results=k) — busca los k chunks más cercanos al embedding de la pregunta.
Devuelve documents, metadatas, distances (distancia coseno) y ids — todo lo que necesitas para montar el prompt y las citas.
collection.add(ids, embeddings, documents, metadatas) — añadir chunks embeados con su fuente y página en metadata.
PersistentClient(path="./chroma") — persiste el índice en disco, no hace falta reindexar en cada ejecución.
⚖️
ChromaDB vs FAISS
+
ChromaDB: más fácil, metadata nativa, perfecto para MVP y aprendizaje
FAISS: más rápido y escalable, mejor para millones de vectores en producción
Ambos son locales y gratis — no requieren API key ni suscripción
Para producción cloud: Pinecone, Weaviate, Milvus — ofrecen escalado automático y backups
