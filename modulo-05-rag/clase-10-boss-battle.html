🎯 Boss Battle: 3 arquitecturas sobre mis apuntes, comparadas

📅 Semana 8 — Martes | ⏱️ 4-6 horas de trabajo serio

Integra todo el módulo en un proyecto único: un chat web que contesta preguntas sobre tus apuntes usando las 3 arquitecturas (stuffing, agentic, RAG) y un botón para cambiar entre ellas. Observability con Langfuse. Deploy público.

📚 Profundiza con el workshop de hola.dev

Para justificar la arquitectura elegida en el informe del Boss Battle:

  • Sección 79 — Arquitectura de aplicaciones con LLM
  • Sección 69 — Árbol de decisión prompt/RAG/FT

Workshop público de hola.dev. Material externo, no entra en el test.

🎯 Objetivo

Construir y desplegar "Pregúntale a Mis Apuntes": un chat web donde Juan (o quien sea) pueda hacer una pregunta y elegir qué arquitectura responde. La app debe medir coste y latencia en vivo, y enviar trazas a Langfuse.

📝 Apuntes

Tus notas del curso (módulos 1-5) como corpus. Mínimo 10 archivos de texto/markdown.

⚙️ 3 arquitecturas

Stuffing con cache, agentic con tool use, RAG clásico con rerank. Las 3 funcionando, seleccionables desde la UI.

🌐 Deploy público

Frontend en Vercel + backend en Render/Railway + tracing en Langfuse cloud.

📦 Stack obligatorio

  • Claude Sonnet 4.6 (claude-sonnet-4-6) para responder en las 3 arquitecturas
  • Claude Haiku 4.5 (claude-haiku-4-5-20251001) para rerank
  • OpenAI text-embedding-3-small para embeddings
  • ChromaDB como vector store
  • FastAPI backend
  • Next.js 14 + Vercel AI SDK frontend con streaming
  • Langfuse para observability
❌ Prohibido: LangChain, LlamaIndex. Usa los SDKs directos de Anthropic y OpenAI. Todo el módulo está diseñado para enseñarte a entender y controlar cada pieza.

🧩 Plan de ataque (los 6 pasos del boss)

Paso 1 · Preparar los apuntes

Junta tus notas del curso o cualquier corpus tuyo (mínimo 10 archivos, 20-50 KB total). Guárdalos en backend/data/apuntes/*.md. Crea apuntes_concat.txt con todo concatenado (para la arq. stuffing).

Paso 2 · Backend con 3 endpoints

FastAPI con tres rutas distintas. Cada una implementa una arquitectura del módulo:

POST /ask/stuffing   → prompt caching con el corpus entero en system
POST /ask/agentic    → loop ReAct con tool search sobre el corpus
POST /ask/rag        → retrieve (k=20) + rerank haiku → sonnet con top-3

Cada endpoint devuelve { answer, latency_ms, cost_usd, trace_id }. Instrumenta con @observe.

Paso 3 · Indexar con ChromaDB

Script ingest.py que chunkea (800 chars, overlap 100), embebe con text-embedding-3-small y guarda en ChromaDB persistente. Debe ser idempotente.

Paso 4 · Frontend Next.js con selector

Chat con useChat del Vercel AI SDK. Un dropdown "Arquitectura: [stuffing | agentic | rag]" decide a qué endpoint se llama. Bajo cada respuesta, muestra coste y latencia de ese turno.

Paso 5 · Observability

Langfuse proyecto nuevo. Cada turno del chat genera una trace con el árbol completo. Bonus: botón 👍 / 👎 que registra langfuse.score().

Paso 6 · Deploy

Backend Python (FastAPI + ChromaDB) en Render o Railway. Frontend Next.js en Vercel. Variables de entorno: ANTHROPIC_API_KEY, OPENAI_API_KEY, LANGFUSE_PUBLIC_KEY, LANGFUSE_SECRET_KEY. Todo público y accesible por URL.
