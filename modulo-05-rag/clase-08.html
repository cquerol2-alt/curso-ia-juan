Clase 08: Frontend con Vercel AI SDK — streaming y UX de IA

📅 Semana 7 — Jueves | 🎬 30 min vídeo + 💻 60 min práctica + 🧪 15 min test

🎯 Qué vas a construir hoy

Un frontend web real (Next.js + Vercel AI SDK) que consume tu RAG de la clase 05 y muestra la respuesta streaming token a token con renderizado de markdown y citas a los chunks fuente. Dejamos atrás Streamlit: esto es lo que se usa en productos reales de IA hoy.

🧭 Filosofía: la UX de IA se gana con streaming. Una respuesta que aparece entera 5s después se siente lenta; la misma respuesta con tokens apareciendo a 1s se siente instantánea. Mismo coste, percepción distinta.

📖 Por qué Vercel AI SDK (y no Streamlit)

AspectoStreamlitVercel AI SDK + Next.js
Streaming token a token Mediocre, necesita trucos Nativo, hook useChat
Control de UI Limitado a widgets Python HTML/CSS/JS total
Hosting Streamlit Cloud / Render Vercel (deploy atómico por push)
Provider switching Manual Cambias 1 línea (anthropic → openai)
Producto real Demos internas Estándar de facto en SaaS de IA (Perplexity, v0, Cursor…)

Paso 2 · Backend route: app/api/chat/route.ts

TypeScript 
import { anthropic } from '@ai-sdk/anthropic';
import { streamText } from 'ai';

export const maxDuration = 30;

// Cliente de tu vector store — placeholder. En real, llama al backend Python
// o mueve ChromaDB a un servicio expuesto.
async function retrieveContext(query: string): Promise<string[]> {
  // fetch a tu API Python que hace retrieve+rerank de la clase 05
  const r = await fetch(`${process.env.RAG_API}/retrieve?q=${encodeURIComponent(query)}`);
  const data = await r.json();
  return data.chunks as string[]; // top-3 chunks
}

export async function POST(req: Request) {
  const { messages } = await req.json();
  const lastUser = messages[messages.length - 1].content as string;

  const chunks = await retrieveContext(lastUser);
  const context = chunks.map((c, i) => `[${i + 1}] ${c}`).join('\n---\n');

  const system = `Eres un asistente que responde SOLO con los chunks dados.
Cita las fuentes como [1], [2]... al final de cada afirmación.
Si la respuesta no está en los chunks, di "No encuentro info en mis apuntes".

CHUNKS:
${context}`;

  const result = streamText({
    model: anthropic('claude-sonnet-4-6'),
    system,
    messages,
  });

  return result.toDataStreamResponse();
}

Paso 3 · Frontend: app/page.tsx

TypeScript + React 
'use client';
import { useChat } from 'ai/react';
import ReactMarkdown from 'react-markdown';

export default function Chat() {
  const { messages, input, handleInputChange, handleSubmit, isLoading } = useChat();

  return (
    <div className="max-w-2xl mx-auto p-4">
      <h1 className="text-2xl font-bold mb-4">Pregúntale a mis apuntes</h1>

      <div className="space-y-4 mb-8">
        {messages.map(m => (
          <div key={m.id} className={m.role === 'user' ? 'bg-slate-700 p-3 rounded' : 'bg-slate-800 p-3 rounded'}>
            <div className="text-xs opacity-60 mb-1">{m.role}</div>
            <ReactMarkdown>{m.content}</ReactMarkdown>
          </div>
        ))}
        {isLoading && <div className="opacity-50">Pensando...</div>}
      </div>

      <form onSubmit={handleSubmit} className="flex gap-2">
        <input
          value={input}
          onChange={handleInputChange}
          placeholder="Tu pregunta sobre los apuntes..."
          className="flex-1 border rounded p-2 bg-slate-900"
        />
        <button type="submit" className="bg-cyan-500 text-black px-4 rounded font-bold">
          Enviar
        </button>
      </form>
    </div>
  );
}

Paso 4 · Arranca el backend Python (RAG) en paralelo

Python — rag_api.py (FastAPI) 
from fastapi import FastAPI
from rag_pipeline import retrieve, rerank  # de la clase 05

app = FastAPI()

@app.get("/retrieve")
def do_retrieve(q: str):
    candidates = retrieve(q, k=20)
    top = rerank(q, candidates, n=3)
    return {"chunks": [c["text"] for c in top]}

# uvicorn rag_api:app --reload --port 8000

Paso 6 · Citas visibles en la UI (bonus)

Haz que el backend devuelva también los chunks como data annotations para renderizarlos como tarjetas debajo de la respuesta. El Vercel AI SDK soporta data fields custom vía streamText({ data }).

🧠 Gotchas comunes

  • CORS: si el front y el back están en dominios distintos, añade CORSMiddleware en FastAPI o llama al back desde el server side de Next.js (que es lo que hace este ejemplo).
  • Rate limits: el streaming mantiene la conexión abierta; añade timeout en tu Vercel function (export const maxDuration = 30).
  • Markdown código: sin react-markdown verás los backticks en bruto. Instálalo desde el minuto 0.
  • Provider switch: para probar GPT-4, cambia anthropic('claude-sonnet-4-6') por openai('gpt-4o'). Instala @ai-sdk/openai primero.
