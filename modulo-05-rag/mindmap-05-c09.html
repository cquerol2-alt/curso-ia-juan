Mind Map — Clase 09

Testing y Evaluación de Respuestas RAG

Mide si tu RAG funciona bien realmente

⚠️
¿Por qué evaluar?
+
Los LLMs alucinan: inventan datos que no están en los documentos
Las fuentes pueden ser irrelevantes: el retrieval devuelve chunks que no tienen respuesta
Sin métricas no sabes si tu RAG funciona bien — solo especulas
📊
Métricas clave
+
Faithfulness: ¿la respuesta es fiel al contexto recuperado? ¿No inventa?
Answer Relevancy: ¿responde realmente a lo que se preguntó?
Context Precision: ¿los documentos recuperados son relevantes para la pregunta?
Context Recall: ¿se recuperó toda la información necesaria para responder?
✍️
Evaluación manual
+
Crear 10-20 pares pregunta-respuesta esperada — casos reales que importan
Ejecutar el RAG con esas preguntas y recopilar respuestas
Comparar respuesta esperada vs. respuesta real
Puntuar cada respuesta del 1 al 5 e identificar patrones de fallo
🤖
RAGAS Framework
+
pip install ragas — framework automático de evaluación
Usa un LLM para evaluar calidad de forma automática
Genera scores de 0 a 1 por cada métrica (Faithfulness, Relevancy, etc.)
Más rápido que evaluación manual para muchos ejemplos
📈
Mejorar basándose en resultados
+
Faithfulness baja: mejorar el prompt del sistema, ser más específico
Context Precision baja: mejorar chunking o agregar reranker (BM25, CrossEncoder)
Context Recall baja: agregar más documentos o reducir chunk_size para más fragmentos
A/B testing: comparar diferentes configuraciones (modelos, parámetros) y ver cuál da mejores scores
