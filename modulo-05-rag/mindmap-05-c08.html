Mind Map — Clase 08

Deploy: Sube tu App RAG a Render

Comparte tu proyecto con el mundo en minutos

🌍
¿Por qué hacer deploy?
+
Compartir tu proyecto con el mundo — amigos, potenciales empleadores, clientes
Imprescindible para portfolio — demuestra que puedes llevar proyectos hasta producción
Demuestra habilidades completas end-to-end — no solo código, sino también infraestructura
📦
Preparar el proyecto
+
requirements.txt — lista todas las dependencias (pip freeze o manual)
.env — guarda API keys (NUNCA en el código hardcodeado)
.gitignore — excluye .env, __pycache__, .venv para que no suban al repo
README.md — instrucciones claras de cómo ejecutar localmente
⚡
Opciones gratuitas
+
Streamlit Community Cloud: el más fácil para Streamlit, gratuito, integración GitHub directa
Render: gratis pero se duerme tras 15 min sin uso, los primeros deployments son rápidos
Hugging Face Spaces: bueno para demos de IA, entorno amigable, muchos templates
🚀
Deploy en Render paso a paso
+
1. Crear repo en GitHub y subir código completo
2. Crear cuenta en render.com (gratis)
3. New Web Service → conectar tu repo de GitHub
4. Configurar: Python 3, comando pip install -r requirements.txt, agregar variables de entorno (API keys como secretos)
5. Deploy — Render construye y ejecuta tu app
🔧
Problemas comunes
+
"Module not found": dependencia que falta en requirements.txt → agrégala y redeploy
API key no funciona: revisar que las variables de entorno estén configuradas en Render dashboard
App crashea: revisar logs en Render dashboard, suelen dar pistas claras
Problemas de memoria: ChromaDB en disco puede dar problemas en free tier, considera usar en-memory o Pinecone
