📖 Las 3 métricas Ragas que importan

MétricaQué mideCómo interpretarla
Faithfulness ¿La respuesta está respaldada por los chunks recuperados? Penaliza hallucination. <0.7 = el modelo inventa. 0.9+ = fiel al contexto.
Answer Relevancy ¿La respuesta responde la pregunta (vs divagar)? <0.6 = el modelo se va por las ramas. 0.8+ = directa.
Context Precision ¿Los chunks recuperados son realmente relevantes? Evalúa el retriever, no el LLM. <0.5 = chunking malo o reranker ausente. 0.8+ = retriever sólido.
Ragas internamente llama a un LLM para estimar cada métrica (descompone en claims, comprueba cobertura, etc.). Por eso ejecutar evals cuesta dinero — presupuéstalo.

Paso 2 · Generar respuestas + contextos con tu RAG

Python — generate_eval_data.py 
from eval_dataset import EVAL_SET
from rag_pipeline import retrieve, rerank, answer  # funciones de la clase 05

results = []
for item in EVAL_SET:
    q = item["question"]
    # Retrieve + rerank (igual que la clase 05)
    candidates = retrieve(q, k=20)
    top = rerank(q, candidates, n=3)
    contexts = [c["text"] for c in top]
    # Genera la respuesta con el mismo pipeline
    resp = answer(q)

    results.append({
        "question": q,
        "answer": resp,
        "contexts": contexts,
        "ground_truth": item["ground_truth"],
    })

import json
with open("eval_results.json", "w") as f:
    json.dump(results, f, ensure_ascii=False, indent=2)
print(f"Guardadas {len(results)} filas con answer + contexts + ground_truth")

Paso 4 · LLM-as-judge con Haiku (sin librería, hecho a mano)

Python — llm_judge.py 
from anthropic import Anthropic
import json

client = Anthropic()
JUDGE_MODEL = "claude-haiku-4-5-20251001"  # barato y suficiente

JUDGE_PROMPT = """Eres un juez imparcial. Recibes una pregunta, una respuesta esperada (ground truth) y una respuesta de un modelo.

Devuelve SOLO JSON con dos campos:
- "correct": true/false — ¿la respuesta del modelo comunica la misma información que la ground truth?
- "reason": 1 frase explicando por qué.

No exijas coincidencia literal. Exige equivalencia semántica.

Pregunta: {question}
Ground truth: {ground_truth}
Respuesta del modelo: {answer}
"""

def judge(question, ground_truth, answer):
    msg = client.messages.create(
        model=JUDGE_MODEL,
        max_tokens=200,
        messages=[{
            "role": "user",
            "content": JUDGE_PROMPT.format(
                question=question,
                ground_truth=ground_truth,
                answer=answer,
            )
        }],
    )
    raw = msg.content[0].text.strip()
    # Quita posibles fences de markdown
    if raw.startswith("```"):
        raw = raw.split("```")[1].lstrip("json\n")
    return json.loads(raw)

# Úsalo sobre los mismos eval_results.json
with open("eval_results.json") as f:
    rows = json.load(f)

scores = []
for r in rows:
    v = judge(r["question"], r["ground_truth"], r["answer"])
    scores.append(v["correct"])
    print(f"{'✓' if v['correct'] else '✗'} {r['question']}")
    print(f"   → {v['reason']}\n")

accuracy = sum(scores) / len(scores)
print(f"LLM-as-judge accuracy: {accuracy:.0%}")

📊 Cómo actuar sobre los resultados

SíntomaDónde actuar
Faithfulness baja El LLM alucina. Baja temperatura, refuerza el system prompt con "responde SOLO con los chunks dados".
Answer relevancy baja El prompt pide "resumen" cuando la pregunta pide "dato puntual". Revisa instrucciones.
Context precision baja El retriever falla. Cambia chunk size, añade/mejora reranker, prueba otro embedding.
Todas altas pero juez humano dice "no" Tu ground_truth está mal escrito o es ambiguo. Revisa el dataset.
Regla de oro: cuando cambies algo en el pipeline (chunk size, rerank n, system prompt), re-ejecuta las evals. El número debe subir. Si baja, revierte. Sin evals, no sabes si has mejorado o empeorado.
