📊 Mind Map: Portfolio Web Personal

Construye tu escaparate profesional paso a paso

🌐 Tu Portfolio Web ▼
¿Qué es?
  • Sitio web que exhibe tu trabajo, habilidades y experiencia
  • Tu escaparate profesional digital
  • Primera impresión para empleadores y clientes
¿Por qué es importante?
  • Diferenciación: el 99% no tiene portfolio profesional
  • Credibilidad: demuestras que sabes lo que haces
  • Control: tuya la narrativa, no dependes de LinkedIn
  • SEO: apareces en Google cuando te buscan
¿Cuándo empieza a funcionar?
  • Mientras estés estudiando: 1-2 proyectos de cursos
  • Mientras trabajas: 3+ proyectos profesionales o personales
  • Ideal: actualizar cada 3-6 meses con nuevo trabajo
📄 Secciones Esenciales ▼
Hero (La Primera Impresión)
  • Nombre prominente y legible
  • Foto/avatar profesional (fondo neutro)
  • Título o tagline: "Full-Stack Developer" o "AI Engineer"
  • Botón CTA: "Ver Proyectos" o "Contacta"
Sobre Mí (La Historia)
  • 2-3 párrafos máximo (la gente no lee)
  • Quién eres, qué te apasiona, por qué programas
  • Evitar: párrafos genéricos, humildad excesiva
  • Conectar con: experiencia, logros pequeños, lo que buscas
Proyectos (El Proof)
  • Mínimo 3, máximo 6-8 (orden: mejor primero)
  • Cada proyecto necesita: screenshot, descripción, tech stack
  • Links: Demo (accesible) + GitHub (código fuente)
  • Brevedad: 1-2 líneas de descripción, no más
Skills (El Stack)
  • Organizar por categorías: Frontend, Backend, DevOps, etc.
  • No listar en párrafo aburrido: usar cards o barras
  • Solo lo que REALMENTE sabes y usas
  • Evitar: listar 50 tecnologías (espanta)
Certificaciones (Validación)
  • Mostrar con logos/badges si es posible
  • Nombre del curso, plataforma, año
  • Solo las relevantes para tu objetivo
Contacto (La Conversión)
  • Email, LinkedIn, GitHub, Twitter (si tienes)
  • Hacer fácil el contacto: botón visible, no escondido
  • Bonus: formulario de contacto simple
🛠️ Cómo Construirlo ▼
Opción 1: GitHub Pages + HTML/CSS/JS
  • ✓ GRATIS (mientras uses .github.io o custom domain)
  • ✓ Control total del código
  • ✓ Rápido: puedes subir en 1-2 horas
  • ⭐ RECOMENDADO si eres principiante
Opción 2: Next.js + Vercel
  • ✓ Más profesional y moderno
  • ✓ Rendimiento optimizado
  • ✓ Deploy automático desde GitHub
  • ⚠ Requiere más setup inicial
Opción 3: Templates Prehechos
  • Hugo, Astro, templates gratuitos de GitHub
  • ✓ Rápido si solo personalizas
  • ⚠ Menos control, posible perder diferenciación
Diseño: Dark Theme Recomendado
  • Fondo oscuro (#0d1117, #1a1a2e), texto claro
  • Acento de color: verde, azul o morado
  • Responsive: funciona en móvil, tablet, desktop
  • Animaciones suaves: no abrumar al usuario
🎥 Vídeo de Presentación (2 min) ▼
¿Por qué incluir un vídeo?
  • Humanización: puts a face to the name
  • Comunicación: demuestra que sabes expresarte
  • Diferenciación: el 99% no lo hace
  • Aumenta 30% probabilidad de ser contactado
Estructura de 2 minutos
  • 0:00-0:20 (20s): Quién eres, qué haces, título
  • 0:20-0:50 (30s): Tu stack técnico, tipos de proyectos
  • 0:50-1:30 (40s): Demo de tu mejor proyecto (screen recording)
  • 1:30-2:00 (30s): Qué buscas + cómo contactarte
Tips Técnicos
  • Luz: frontal, sin sombras en la cara
  • Fondo: pared blanca o blur de escritorio
  • Micrófono: AirPods, headset, o micrófono USB (NO laptop)
  • Ropa: casual-profesional (no camiseta, no traje)
  • Editar con CapCut (gratis, fácil)
Dónde Subirlo
  • YouTube: Unlisted o Público (tu elección)
  • Embeber en tu portfolio (iframe)
  • Compartir en LinkedIn Featured (máxima visibilidad)
  • Bonus: incluir en cada aplicación a ofertas
🎯 Qué Proyectos Mostrar ▼
Criterios de Selección
  • ✓ Completos: versión production, no borradores
  • ✓ Accesibles: demo en vivo, no solo GitHub
  • ✓ Tu trabajo: código tuyo, no clones 1:1
  • ✓ Relevantes: que demuestren tu stack target
Tipos de Proyectos
  • De cursos: si son de buena calidad y los personalizaste
  • Freelance: proyectos reales de clientes
  • Personales: ideas propias (no siempre es mejor)
  • Open source: contribuciones importantes a librerías
Cómo Presentar Cada Proyecto
  • 1. Título: claro y específico
  • 2. Screenshot: imagen de 300x200, mostrando features
  • 3. Descripción: 1-2 líneas, qué problema resuelve
  • 4. Tech Stack: React, Node, MongoDB, etc.
  • 5. Links: Demo (live) + GitHub (código)
Qué Evitar
  • ❌ Proyectos incompletos o "en construcción"
  • ❌ Clones exactos de tutoriales (sin personalización)
  • ❌ 10+ proyectos (abrumar al visitante)
  • ❌ Proyectos de 5 años sin mantenimiento
🚀 Desplegar a Internet ▼
Paso 1: Crear Repo GitHub
  • Nombre: username.github.io
  • Ejemplo: juanquerol.github.io
  • Público (para que sea accesible)
  • Clonar a local y agregar archivos HTML/CSS
Paso 2: Push a GitHub
  • git add .
  • git commit -m "Initial portfolio"
  • git push origin main
Paso 3: Esperar Deploy Automático
  • GitHub Pages se activa automáticamente
  • Accesible en: https://username.github.io
  • Tarda 1-5 minutos en estar online
Paso 4: Dominio Custom (Opcional)
  • Comprar dominio: Namecheap, Google Domains, etc.
  • Apuntar DNS a GitHub Pages
  • Agregar CNAME file en repo
⚙️ Mantenimiento Continuo ▼
Actualizar Cada 3-6 Meses
  • Agregar nuevo proyecto terminado
  • Actualizar skills si aprendiste algo nuevo
  • Quitar proyectos viejos o mediocres
  • Mejorar screenshots, descripciones
Checklist de Mantenimiento
  • ✓ Links de demo funcionan (no 404)
  • ✓ GitHub repos están públicos y con buen README
  • ✓ No hay typos o errores gramaticales
  • ✓ Responsive design: probar en móvil
  • ✓ Consola JavaScript sin errores
Análisis y Mejora
  • Google Analytics: ¿Quién visita? ¿Cuánto tiempo pasan?
  • Feedback: pedirle opinión a mentores, amigos
  • A/B Testing: cambiar algo, medir impacto
  • SEO: mejorar keywords, descripción meta
