📚 Resumen de Conceptos

GitHub ES tu Portfolio

El 87% de los hiring managers revisan GitHub antes de una entrevista. Tu perfil de GitHub no es solo código; es tu marca profesional como desarrollador. Un perfil bien optimizado puede abrir puertas más que cualquier currículum.

Elementos de un Perfil Profesional de GitHub

1. Foto Profesional

  • La misma foto que en LinkedIn para consistencia visual
  • Cara clara, bien iluminada, fondo neutro
  • Resolución mínima 200x200px
  • Evita filtros excesivos; sé profesional

2. Bio Clara (Bio field en tu perfil)

Ejemplo de bios profesionales:

Ejemplos 
AI Developer | Python, React, LangChain | Building intelligent apps

Full-Stack Developer | Node.js + React + PostgreSQL | Open Source Contributor

ML Engineer | PyTorch, TensorFlow, Computer Vision | Passionate about AI Ethics

Backend Specialist | Go, PostgreSQL, Kubernetes | Cloud Infrastructure Enthusiast

Regla: Máximo 160 caracteres. Incluye: especialidad + tech stack + uno valores/intereses.

3. Location, Company, Website Links

  • Location: Ciudad real donde trabajas o estudias (ej: "Madrid, Spain")
  • Company: Tu empresa actual o "Learning AI" si eres estudiante
  • Website: Enlace a tu portfolio web, blog o LinkedIn
  • Email: Puedes agregar un email de contacto (aparecerá en "Contributions")

4. README del Perfil (El Repo Especial username/username)

Es un repositorio público con el mismo nombre que tu usuario. Cuando alguien visita tu perfil, lo ve primero.

Estructura recomendada:

Markdown 
# 👋 Hola, soy [Tu Nombre]

**AI Developer | Python | React | LangChain**

Desarrollo aplicaciones inteligentes usando IA y backend sólido. Apasionado por el código limpio y la experiencia de usuario.

5. Pinned Repos: Estrategia

Puedes "pinear" (fijar) hasta 6 repositorios en tu perfil. Estos aparecen arriba del todo.

Mezcla estratégica recomendada:

Tipo Cantidad Ejemplo Por qué
Proyectos Portfolio 2 ChatGPT Clone, SaaS completo Demuestra capacidad full-stack
Open Source 1 PR merged a FastAPI, Django Prueba que colaboras en comunidad
Proyecto Personal/Fun 1 Game, Script útil, Herramienta Muestra creatividad y pasión
Libre (2-3 adicionales) 2-3 Contribuciones, Experimentos, Templates Demuestra amplitud de habilidades

Cada repo pinneado debe tener:

  • README.md pulido (clase anterior)
  • Descripción clara en el repo (editar en settings)
  • Topics/tags relevantes (#python, #react, #ai, etc.)
  • Licencia explícita (MIT, Apache 2.0, GPL)

6. Contribution Graph (El Calendario Verde)

No se trata de commits vacíos, sino de actividad consistente real:

  • Contribuciones a open source: PRs, issues, documentación
  • Proyectos personales: Commits regulares, incluso proyectos pequeños
  • Consistencia: Es mejor tener commits pequeños regulares que 1 mega-commit al mes
  • Lo que cuenta como "contribución": Commits a cualquier rama, PRs abiertas, issues creadas, código en repositorios públicos

⚙️ Práctica Guiada: Optimiza tu Perfil de GitHub

Sigue estos pasos para convertir tu perfil de GitHub en un portfolio profesional.

Paso 1: Crea el README Especial (username/username)

En GitHub, crea un repositorio con exactamente tu nombre de usuario:

  • Si tu usuario es juanpayero, el repo debe ser juanpayero/juanpayero
  • Marcar como Public
  • Marcar como "Add a README.md" al crear

GitHub lo detectará automáticamente y mostrará este README en la parte superior de tu perfil.

Paso 2: Estructura tu README con Markdown

Markdown — README.md 
# 👋 Hola, Soy Juan Payero

**Full-Stack Developer | Python | React | Cloud Architecture**

Desarrollo aplicaciones web escalables y sistemas con IA integrada. Apasionado por la calidad del código y la experiencia del usuario.

Paso 3: Pineados tus 6 Mejores Repos

En tu perfil de GitHub:

  1. Ve a tu perfil (github.com/tuusuario)
  2. Debajo de tu nombre, haz clic en "Customize your pins"
  3. Selecciona exactamente 6 repositorios públicos
  4. Ordénalos por importancia (drag & drop)
  5. Guarda los cambios

Recomendación de orden:

  1. Tu proyecto más impresionante (más stars, mejor README)
  2. Otro proyecto portfolio sólido
  3. Contribución a open source (PR merged)
  4. Proyecto personal/fun que muestra creatividad
  5. Proyecto reciente o en desarrollo
  6. Herramienta o librería útil

Paso 4: Actualiza tu Bio, Location y Links

En Settings > Profile:

  • Name: Tu nombre completo (ej: "Juan Payero")
  • Bio: 160 caracteres máximo (ej: "AI Developer | Python, React, LangChain | Building intelligent apps")
  • Company: Tu empresa actual o "Learning AI"
  • Location: Tu ciudad (ej: "Madrid, Spain")
  • Website: URL a tu portfolio o blog

Paso 5: Agrega Topics a Tus Repos

Cada repositorio puede tener tags. Para cada repo importante:

  1. Abre el repo
  2. Ve a "About" (ícono de engranaje arriba a la derecha)
  3. Agrega topics: #python #react #fastapi #langchain #ai

Topics recomendados (según tu stack):

Ejemplo 
python, fastapi, langchain, ai, backend, rest-api
react, typescript, frontend, tailwind-css, web-development
full-stack, docker, kubernetes, aws, devops
open-source, machine-learning, computer-vision

Paso 6: Certifica que tus READMEs son Profesionales

Cada repositorio pinneado debe tener un README con:

  • Título y descripción clara — ¿Qué hace?
  • Screenshot o demo — Una imagen vale mil palabras
  • Instalación en 3 pasos — Fácil de reproducir
  • Uso/Ejemplos — Código funcionando
  • Tech Stack — Qué tecnologías usa
  • Features principales — Viñetas de qué hace
  • Cómo contribuir — Abierto a PRs
  • Licencia — MIT, Apache 2.0, GPL, etc.
