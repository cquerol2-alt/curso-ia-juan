🎯 Tu GitHub ES Tu Portfolio
  • 87% de hiring managers revisan tu GitHub antes de entrevista
  • Primera impresión digital más importante que CV
  • GitHub complementa CV: demuestra habilidades reales
  • Un perfil GitHub bien estructurado = diferencial en procesos de selección
  • Los reclutadores juran más en código real que en promesas de CV
📸 Foto y Bio
Foto Profesional (Avatar)
• Consistente con LinkedIn (misma foto)
• Fondo limpio, buena iluminación
• Que se vea clara en tamaño 64x64px
• Sonrisa natural, profesional pero amigable
Bio (160 caracteres max)
• Formato: [Rol] | [Top 2-3 Skills] | [Diferenciador]
• Ejemplo: "AI Developer | Python, FastAPI, LLMs | Building intelligent solutions"
• Keywords: AI, Developer, Python, React, Full-Stack
• Emojis permitidos pero usa solo 1-2
Ubicación + Links
• Location: tu ciudad (búsquedas locales)
• Website: link a tu portfolio web
• Email: visible en perfil público
• Social: Twitter/LinkedIn (red profesional)
📄 README del Perfil (username/username)
¿Qué es? Repo especial
• Crear repo: username/username (ej: juanquerol/juanquerol)
• GitHub lo detecta automáticamente
• Se muestra en tu perfil como "Presentación"
• Visible a todos, primera impresión crucial
Estructura Recomendada
• H1: Saludo + emoji personalizado
• About: 2-3 párrafos sobre quién eres
• Tech Stack: badges con tecnologías (shields.io)
• Proyectos Destacados: 3-4 con descripción
• GitHub Stats: gráficos de actividad
• Contacto: links a email, LinkedIn, Twitter
Herramientas Útiles
• GitHub Profile README Generator: rapidg.github.io
• shields.io: badges personalizados
• readme.so: editor visual simple
• Markdown: headers, listas, emojis, links

📌 Pinned Repos (Máximo 6)
Estrategia de Selección
• Mezcla: Portfolio + Open Source + Personal
• Orden: Los 2-3 mejores primero (recruiters no bajan)
• Mínimo 4 repos pinned (6 máximo)
• NO pinear repos vacíos o incompletos
Tipos de Repos a Pinear
• 1-2 Portfolio: tu mejor proyecto con demo live
• 1-2 Open Source: contribuciones destacadas
• 1-2 Personal: proyectos de aprendizaje relevantes
• Total: 4-6, máximo variedad de skills
README Profesional (CADA repo)
• Título + descripción clara
• 📋 Tabla de contenidos
• 🎯 Features principales
• 🛠️ Tech stack usado
• 🚀 Cómo ejecutar (instalación + demo)
• 📸 Screenshot o GIF demo
• 📄 Licencia + Contacto
Cómo Pinear
• Ir a tu perfil GitHub
• Repos: mostrar 6 pinned
• Click en repo → "Pin to profile"
• Reordenar: arrastra icons
• Máximo 6 (GitHub limita)
📊 Contribution Graph
Consistencia > Cantidad
• Verde cada día = dedicación
• Reclutadores valoran continuidad
• 1 commit/día relevante > 20 commits vacíos
• Semanas completas verdes impactan más
Cómo Mantener Consistencia
• Commits: incluso cambios pequeños cuentan
• Dailies: problemas de CodeWars, Leetcode
• Proyectos personales: commit de feature/fix/doc
• Open Source: issues, PRs, comentarios
Contribuciones que Cuentan
• ✅ Commits a repos propios
• ✅ PRs aceptados en open source
• ✅ Issues creados/resueltos
• ✅ Documentación
• ✅ Código review en otros repos
• ⚠️ Solo cuenta en repos públicos
Impacto Visual
• Perfil con "graph verde" = comprometido
• Gaps visibles = abandono percibido
• Mes completo verde = muy atractivo
• GitHub Stats cardan con ésta info

✅ Checklist: Perfil GitHub Completo
  • ☐ Foto de perfil profesional (64x64px mínimo)
  • ☐ Bio (160 chars): [Rol | Skills | Diferenciador]
  • ☐ Location + Website + Email visibles
  • ☐ README perfil especial (username/username creado)
  • ☐ README con saludo, about, tech stack badges
  • ☐ GitHub Stats y Streak Stats embebidos
  • ☐ 4-6 repos pinned (mejores primero)
  • ☐ Cada repo pinned tiene README profesional
  • ☐ README repos: descripción, tech stack, cómo ejecutar, demo
  • ☐ Contribuciones diarias consistentes (verde)
  • ☐ Al menos 1 repo con actividad reciente
  • ☐ Perfiles en GitHub, LinkedIn y Twitter enlazados
🚩 Errores Comunes & 🎯 Best Practices
❌ Errores Frecuentes
• Bio genérica: "Student" (sin skills)
• README perfil vacío o solo markdown
• Repos pinned sin documentación
• Contribution graph con gaps grandes
• Repos viejos/abandonados pinned
• Sin portfolio/demo projects visibles
• Links rotos en README
• Código de calidad mala (copy-paste del tutorial)
✅ Best Practices
• Bio especifica: rol + 2-3 skills principales
• README completo: saludo + about + tech + projects
• Repos documentados profesionalmente
• Contribuciones consistentes (verde diario)
• Pinear solo lo mejor (quality > quantity)
• Mix: personal + open source + portfolio
• Stats embebidos (GitHub Readme Stats)
• Código limpio, buenas prácticas
💡 Conclusión: Tu GitHub es tu portfolio interactivo. Un perfil bien estructurado con README llamativo, repos documentados y contribuciones consistentes es un imán de oportunidades. Invierte 3 horas ahora: bio + README + 6 repos pinned. Reclutadores lo verán durante meses.
