Mind Map: OpenAI Function Calling / Tool Use

← Volver a la Clase 02
Function Calling ▼
Qué es: Capacidad del LLM de invocar funciones externas
Por qué: Acceso a datos reales, ejecución de acciones, integración con APIs
GPT 3.5+: Soporta function calling nativamente
Ventaja: LLM decide cuándo y cómo usar herramientas
Flujo de Ejecución ▼
1. Prompt: Usuario envía mensaje + definiciones de tools
2. Tool Call: LLM responde con tool_call (nombre + parámetros)
3. Execute: Cliente ejecuta la función localmente
4. Result: Envía resultado al LLM en el historial
5. Response: LLM formula respuesta final basada en resultado
JSON Schema ▼
name: Identificador único de la función
description: Qué hace la función (LLM la lee)
parameters: Definición de argumentos (type, properties, required)
strict: Si true, LLM respeta schema exactamente
Ejemplo: name: "search", description: "Buscar en web", params: {query}
tool_choice Parameter ▼
"auto": LLM elige si usar tool o no (por defecto)
"required": LLM DEBE usar una tool
"none": LLM NO usa tools, solo genera texto
Especifica: {"type": "function", "function": {"name": "search"}}
Parallel Tool Calls ▼
Multiple Calls: LLM puede solicitar varias tools a la vez
Eficiencia: Ejecuta todas en paralelo, luego envía resultados
Caso: Buscar 3 productos, 3 precios, 3 reviews en una sola petición
Implementation: Procesar todas las tool_calls antes de siguiente vuelta
Agent Loop ▼
While Loop: Mientras haya tool_calls, ejecuta
Condición: Si LLM responde "tool_calls", continúa ciclo
Si LLM responde "text": Retorna respuesta, sale del loop
Max Iterations: Limite seguridad para evitar loops infinitos
Structured Outputs (Strict Mode) ▼
Objetivo: Garantizar que LLM respete el schema exactamente
Validación: LLM genera JSON que pasa schema validation
GPT-4 Turbo+: Soporta strict mode en function calling
Ventaja: Parseo sin errores, código más robusto
Errores Comunes ▼
Schema Incorrecto: Parámetros mal definidos → LLM confundido
No Enviar Resultado: Tool se ejecuta pero no vuelve al LLM → agente desorienta
Loop Infinito: LLM siempre pide misma tool, sin condición de salida
Sin Manejo Errores: Tool falla → crash, no le dice al LLM
