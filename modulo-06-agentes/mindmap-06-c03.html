🎯 CrewAI

Framework Python para construir sistemas multi-agente.
Orquestación: Coordina múltiples agentes que colaboran.
Ventaja vs 1 Agente: Especialización, delegación, mejor explicabilidad.
Casos de uso: Investigación, análisis, generación de contenido, automatización compleja.

⚙️ Componentes Core

Agent: Entidad autónoma especializada.
Role, Goal, Backstory, Tools, Memory
Task: Trabajo específico.
Description, Expected Output, Agent, Tools
Crew: Orquestador del equipo.
Agents, Tasks, Process (Sequential/Hierarchical), Verbose
Tools: Acciones que pueden usar.
SerperDevTool, ScrapeWebsiteTool, FileReadTool, Custom Tools

🤖 Agent Deep Dive

Role: Su función en el equipo.
ej: "Researcher", "Writer", "Analyst"
Goal: Objetivo principal que persigue.
Define qué es éxito para este agente
Backstory: Contexto y personalidad.
Influye en cómo actúa, toma decisiones, responde
Tools: Herramientas disponibles.
Búsqueda, web scraping, archivos, código, custom
Memory: Acceso a contexto.
Short-term (conversación actual), Long-term (persistente)

✅ Task Deep Dive

Description: Qué debe hacer.
Explícita, detallada, sin ambigüedad
Expected Output: Formato esperado.
Define estructura y contenido del resultado
Agent: Quién ejecuta.
Asignación específica a un agente del crew
Tools: Herramientas para esta tarea.
Subset de tools del agent si es necesario
Orden: Secuencia en la ejecución.
Cada task ve outputs de tasks anteriores

🔄 Procesos de Ejecución

Sequential:
Task1 → Task2 → Task3
Cada task espera a la anterior
Bueno para workflows lineales
Hierarchical:
Manager coordina agentes
Permite paralelización
Para equipos grandes y complejos
Verbose Mode:
Muestra logs detallados de cada paso
Ideal para debug y aprendizaje

🛠️ Herramientas Disponibles

SerperDevTool: Búsqueda en Google (requiere API key)
ScrapeWebsiteTool: Extraer contenido de páginas
FileReadTool: Leer archivos locales
FileWriteTool: Crear/escribir archivos
DirectoryReadTool: Listar directorios
CodeInterpreterTool: Ejecutar código Python
Custom Tools: Crear con @tool decorator

💾 Memory en CrewAI

Short-term Memory:
Contexto de conversación actual
Se borra al terminar sesión
Long-term Memory:
Información persistente entre sesiones
Requiere base de datos
Entity Memory:
Recuerda entidades específicas
Personas, empresas, conceptos clave

🚀 Patrones Avanzados

Delegación:
Un agente pide ayuda a otro
Descomposición elegante de problemas
Retroalimentación:
Refinar outputs iterativamente
Quality control integrado
Specialización:
Cada agente = expertise específica
Better results que 1 agente genérico
Callbacks:
Hook en eventos del crew
Logging, monitoring, integración
