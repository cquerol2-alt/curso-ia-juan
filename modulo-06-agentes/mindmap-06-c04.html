Mind Map: LangGraph — Agentes con Estado

← Volver a la Clase 04
LangGraph ▼
Qué es: Framework para construir agentes con estado persistente
Cómo: Define nodos (funciones) y aristas (transiciones) en un grafo
Cuándo usar: Agentes complejos, flujos condicionados, human-in-the-loop
Ventaja: Control explícito del flujo, debugging más fácil
Conceptos Clave ▼
State: Objeto que fluye por el grafo, contiene contexto
Node: Función que procesa el state y lo modifica
Edge: Conexión entre nodos, define flujo secuencial
Conditional Edge: Decide siguiendo nodo basado en state
StateGraph ▼
TypedDict: Esquema Python que define estructura del state
Campos: messages (list), tools_called (dict), result (str), etc.
Shared State: Todos los nodos acceden al mismo state
Actualizaciones: Cada nodo retorna cambios parciales que se mergean
Nodos ▼
Funciones Python: Reciben state, retornan cambios
Firma: def my_node(state: AgentState) → dict
Dentro: Lógica arbitraria, llamadas LLM, tools, etc.
Retorno: Dict con claves que actualizan el state
Routing (Conditional Edges) ▼
Función: Examina state y decide siguiente nodo
should_continue: ¿Hay tool_calls? → "tools" : "end"
Sintaxis: add_conditional_edges(source_node, routing_fn, path_map)
Ejemplo: Si state["messages"][-1].tool_calls → nodo "execute_tools"
Patrón ReAct ▼
Reason: Nodo LLM piensa y decide qué tool usar
Act: Nodo ejecuta la tool y captura resultado
Observe: Resultado se añade al state (messages)
Loop: Si hay más tool_calls → vuelve a Reason
Checkpointing ▼
Guardar Estado: LangGraph puede persistir snapshots
Reanudar: Reiniciar desde checkpoint anterior
Caso Uso: Investigación larga, debugging, auditoría
Storage: Memory, SQLite, PostgreSQL, Redis, etc.
Human-in-the-Loop ▼
Interrupciones: Pausar en ciertos nodos (e.g., antes de ejecutar tool)
Aprobación Manual: Humano revisa y aprueba antes de continuar
Seguridad: Evita acciones peligrosas (borrar, enviar email)
Sintaxis: add_node(..., interrupt_before=True)
