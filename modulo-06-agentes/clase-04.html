📖 Resumen de conceptos

¿Qué es LangGraph?

LangGraph es un framework de LangChain que permite construir agentes con grafos de estado. A diferencia de agentes simples, LangGraph te da control total sobre:

  • El flujo de ejecución (qué nodo se ejecuta después de cuál)
  • El estado compartido entre pasos
  • Las decisiones condicionales del agente
  • La posibilidad de pausar y reanudar (checkpointing)

Conceptos Clave

State (Estado): Un TypedDict que define la estructura de datos compartida entre nodos. Por ejemplo, qué información mantiene el agente mientras trabaja.
Python 
from typing import TypedDict

class AgentState(TypedDict):
    input: str                    # La pregunta del usuario
    messages: list                # Historial de mensajes
    next: str                     # Qué nodo ejecutar siguiente
Nodes (Nodos): Funciones Python que reciben el estado, lo procesan, y retornan un estado actualizado. Son los "pasos" del grafo.
Python 
def process_node(state: AgentState) -> AgentState:
    # Recibe estado, lo procesa
    messages = state["messages"]
    # ... lógica del agente ...

    # Retorna estado actualizado
    return {"messages": messages, "next": "siguiente_nodo"}
Edges (Aristas): Conexiones entre nodos que definen el flujo. Las aristas pueden ser simples (A → B siempre) o condicionales (si X, entonces A → B; si no, A → C).

El Grafo como Flujo de Trabajo

Un grafo típico de LangGraph funciona así:

  1. Nodo de entrada: Recibe la pregunta del usuario
  2. Nodo de proceso: El agente analiza la pregunta y decide qué hacer
  3. Decisión condicional: ¿Necesito herramientas o puedo responder directamente?
  4. Nodos de acción: Si necesita herramientas, las ejecuta
  5. Bucle o salida: ¿Necesito más información? Si sí, repite. Si no, retorna la respuesta.

StateGraph: Definiendo el Grafo

En LangGraph defines el grafo con StateGraph, donde especificas el estado, los nodos y las aristas:

Python 
from langgraph.graph import StateGraph

# 1. Crear el grafo
workflow = StateGraph(AgentState)

# 2. Añadir nodos
workflow.add_node("analyze", analyze_node)
workflow.add_node("search", search_node)
workflow.add_node("respond", respond_node)

# 3. Añadir aristas (conexiones)
workflow.add_edge("START", "analyze")           # Inicio → analyze
workflow.add_conditional_edges(                 # Decisión
    "analyze",
    decide_action,                              # Función que decide
    {
        "search": "search",                     # Si busca, va a search
        "respond": "respond"                    # Si responde, va a respond
    }
)
workflow.add_edge("search", "respond")          # search → respond

# 4. Compilar
graph = workflow.compile()

Conditional Edges: El Agente Decide

Las aristas condicionales permiten que el agente tome decisiones. Una función retorna el siguiente nodo basado en el estado actual:

Python 
def decide_action(state: AgentState) -> str:
    """Retorna el siguiente nodo según el estado"""
    # Analizar el mensaje del agente
    last_message = state["messages"][-1]

    if "búsqueda" in last_message.content:
        return "search"
    else:
        return "respond"

Checkpointing: Guardar Estado para Reanudar

LangGraph soporta checkpointing, que permite pausar un grafo, guardar su estado, y reanudarlo después. Útil para agentes de larga duración.

Python 
from langgraph.checkpoint.memory import MemorySaver

# Crear grafo con checkpointing
checkpointer = MemorySaver()
graph = workflow.compile(checkpointer=checkpointer)

# Ejecutar y guardar checkpoint
result = graph.invoke(
    initial_input,
    config={"configurable": {"thread_id": "user_123"}}
)

Human-in-the-Loop: Pausar para Aprobación

Puedes pausar el grafo en ciertos puntos para que un humano apruebe acciones antes de continuar:

Python 
from langgraph.types import interrupt

def action_node(state: AgentState) -> AgentState:
    action = {"tool": "search", "query": "..."}

    # Pausar y esperar aprobación humana
    interrupt(action)  # Usuario aprueba o rechaza

    # Continuar si aprobó
    return state

LangGraph vs CrewAI: Comparación Rápida

Aspecto LangGraph CrewAI
Enfoque Control total del grafo de estado Equipos de agentes con roles específicos
Curva de aprendizaje Media (necesitas diseñar el grafo) Baja (más abstracta, menos control)
Flexibilidad Muy alta (defines cada aspecto) Media (templated, menos customizable)
Mejor para Flujos complejos, custom, research agents Tareas estructuradas, múltiples agentes
Human-in-the-loop Soporte nativo (interrupts) Soporte limitado

Patrón ReAct: Reason + Act en Loop

ReAct (Reasoning + Acting) es un patrón común en LangGraph donde el agente:

  1. Reason: Piensa qué acción tomar
  2. Act: Ejecuta una herramienta
  3. Observe: Ve el resultado
  4. Loop: Repite hasta tener respuesta final
Python 
# El grafo ReAct
# START → think → decide_action → (tool_1 | tool_2 | respond)
#                                      ↓
#                                  observe (loop back to think)

def should_continue(state: AgentState) -> str:
    """Decide si continuar el loop o terminar"""
    if state["final_answer"]:
        return "respond"
    else:
        return "think"  # Continuar razonando

💻 Práctica guiada

Sigue estos pasos para construir tu primer agente con LangGraph. Escribe el código en un archivo langgraph_demo.py.

Step 1: Instalar dependencias

Bash 
pip install langgraph langchain-openai langchain-core

Step 2: Grafo simple con 2 nodos

Crea un grafo que tenga un nodo inicial y uno de respuesta:

Python 
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# 1. Definir el estado
class SimpleState(TypedDict):
    input: str
    output: str

# 2. Definir nodos (funciones)
def process_node(state: SimpleState) -> SimpleState:
    # Procesar input
    output = f"Procesado: {state['input'].upper()}"
    return {"input": state["input"], "output": output}

# 3. Construir el grafo
workflow = StateGraph(SimpleState)
workflow.add_node("process", process_node)
workflow.add_edge(START, "process")
workflow.add_edge("process", END)

# 4. Compilar y ejecutar
graph = workflow.compile()

result = graph.invoke({"input": "hola mundo"})
print(result)  # {'input': 'hola mundo', 'output': 'Procesado: HOLA MUNDO'}

Step 3: Añadir enrutamiento condicional

Ahora el agente decide si buscar información o responder directamente:

Python 
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, START, END

class AgentState(TypedDict):
    question: str
    needs_search: bool
    search_result: str
    final_answer: str

def analyze_node(state: AgentState) -> AgentState:
    """Analiza la pregunta y decide si buscar"""
    question = state["question"]

    # Simple heuristic: si tiene "qué es" o "cuál es", busca
    needs_search = "qué es" in question.lower() or "cuál es" in question.lower()

    return {
        **state,
        "needs_search": needs_search,
        "final_answer": ""
    }

def search_node(state: AgentState) -> AgentState:
    """Simula una búsqueda web"""
    search_result = f"Resultado de búsqueda para: {state['question']}"
    return {**state, "search_result": search_result}

def respond_node(state: AgentState) -> AgentState:
    """Genera la respuesta final"""
    if state["needs_search"]:
        answer = f"Basándome en: {state['search_result']}"
    else:
        answer = f"Sé que: {state['question']}"

    return {**state, "final_answer": answer}

def decide_action(state: AgentState) -> Literal["search", "respond"]:
    """Enrutamiento condicional"""
    if state["needs_search"]:
        return "search"
    else:
        return "respond"

# Construir
workflow = StateGraph(AgentState)
workflow.add_node("analyze", analyze_node)
workflow.add_node("search", search_node)
workflow.add_node("respond", respond_node)

workflow.add_edge(START, "analyze")
workflow.add_conditional_edges(
    "analyze",
    decide_action,
    {"search": "search", "respond": "respond"}
)
workflow.add_edge("search", "respond")
workflow.add_edge("respond", END)

graph = workflow.compile()

# Test
result = graph.invoke({"question": "¿Qué es LangGraph?"})
print(result["final_answer"])
# Output: Basándome en: Resultado de búsqueda para: ¿Qué es LangGraph?

Step 4: Agente ReAct completo con herramientas

Ahora un agente completo que puede usar múltiples herramientas en loop:

Python 
from typing import TypedDict, Literal, Any
from langgraph.graph import StateGraph, START, END
from langchain_openai import ChatOpenAI

# 1. Definir estado
class ReactState(TypedDict):
    question: str
    messages: list
    tools_used: list
    final_answer: str

# 2. Herramientas simuladas
def search_tool(query: str) -> str:
    return f"Resultado web para '{query}': información relevante aquí"

def math_tool(expression: str) -> str:
    try:
        result = eval(expression)
        return f"Resultado: {result}"
    except:
        return "Error en el cálculo"

# 3. Nodo Think: El agente decide qué hacer
def think_node(state: ReactState) -> ReactState:
    llm = ChatOpenAI(model="gpt-4o-mini")

    # Prompt: decide qué herramienta usar o si responder
    system = """Eres un asistente. Decide:
    1. search(query) - buscar en web
    2. math(expression) - calcular
    3. answer(text) - responder directamente
    Responde SOLO con: search(...), math(...) o answer(...)"""

    prompt = f"Pregunta: {state['question']}"

    # En implementación real, parsearias la respuesta del LLM
    # Para demo, decision simple:
    if "cuánto es" in state["question"].lower():
        decision = "math(2+2)"
    else:
        decision = f"search({state['question'][:20]})"

    return {**state, "messages": [*state["messages"], decision]}

# 4. Nodo de herramientas
def execute_tool(state: ReactState) -> ReactState:
    last_action = state["messages"][-1]

    if "search(" in last_action:
        query = last_action.split("(")[1].split(")")[0]
        result = search_tool(query)
    elif "math(" in last_action:
        expr = last_action.split("(")[1].split(")")[0]
        result = math_tool(expr)
    else:
        result = "Sin acción"

    return {
        **state,
        "messages": [*state["messages"], result],
        "tools_used": [*state["tools_used"], last_action]
    }

# 5. Nodo de respuesta final
def answer_node(state: ReactState) -> ReactState:
    # Combinar toda la información para responder
    final = f"Respuesta a '{state['question']}': {state['messages'][-1]}"
    return {**state, "final_answer": final}

# 6. Función de decisión: ¿continuar loop o responder?
def should_continue(state: ReactState) -> Literal["tool", "answer"]:
    # Simple: máximo 3 herramientas
    if len(state["tools_used"]) >= 3:
        return "answer"

    last_message = state["messages"][-1]
    if "answer(" in last_message:
        return "answer"
    else:
        return "tool"

# 7. Compilar grafo
workflow = StateGraph(ReactState)
workflow.add_node("think", think_node)
workflow.add_node("tool", execute_tool)
workflow.add_node("answer", answer_node)

workflow.add_edge(START, "think")
workflow.add_conditional_edges(
    "think",
    should_continue,
    {"tool": "tool", "answer": "answer"}
)
workflow.add_edge("tool", "think")  # Loop back
workflow.add_edge("answer", END)

graph = workflow.compile()

# Test
result = graph.invoke({
    "question": "¿Cuánto es 10 + 5?",
    "messages": [],
    "tools_used": [],
    "final_answer": ""
})

print(result["final_answer"])

🎯 Mini-reto del día

🔍 Research Agent con Decisión Automática

Construye un agente que automáticamente decide si necesita buscar información o puede responder directamente:

Requerimientos:

  • 3 nodos: classify, search, answer
  • El nodo classify analiza la pregunta y decide: ¿es una pregunta de conocimiento general (responde) o de información actual (busca)?
  • Si busca, simula una búsqueda web
  • Si responde, genera una respuesta basada en lo que sabe
  • El flujo debe ser: START → classify → (search OR answer) → END

Ejemplos de preguntas:

  • "¿Quién ganó el último Mundial?" → buscar (información actual)
  • "¿Cuál es la capital de España?" → responder (conocimiento general)
  • "¿Qué es Python?" → responder
  • "¿Cuáles son las últimas noticias de tecnología?" → buscar

Bonus: Añade checkpointing para poder pausar y reanudar el grafo.
