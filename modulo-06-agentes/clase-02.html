📋 Resumen de Conceptos Clave

¿Qué es Function Calling / Tool Use?

Function Calling (o Tool Use) es la capacidad de un modelo de lenguaje para determinar qué función o herramienta externa debe ejecutarse, basándose en el input del usuario y el contexto. El modelo NO ejecuta la función directamente — en su lugar, describe cuál es la mejor función a llamar y qué argumentos pasarle. Tu código es responsable de ejecutarla y devolver el resultado.

El Flujo de Function Calling

  1. Usuario envía prompt: "¿Cuál es el tiempo en Barcelona?"
  2. Modelo analiza: Reconoce que necesita llamar a get_weather
  3. Modelo sugiere tool_call: Devuelve {"name": "get_weather", "arguments": {"city": "Barcelona"}}
  4. Tu código ejecuta: Llamas a tu función get_weather("Barcelona")
  5. Devuelves resultado: Envías el resultado (ej: "24°C, soleado") de vuelta al modelo
  6. Modelo genera respuesta final: "En Barcelona hace 24°C y está soleado"

Definir Herramientas con JSON Schema

Las herramientas se definen usando JSON Schema en el parámetro tools:

tools = [ { "type": "function", "function": { "name": "get_weather", "description": "Obtiene el tiempo actual de una ciudad", "parameters": { "type": "object", "properties": { "city": { "type": "string", "description": "Nombre de la ciudad" }, "units": { "type": "string", "enum": ["celsius", "fahrenheit"], "description": "Unidad de temperatura" } }, "required": ["city"] } } } ]

Parámetros Clave de tool_choice

  • auto (defecto): El modelo decide si llamar herramientas o no
  • required: Fuerza al modelo a SIEMPRE usar una herramienta
  • none: El modelo NUNCA usa herramientas
  • Específica: Fuerza el uso de una herramienta concreta: {"type": "function", "function": {"name": "get_weather"}}

Tabla: Parámetros Clave de Herramientas

Parámetro Tipo Descripción Ejemplo
name string Identificador único de la herramienta (snake_case) get_weather
description string Explicación clara de qué hace la herramienta Obtiene el tiempo actual
parameters JSON Schema Define los argumentos que acepta la función {"type": "object", "properties": {...}}
strict boolean Si es true, el modelo SIEMPRE devuelve JSON válido según el schema true

Múltiples Herramientas en una Llamada (Parallel Tool Calls)

El modelo puede sugerir varias herramientas en una sola respuesta. Por ejemplo, si le pides "Calcula 2+2 y dame el tiempo en Madrid", el modelo puede devolver:

tool_calls = [ { "name": "calculator", "arguments": {"operation": "+", "a": 2, "b": 2} }, { "name": "get_weather", "arguments": {"city": "Madrid"} } ]

Errores Comunes y Cómo Evitarlos

  • Descripción vaga: Si el modelo no entiende bien qué hace una herramienta, la llamará mal o no la usará. Sé específico.
  • Schema incompleto: Siempre incluye required para los parámetros obligatorios.
  • No manejar errores: Si una función falla, devuelve el error al modelo para que lo intente de otra forma.
  • Loop infinito: Asegúrate de que el agente eventualmente deje de llamar herramientas y genere una respuesta final.
  • Datos sensibles: NUNCA expongas API keys o datos confidenciales en las tool_calls — manéjalos en el backend.

💻 Práctica Guiada: Mi Primer Agente con Function Calling

Step 2: Primera Función — get_weather con Function Calling

Crea agent_weather.py:

import json from openai import OpenAI from dotenv import load_dotenv load_dotenv() client = OpenAI() # Definir la herramienta tools = [ { "type": "function", "function": { "name": "get_weather", "description": "Obtiene el tiempo actual para una ciudad", "parameters": { "type": "object", "properties": { "city": { "type": "string", "description": "Nombre de la ciudad (ej: Barcelona)" }, "units": { "type": "string", "enum": ["celsius", "fahrenheit"], "description": "Unidad de temperatura (default: celsius)" } }, "required": ["city"] } } } ] # Función real que simula una API de tiempo def get_weather(city, units="celsius"): """Simula una llamada a una API de tiempo real""" weather_data = { "Barcelona": {"temp": 24, "condition": "soleado"}, "Madrid": {"temp": 22, "condition": "nublado"}, "Valencia": {"temp": 26, "condition": "parcialmente nublado"} } if city not in weather_data: return f"No tengo datos del tiempo para {city}" data = weather_data[city] if units == "fahrenheit": data["temp"] = int(data["temp"] * 9/5 + 32) return f"{city}: {data['temp']}°{units[0].upper()}, {data['condition']}" # Llamada al modelo def chat_with_function_calling(user_message): print(f"\n👤 Usuario: {user_message}\n") messages = [{"role": "user", "content": user_message}] # Primera llamada al modelo response = client.chat.completions.create( model="gpt-4o-mini", messages=messages, tools=tools, tool_choice="auto" ) # Procesar la respuesta if response.choices[0].finish_reason == "tool_calls": # El modelo quiere usar una herramienta tool_calls = response.choices[0].message.tool_calls for tool_call in tool_calls: tool_name = tool_call.function.name tool_args = json.loads(tool_call.function.arguments) print(f"🔧 El modelo llamó a: {tool_name}") print(f" Argumentos: {tool_args}") # Ejecutar la función real if tool_name == "get_weather": result = get_weather(**tool_args) else: result = "Función desconocida" print(f" Resultado: {result}\n") # Enviar el resultado de vuelta al modelo messages.append({"role": "assistant", "content": response.choices[0].message.content}) messages.append({ "role": "tool", "tool_call_id": tool_call.id, "content": result }) # Segunda llamada: el modelo genera la respuesta final final_response = client.chat.completions.create( model="gpt-4o-mini", messages=messages ) answer = final_response.choices[0].message.content else: # El modelo no necesitó herramientas answer = response.choices[0].message.content print(f"🤖 Asistente: {answer}") return answer # Test if __name__ == "__main__": chat_with_function_calling("¿Cuál es el tiempo en Barcelona?") chat_with_function_calling("Dime el tiempo en Madrid y Valencia en Fahrenheit")

Step 3: Agent Loop Completo

Crea agent_loop.py para un agente que pueda hacer múltiples llamadas:

import json from openai import OpenAI from dotenv import load_dotenv load_dotenv() client = OpenAI() tools = [ { "type": "function", "function": { "name": "get_weather", "description": "Obtiene el tiempo actual para una ciudad", "parameters": { "type": "object", "properties": { "city": {"type": "string", "description": "Nombre de la ciudad"} }, "required": ["city"] } } } ] def get_weather(city): weather_db = { "Barcelona": "24°C, soleado", "Madrid": "22°C, nublado" } return weather_db.get(city, f"Datos no disponibles para {city}") def run_agent(user_message, max_iterations=10): """Ejecuta el agent loop completo""" print(f"\n👤 Usuario: {user_message}\n") messages = [{"role": "user", "content": user_message}] iteration = 0 while iteration < max_iterations: iteration += 1 print(f"[Iteración {iteration}]") response = client.chat.completions.create( model="gpt-4o-mini", messages=messages, tools=tools, tool_choice="auto" ) if response.choices[0].finish_reason == "tool_calls": # Procesar tool_calls messages.append(response.choices[0].message) for tool_call in response.choices[0].message.tool_calls: tool_name = tool_call.function.name tool_args = json.loads(tool_call.function.arguments) print(f" 🔧 Llamando a {tool_name} con {tool_args}") if tool_name == "get_weather": result = get_weather(**tool_args) else: result = "Desconocida" messages.append({ "role": "tool", "tool_call_id": tool_call.id, "content": result }) print(f" Resultado: {result}") else: # El modelo ha generado la respuesta final final_answer = response.choices[0].message.content print(f"\n🤖 Respuesta Final:\n{final_answer}") return final_answer print("⚠️ Máximo de iteraciones alcanzado") return None # Test if __name__ == "__main__": run_agent("¿Cuál es el tiempo en Barcelona y Madrid?")

Step 4: Múltiples Herramientas (Weather + Calculator)

Crea agent_multi_tools.py:

import json from openai import OpenAI from dotenv import load_dotenv load_dotenv() client = OpenAI() tools = [ { "type": "function", "function": { "name": "get_weather", "description": "Obtiene el tiempo de una ciudad", "parameters": { "type": "object", "properties": { "city": {"type": "string"} }, "required": ["city"] } } }, { "type": "function", "function": { "name": "calculator", "description": "Realiza operaciones matemáticas", "parameters": { "type": "object", "properties": { "expression": { "type": "string", "description": "Expresión matemática (ej: 2 + 2)" } }, "required": ["expression"] } } } ] def get_weather(city): return {"Barcelona": "24°C, soleado", "Madrid": "22°C, nublado"}.get(city, "No disponible") def calculator(expression): try: result = eval(expression) return str(result) except Exception as e: return f"Error: {str(e)}" def run_multi_agent(user_message): print(f"\n👤 Usuario: {user_message}\n") messages = [{"role": "user", "content": user_message}] response = client.chat.completions.create( model="gpt-4o-mini", messages=messages, tools=tools, tool_choice="auto" ) if response.choices[0].finish_reason == "tool_calls": messages.append(response.choices[0].message) for tool_call in response.choices[0].message.tool_calls: tool_name = tool_call.function.name tool_args = json.loads(tool_call.function.arguments) print(f"🔧 {tool_name}: {tool_args}") if tool_name == "get_weather": result = get_weather(**tool_args) elif tool_name == "calculator": result = calculator(**tool_args) else: result = "Unknown" messages.append({ "role": "tool", "tool_call_id": tool_call.id, "content": result }) final = client.chat.completions.create(model="gpt-4o-mini", messages=messages) answer = final.choices[0].message.content else: answer = response.choices[0].message.content print(f"\n🤖 {answer}") return answer # Test if __name__ == "__main__": run_multi_agent("¿Cuál es el tiempo en Barcelona? Y calcula 15 * 8")
