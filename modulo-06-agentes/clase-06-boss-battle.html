⚔️ El Reto: Tu Agente Investigador

Tu misión es construir un agente IA autónomo que sea capaz de investigar un tema, resumir información de múltiples fuentes y generar un informe profesional. Este es el tipo de sistema que usan startups, consultoría y medios digitales para escalar análisis y investigación.

Lo que tu agente debe hacer:

  1. Recibir un tema del usuario (ej: "Tendencias en IA 2026")
  2. Buscar en internet artículos relevantes usando DuckDuckGo, Google Custom Search, o Tavily
  3. Leer y resumir el contenido de las 3-5 mejores fuentes
  4. Generar un informe estructurado en Markdown con:
    • Resumen ejecutivo (2-3 párrafos)
    • Hallazgos clave (bullet points organizados por tema)
    • Conclusiones y tendencias
    • Fuentes citadas (links + títulos)
  5. Guardar el informe a un archivo (`.md` o `.txt`)

Tú eliges el framework:

Puedes usar CrewAI (multi-agente, más complejo pero potente), LangGraph (single agente con estado y herramientas), o puro OpenAI function calling (más control, menos abstracciones). Todas son válidas. Elige la que mejor se ajuste a tu nivel de comodidad.

✅ Requisitos Técnicos (Checklist)

🌐 Búsqueda Real

El agente debe hacer búsquedas reales en internet (no datos inventados). Usa APIs como Tavily, Perplexity, o DuckDuckGo Search API.

📚 Resumen de Múltiples Fuentes

Debe procesar y resumir contenido de al menos 3 artículos. Cada resumen debe ser coherente y distinto.

📊 Informe Estructurado

Output debe ser Markdown bien formateado con secciones, bullet points, headers y citas. Profesional y legible.

💾 Guardar a Archivo

El informe se debe guardar en un archivo (`.md`, `.txt` o `.pdf`). Ruta y nombre deben ser lógicos.

🛡️ Manejo de Errores

Si falla la búsqueda, si no hay resultados, o si hay timeout: el agente debe manejar gracefully (no crash).

🌟 Bonus: Parámetro Depth

Opcional pero imprésionante: permite al usuario elegir "quick" (resumen rápido) vs "deep" (análisis profundo).

Opción B: LangGraph (Grafo de Estado — Máximo control)

Un único agente que transita por estados: search → fetch → summarize → write.

    """Extrae y resume un artículo"""
    # requests + BeautifulSoup + OpenAI para resumen
    return "Resumen del artículo..."

# Nodos del grafo
def node_search(state: ResearchState):
    state.search_results = search(state.topic)
    return state

def node_fetch(state: ResearchState):
    state.summaries = [fetch_and_summarize(url)
                       for url in state.search_results[:3]]
    return state

def node_write(state: ResearchState):
    # Usa OpenAI para escribir informe final
    state.report = "# Informe sobre " + state.topic + "\n\n"
    for i, summary in enumerate(state.summaries, 1):
        state.report += f"## Fuente {i}\n{summary}\n\n"
    return state

# Construir grafo
graph = StateGraph(ResearchState)
graph.add_node("search", node_search)
graph.add_node("fetch", node_fetch)
graph.add_node("write", node_write)

graph.add_edge("search", "fetch")
graph.add_edge("fetch", "write")
graph.add_edge("write", END)

graph.set_entry_point("search")
runnable = graph.compile()

# Ejecutar
state = ResearchState()
state.topic = "Tendencias IA 2026"
result = runnable.invoke(state)

# Guardar
with open("informe.md", "w") as f:
    f.write(result.report)

Ventajas: Total control de flujo, debugging fácil, escalable.

Instalación: pip install langgraph langchain

Opción C: OpenAI Function Calling (Sin frameworks — Más bajo nivel)

Construyes todo manualmente con function calling de OpenAI.

import openai
import json
import requests
from bs4 import BeautifulSoup

client = openai.OpenAI()

# Funciones que OpenAI puede llamar
tools = [
    {
        "type": "function",
        "function": {
            "name": "search_web",
            "description": "Search the web for articles",
            "parameters": {
                "type": "object",
                "properties": {
                    "query": {"type": "string"}
                },
                "required": ["query"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "fetch_url",
            "description": "Fetch and summarize a URL",
            "parameters": {
                "type": "object",
                "properties": {
                    "url": {"type": "string"}
                },
                "required": ["url"]
            }
        }
    }
]

def search_web(query: str):
    # Simulado o llamada real a API
    return ["url1", "url2", "url3"]

def fetch_url(url: str):
    try:
        resp = requests.get(url, timeout=10)
        soup = BeautifulSoup(resp.content, 'html.parser')
        return soup.get_text()[:2000]  # Primeros 2000 chars
    except:
        return "Error fetching URL"

# Loop principal
topic = "Tendencias IA 2026"
messages = [
    {"role": "system",
     "content": "Eres un analista. Tu tarea: buscar, resumir y generar informe sobre: " + topic},
    {"role": "user",
     "content": f"Investiga {topic} y genera un informe Markdown. Usa las herramientas."}
]

report = ""

for _ in range(5):  # Max 5 iteraciones
    response = client.chat.completions.create(
        model="gpt-4",
        messages=messages,
        tools=tools,
        tool_choice="auto"
    )

    if response.stop_reason == "end_turn":
        report = response.choices[0].message.content
        break

    # Procesar tool calls
    for tool_call in response.choices[0].message.tool_calls:
        if tool_call.function.name == "search_web":
            result = search_web(tool_call.function.arguments["query"])
        elif tool_call.function.name == "fetch_url":
            result = fetch_url(tool_call.function.arguments["url"])

        messages.append({"role": "assistant", "content": response.choices[0].message.content})
        messages.append({"role": "user", "content": json.dumps(result)})

# Guardar
with open("informe.md", "w") as f:
    f.write(report)

Ventajas: Máximo control, entiende bien cómo funciona.

Requiere: pip install openai requests beautifulsoup4
