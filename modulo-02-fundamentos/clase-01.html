📖 Resumen de conceptos

a) ¿Qué es la Inteligencia Artificial?

Definición: La IA es la capacidad de una máquina para realizar tareas que normalmente requieren inteligencia humana. Estas incluyen aprender de la experiencia, reconocer patrones, entender lenguaje y tomar decisiones.

Dos tipos principales:

  • IA Estrecha (Narrow AI): Diseñada para una tarea específica. Ejemplos: Siri (procesamiento de voz), GPS (navegación), recomendaciones de Netflix (filtrado de películas). Es la que existe hoy.
  • IA General (AGI): Hipotética, podría hacer cualquier tarea intelectual que hace un humano. No existe todavía.

Para un desarrollador web:

"Ya usas IA todos los días sin saberlo: el autocompletado en VS Code, los filtros de spam en Gmail, la búsqueda de Google, las recomendaciones en Amazon. Todo eso es IA."

b) Machine Learning: El motor de la IA moderna

¿Cuál es la diferencia con la programación tradicional?

En programación clásica, tú defines las reglas:

if (temperatura > 25) {
    encenderAire();
} else {
    apagarAire();
}

En Machine Learning, le das datos y la máquina aprende los patrones automáticamente. El modelo descubre las reglas por sí solo.

Tres tipos de Machine Learning:

  • Supervised Learning (Aprendizaje Supervisado): Tienes datos etiquetados. Ejemplo: fotos de gatos y perros etiquetadas. El modelo aprende a clasificar.
    • Clasificación: ¿Es gato o perro? (clases discretas)
    • Regresión: ¿Cuál será el precio de esta casa? (número continuo)
  • Unsupervised Learning: Datos sin etiquetar. El modelo busca patrones por sí solo. Ejemplo: agrupar clientes por comportamiento de compra.
  • Reinforcement Learning: El modelo aprende por prueba y error, recibiendo recompensas. Ejemplo: un robot que aprende a caminar.

Analogía para desarrolladores: Si JavaScript es escribir funciones manualmente, ML es un framework que genera las funciones optimales basándose en ejemplos. El modelo es como una función gigante con millones de parámetros.

c) Deep Learning: Redes neuronales profundas

¿Qué es una red neuronal? Un modelo inspirado en el cerebro humano, compuesto por capas de "neuronas" artificiales conectadas.

Estructura básica:

ENTRADA       CAPAS OCULTAS              SALIDA
(datos)    (procesan info)          (predicción)

  [o]  --\
  [o]    |--[o]--[o]--[o]--\
  [o]  --/                  |--[resultado]
                            /
            capa 1    capa 2  capa 3

¿Por qué "Deep" Learning? Porque usa muchas capas ocultas (profundidad). Una red con 10+ capas es considerada "profunda".

Aplicaciones de Deep Learning:

  • Computer Vision: Reconocimiento facial, detección de objetos, conducción autónoma
  • NLP (Natural Language Processing): ChatGPT, traducción automática, análisis de sentimiento
  • Speech: Reconocimiento de voz (Alexa, Google Assistant)

Definiciones simplificadas:

  • IA: Campo amplio. Cualquier técnica que haga que máquinas parezcan inteligentes.
  • ML: Subset de IA. Máquinas que aprenden de datos, en lugar de ser programadas.
  • DL: Subset de ML. Usa redes neuronales con muchas capas.
  • LLMs: Subset de DL. Modelos gigantes de lenguaje entrenados en trillones de palabras (GPT, Claude, Gemini).

e) Conceptos clave para un desarrollador

Cuando leas sobre IA/ML, verás estos términos constantemente:

  • Dataset: El conjunto de datos que usas para entrenar. Ejemplo: 10,000 fotos de gatos y perros etiquetadas.
  • Modelo: La red neuronal misma. El objeto que contiene los parámetros aprendidos.
  • Entrenamiento (Training): El proceso de ajustar los parámetros del modelo para que prediga mejor.
  • Inferencia (Inference): Usar el modelo ya entrenado para hacer predicciones en datos nuevos. Es cuando el modelo "produce".
  • Parámetros: Los "pesos" de cada neurona. Son lo que realmente "aprendió" el modelo. Un LLM como GPT-3 tiene 175 mil millones de parámetros.
  • Época (Epoch): Una pasada completa por todo el dataset durante el entrenamiento. Si tienes 1000 imágenes y las procesas todas una vez = 1 época.
  • Overfitting: Cuando el modelo memoriza el dataset de entrenamiento pero falla con datos nuevos. Aprende de más.
💡 Analogía JS para entrenamiento:
for (let epoch = 0; epoch < 100; epoch++) { for (let sample of trainingData) { // Predice, calcula error error = predict(sample) - realValue; // Ajusta pesos (descenso de gradiente) adjustWeights(error); } }

El entrenamiento es un bucle gigante que se repite millones de veces, ajustando millones de pesos. Por eso tarda días o semanas.
