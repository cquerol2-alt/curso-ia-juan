🎯 Boss Battle: 5 Problemas Reales con Prompts

📅 Semana 3 — Viernes | 🏆 50 XP | ⚔️ Demuestra todo lo aprendido esta semana

⚔️ Misión

Esta semana has aprendido qué es la IA, cómo funcionan los LLMs, prompt engineering avanzado y modelos open source. Ahora toca demostrar que puedes resolver problemas REALES usando solo prompts — sin escribir ni una línea de código.

Tienes 5 retos. El último debe hacerse con Ollama (modelo local).

⚙️ Reglas del juego

  • Puedes usar ChatGPT, Claude o cualquier LLM (excepto el reto 5, que debe ser con Ollama)
  • Guarda las conversaciones completas (prompt + respuesta)
  • Para cada reto, usa al menos una técnica avanzada de prompting (CoT, few-shot, role prompting, etc.)
  • Evalúa la calidad de la respuesta del 1 al 10

🎯 Reto 1: Analista de Negocio

Un restaurante nuevo te pide que analices si abrir en el barrio de Ruzafa (Valencia). Usando SOLO prompts (sin buscar en Google), consigue que el LLM te dé:

  • Análisis demográfico del barrio
  • Competencia de restaurantes
  • Ticket medio estimado
  • 3 riesgos principales
  • Una recomendación final
💡 Técnica sugerida: Role prompting + Chain of Thought
✓ Criterio de éxito: La respuesta debe incluir datos concretos (aunque estimados) y una recomendación argumentada.

🎯 Reto 2: Generador de Código

Necesitas una función Python que reciba una lista de pedidos (cada pedido es un diccionario con 'cliente', 'productos', 'total') y devuelva:

  • El cliente que más ha gastado
  • El producto más vendido
  • El ticket medio

Usa prompt engineering para que el LLM te dé código limpio, documentado y con tests.

💡 Técnica sugerida: Few-shot + output formatting (pide JSON de ejemplo primero)
✓ Criterio de éxito: El código debe funcionar si lo copias y pegas en Python.

🎯 Reto 3: Traductor Técnico

Traduce la página 'About' de un proyecto open source del inglés al español. No una traducción literal: adapta el tono para el mercado español, mantén los términos técnicos correctos y sugiere mejoras de copywriting.

Texto a traducir:

"We're building the future of AI-powered development tools. Our platform helps developers ship faster, write better code, and focus on what matters. With over 100K developers already on board, we're just getting started. Join us and transform how you build software."
💡 Técnica sugerida: System prompt detallado + constraints
✓ Criterio de éxito: Traducción natural, no literal. Tono profesional pero cercano.

🎯 Reto 4: Debugger de Prompts

El siguiente prompt da malos resultados. Reescríbelo usando las técnicas aprendidas para que el output sea útil.

❌ Prompt malo:

"dime cosas sobre hacer una app"

Tu tarea: Reescrbirlo aplicando role prompting, contexto, constraints, formato de salida y Chain of Thought. Ejecuta ambas versiones y compara los resultados.

💡 Técnica sugerida: Aplicar TODAS las técnicas aprendidas
✓ Criterio de éxito: El prompt mejorado debe producir una respuesta 3x más útil y específica.

🎯 Reto 5: Ollama Challenge (Modelo Local)

⚠️ Este reto DEBE hacerse con Ollama

Usando SOLO Ollama con un modelo local (llama3.2 o phi3), crea un prompt que genere 5 preguntas de entrevista técnica para un puesto de 'Junior AI Developer'. Las preguntas deben cubrir:

  • Python
  • APIs
  • LLMs
  • Prompt engineering
  • Un caso práctico
💡 Técnica sugerida: System prompt + few-shot + output formatting
✓ Criterio de éxito: Las preguntas deben ser realistas, progresivas en dificultad y con respuestas modelo.

🌟 Bonus: Compara la calidad del resultado con la misma pregunta en ChatGPT/Claude. ¿Qué diferencias notas?

📋 Entrega

Para completar el Boss Battle, prepara un documento (puede ser un .md, .txt o .doc) con:

  1. Los 5 prompts que usaste (versión final)
  2. Las respuestas del LLM (resumidas si son muy largas)
  3. Tu evaluación de cada respuesta (1-10) y por qué
  4. Qué técnica de prompting usaste en cada reto
  5. Reflexión final: ¿Qué has aprendido sobre cómo comunicarte con una IA?

Envía el documento a través del formulario de entregas o comparte el link del drive.
