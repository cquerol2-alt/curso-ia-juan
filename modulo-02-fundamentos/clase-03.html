📖 Resumen

a) System Prompts, User Prompts y Assistant Messages

Los mensajes en una conversación con IA tienen tres actores principales:

  • System: Instrucciones que definen el comportamiento y la personalidad del asistente.
  • User: El mensaje que envía el usuario (la pregunta o solicitud).
  • Assistant: La respuesta que genera el modelo de IA.

Estos tres roles se repiten en una conversación. El system prompt generalmente se establece una sola vez al inicio.

System: "Eres un experto en cocina española. Responde siempre con tono amable y proporciona recetas detalladas." User: "¿Cómo hago una paella?" Assistant: "Para hacer una buena paella valenciana, necesitas arroz bomba de calidad, azafrán, caldo de pollo caliente... [respuesta completa]"
💡 Tip: El system prompt es como el constructor de una clase: define el comportamiento base que todo el asistente seguirá.

b) Zero-shot, One-shot y Few-shot Prompting

Estos términos describen cuántos ejemplos proporciona antes de pedir al modelo que resuelva una tarea:

Zero-shot (sin ejemplos):

Prompt: "Clasifica este texto como positivo o negativo: 'El producto es terrible'"
Respuesta: "Negativo"

Cuándo usarlo: Tareas simples que el modelo entiende sin necesidad de ejemplos.

One-shot (un ejemplo):

Prompt:
"Clasifica sentimientos. Ejemplo: 'El servicio fue excelente' = Positivo
Ahora clasifica: 'El producto es terrible'"
Respuesta: "Negativo"

Cuándo usarlo: Cuando quieres aclarar el formato o estilo esperado.

Few-shot (3-5 ejemplos):

Prompt:
"Clasifica sentimientos de reseñas:
1. 'Producto excelente, lo recomiendo' = Positivo
2. 'Entrega lenta y producto roto' = Negativo
3. 'Es un producto normal, nada especial' = Neutro
4. 'Servicio rápido pero comida fría' = Negativo

Ahora clasifica: 'La pizza estaba deliciosa y el mesero muy atento' = ?"
Respuesta: "Positivo"

Cuándo usarlo: Tareas complejas o cuando necesitas máxima precisión.

c) Chain of Thought (CoT): Piensa Paso a Paso

Pedir al modelo que "piense paso a paso" o "razone" antes de responder mejora dramáticamente la calidad de sus respuestas, especialmente en problemas lógicos y matemáticos.

Sin Chain of Thought:

Prompt: "Si un restaurante pide 20 platos a 15€ cada uno con 15% de descuento, ¿cuál es el total?"
Respuesta (a menudo incorrecta): "300€"

Con Chain of Thought:

Prompt: "Si un restaurante pide 20 platos a 15€ cada uno con 15% de descuento, ¿cuál es el total? Piensa paso a paso."
Respuesta: "Paso 1: 20 platos × 15€ = 300€
Paso 2: 15% de 300€ = 0.15 × 300 = 45€
Paso 3: 300€ - 45€ = 255€
Total: 255€"

d) Técnicas Avanzadas

Role Prompting (Juego de Roles):

"Actúa como un ingeniero senior de Google con 10 años de experiencia. ¿Qué preguntas harías en una entrevista de técnica a un candidato junior?"

El modelo responde desde esa perspectiva, elevando la calidad.

Output Formatting (Formato de Salida):

"Analiza este producto y responde en formato JSON: { 'nombre': '', 'precio': '', 'disponible': true/false, 'calificación': 0-5 } Producto: Arroz Bomba, 12€, en stock, 4.8 estrellas"

Constraints (Restricciones):

"Explica qué es Machine Learning: - Máximo 100 palabras - No uses jerga técnica - Incluye un ejemplo real"

Delimiters (Delimitadores):

"Analiza el texto entre los --- : --- Texto a analizar --- "El servicio fue rápido pero la comida llegó fría." --- Extrae: 1. Aspectos positivos 2. Aspectos negativos 3. Sentimiento general"

Self-Consistency (Auto-verificación):

"Resuelve: Si Juan tiene 3 veces más dinero que María, y juntos tienen 400€, ¿cuánto tiene cada uno? Resuelve el problema, luego verifica tu respuesta con una ecuación."

e) Prompt Templates: Reutiliza lo que Funciona

Un template es un prompt con variables que se adapta a diferentes inputs. Es perfecto para tareas repetitivas.

TEMPLATE: Analiza el siguiente {tipo_contenido} y extrae: 1. Tema principal 2. Sentimiento (positivo/negativo/neutro) 3. 3 puntos clave Contenido: {contenido} Formato de salida: JSON

Uso: Reutiliza reemplazando {tipo_contenido} y {contenido}.

Ejemplo 1: tipo_contenido="reseña de producto", contenido="Este laptop es rápido pero muy caro"

Ejemplo 2: tipo_contenido="artículo de blog", contenido="Los 5 mejores restaurantes de Valencia..."

f) Errores Comunes en Prompting

Error Mal (❌) Bien (✅)
Ser demasiado vago "Háblame de IA" "Explica en 3 párrafos qué es una red neuronal, dirigido a un programador junior con ejemplos de código"
No dar contexto "¿Debo usar A o B?" "Tengo un proyecto web con 100k usuarios. ¿Debo usar React o Vue? Necesito máxima velocidad de desarrollo"
No especificar formato "Dame ideas de marketing" "Dame 5 ideas de marketing para un restaurante español. Formato: lista con [Idea | Presupuesto | ROI esperado]"

💻 Práctica Guiada

Realiza estos 4 ejercicios para practicar las técnicas de prompt engineering avanzado. Puedes usar ChatGPT, Claude o Gemini.

✏️ Ejercicio 1: System Prompt para Chatbot

Tarea: Escribe un system prompt para un chatbot de Rice in Action (empresa de arroces preelaborados para canal HORECA).

El chatbot debe:

  • Responder preguntas sobre productos (variedades, precios, cantidades mínimas)
  • Ser amable, profesional y rápido
  • Sugerir productos según las necesidades del cliente
  • Ofrecer contacto con ventas para pedidos grandes

Entrega: Copia el system prompt que escribas en la sección de comentarios.

✏️ Ejercicio 2: Few-shot Prompting

Tarea: Usa few-shot prompting para clasificar 5 reseñas de restaurantes.

Formato esperado: Proporciona 3 ejemplos de reseñas clasificadas (positivo/negativo/neutro), y luego pide al modelo que clasifique 5 nuevas reseñas.

Reseñas para clasificar:

  1. "Comida excelente, servicio rápido y meseros atentos. Volveré pronto."
  2. "El arroz llegó frío y el camarero fue grosero."
  3. "La paella estaba bien, nada del otro mundo."
  4. "¡Mejor restaurante de Valencia! Impresionante en todos los sentidos."
  5. "Precios altos y porciones pequeñas. Decepcionante."

Entrega: Pega el prompt completo (ejemplos + reseñas) y las clasificaciones en los comentarios.

✏️ Ejercicio 3: Chain of Thought

Tarea: Resuelve este problema de dos formas: primero sin Chain of Thought, luego con CoT.

Problema:
"Si un restaurante pide 15 cajas de arroz a 12€ cada una con un 10% de descuento por volumen, y el envío cuesta 8€, ¿cuál es el total?"

Entrega: Muestra ambas respuestas (sin CoT y con CoT) y explica cuál es más precisa.

✏️ Ejercicio 4: Prompt Template Reutilizable

Tarea: Crea un prompt template para generar descripciones de producto para una web de e-commerce.

El template debe incluir variables para:

  • Nombre del producto
  • Características principales
  • Precio
  • Público objetivo

Bonus: Prueba el template con 2 productos diferentes (ej: "Arroz Bomba Premium" y "Arroz Integral Ecológico").

Entrega: Copia el template y los 2 ejemplos generados.

🧪 Mini-Reto del Día: Prompt Battle

Objetivo: Comparar la calidad de respuestas usando prompts malos vs. prompts buenos.

Tarea: Usa ChatGPT, Claude o Gemini para generar el código HTML + CSS de una landing page para un restaurante.

Intento 1: Prompt Vago (❌ Malo)

"Crea una landing page para un restaurante"

Resultado: Copia el HTML generado. Observa: ¿es profesional? ¿tiene buen diseño? ¿incluye llamadas a la acción?

Intento 2: Prompt Optimizado (✅ Bueno)

"Eres un desarrollador senior de landing pages. Crea una landing page HTML + CSS para 'El Arrocero Valenciano', restaurante especializado en paellas auténticamente valencianas. Requisitos: - Debe incluir: hero section, galería de platos, testimonios, formulario de reserva - Diseño: moderno, responsive (mobile-first), colores cálidos (naranjas, marrones) - Llamadas a la acción: 'Reservar ahora', 'Ver menú' - Estructura: header con nav, main content, footer con contacto - Especifica que es Michelin (si es necesario, pon ⭐⭐) Formato: HTML completo con CSS embebido, listo para copiar-pegar."

Resultado: Copia el HTML generado. Observa: ¿es mucho mejor? ¿incluye más detalles? ¿se puede usar directamente?

Análisis Final:

Pregunta: ¿Cuál es mejor y por qué? Describe las diferencias en calidad, detalle y usabilidad. Copia tu análisis en los comentarios.
