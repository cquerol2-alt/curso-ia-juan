📖 Resumen

El ecosistema de modelos de IA en 2026

En 2026, tienes dos caminos principales para usar modelos de lenguaje: APIs propietarias (cerradas) u open-source (o "open weights"). Cada una tiene ventajas y limitaciones.

Modelos Cerrados (Closed-Source)

Son modelos desarrollados por grandes empresas y accesibles solo a través de APIs. Tú no tienes acceso al código ni a los pesos (parámetros) del modelo.

Comparativa de Modelos Cerrados 
┌─────────────────┬──────────┬────────────────────────────────┬──────────────────────────┐
│ Modelo          │ Empresa  │ Ventajas                       │ Limitaciones             │
├─────────────────┼──────────┼────────────────────────────────┼──────────────────────────┤
│ GPT-4o          │ OpenAI   │ Muy capaz, multimodal          │ Caro para alto volumen   │
│                 │          │ Ecosistema enorme              │ Datos en servidores      │
│                 │          │                                │ ~$2.50/1M tokens input   │
├─────────────────┼──────────┼────────────────────────────────┼──────────────────────────┤
│ Claude 3.5      │ Anthropic│ Excelente en código            │ Menos ecosistema         │
│ Sonnet          │          │ Muy buen razonamiento          │ ~$3/1M tokens input      │
│                 │          │ Ventana 200K tokens            │                          │
├─────────────────┼──────────┼────────────────────────────────┼──────────────────────────┤
│ Gemini 1.5 Pro  │ Google   │ Ventana enorme (1M tokens)     │ Menos consistente        │
│                 │          │ Integración Google Workspace   │ ~$1.25/1M tokens input   │
│                 │          │                                │                          │
└─────────────────┴──────────┴────────────────────────────────┴──────────────────────────┘
Pro tip: Para un proyecto personal o startup, estas APIs son geniales. Para una empresa con datos sensibles o alto volumen, puede que necesites open source.

Modelos Open Source (Open Weights)

Open weights significa que los pesos (parámetros) del modelo están disponibles para descargar y usar libremente. No es lo mismo que código "open source" tradicional. Los pesos son el "cerebro" del modelo.

⚠️ Advertencia importante: Open source no significa gratis en producción. Necesitas servidores (GPU) para correrlos. Pero tienes control total de tus datos y evitas dependencia de un proveedor.

¿Cuándo usar cada tipo?

Marco de decisión 
USA API CERRADA si:
  ✓ Necesitas un prototipo rápido
  ✓ No tienes GPU disponible
  ✓ Quieres el modelo más potente en este momento
  ✓ Tus datos no son sensibles/confidenciales
  ✓ Puedes pagar por uso (pago por token)

USA OPEN SOURCE si:
  ✓ Trabajas con datos sensibles/privados
  ✓ Quieres control total del modelo
  ✓ Quieres evitar dependencia de un proveedor
  ✓ Tienes costes predecibles (hardware propio)
  ✓ Tu caso de uso es específico (puedes fine-tunear)

Ollama: Corre Modelos en Tu Ordenador

Ollama es una herramienta simple que te permite descargar y ejecutar modelos de IA localmente (en tu ordenador). Sin APIs externas, sin pagos por token, sin enviar datos a internet.

Paso 1: Descarga Ollama desde ollama.com/download. Disponible para macOS, Linux y Windows. La instalación tarda menos de 2 minutos.
Instalación de Ollama 
# Instalar Ollama en macOS/Linux (alternativa por terminal)
curl -fsSL https://ollama.com/install.sh | sh

# En Windows: descarga el instalador desde https://ollama.com/download

# Descargar y ejecutar un modelo
ollama run llama3.2

# Listar modelos descargados
ollama list

# Correr un modelo más pequeño (para PCs con poca RAM)
ollama run phi3
📌 Requisitos mínimos: 8GB RAM para modelos 7B, 16GB para modelos 13B. Si tu PC tiene menos, usa los modelos pequeños (phi3, gemma2:2b). Mejor si tienes GPU NVIDIA (CUDA) o AMD (ROCm).

Ollama como API Local

Ollama no solo es una aplicación de terminal. Expone una API REST en localhost:11434 para que puedas usarla desde tus aplicaciones.

Llamar a Ollama con curl 
# Ollama sirve una API REST en localhost:11434
curl http://localhost:11434/api/generate -d '{
  "model": "llama3.2",
  "prompt": "Explica qué es Python en 2 frases"
}'

# La respuesta es JSON con el texto generado:
# {
#   "model": "llama3.2",
#   "response": "Python es un lenguaje de programación...",
#   "done": true
# }
Llamar a Ollama con Python 
import requests

# Llamar a Ollama local
response = requests.post('http://localhost:11434/api/generate', json={
    'model': 'llama3.2',
    'prompt': '¿Qué es una API REST?',
    'stream': False
})

# Extraer la respuesta
resultado = response.json()['response']
print(resultado)
Pro tip: Esto es exactamente igual que llamar a la API de OpenAI, pero gratis y en tu máquina. En el Módulo 3 verás cómo integrar tanto APIs cerradas como Ollama en la misma aplicación.

Hugging Face: El GitHub de la IA

Hugging Face Hub es el repositorio central de modelos de IA. Es como GitHub, pero para ML. Aquí encuentras:

  • Modelos pre-entrenados (miles de ellos)
  • Datasets públicos para entrenamiento
  • Spaces: aplicaciones web de demostración
  • Librerías: Transformers (la estándar de facto)
Usar Hugging Face desde Python 
from transformers import pipeline

# Cargar un modelo desde Hugging Face
classifier = pipeline('sentiment-analysis',
                     model='nlptown/bert-base-multilingual-uncased-sentiment')

# Usar el modelo
resultado = classifier("¡Me encanta este curso de IA!")
print(resultado)
# Salida: [{'label': 'positive', 'score': 0.999}]
🔗 Enlace: https://huggingface.co/ — Explora miles de modelos, datasets y herramientas.
