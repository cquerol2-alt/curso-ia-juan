Clase 06: FastAPI — APIs Modernas para IA

Aprenderás a construir APIs profesionales, escalables y con soporte nativo para streaming LLM. FastAPI es el estándar en startups de IA y en ofertas de trabajo como AI Engineer.

📺 Vídeo Principal

Duración: ~30 min | Idioma: Español | Nivel: Principiante → Intermedio

⚡ Parte A: De Flask a FastAPI — Por Qué Dar el Salto

Ya conoces Flask desde el Módulo 1. FastAPI toma lo mejor de Flask, añade features modernas y optimiza específicamente para APIs (no para templates HTML). Es el framework elegido en la mayoría de startups de IA.

Ventajas de FastAPI sobre Flask:

  • Async nativo: Flask es sincrónico por defecto. FastAPI es async desde el día uno → mejor concurrencia.
  • /docs automática: Swagger UI y ReDoc gratis con cero configuración.
  • Pydantic integrado: Validación automática de request/response, conversión de tipos, documentación.
  • Type hints: Code hints completos gracias a tipos Python.
  • Performance: ~3x más rápido que Flask puro (cercano a Go/Node).
  • Streaming: Soporte nativo para SSE y streaming responses (crítico para LLMs).

Cuándo usar cada uno:

Aspecto Flask FastAPI
Prototipo rápido ✅ Excelente (curva rápida) ✅ Muy bueno
API REST pura ⚠️ Funciona, requiere extra ✅ Hecho para esto
Streaming/SSE ⚠️ Posible con trabajo ✅ Nativo, simple
Concurrencia alta ⚠️ Threads, lento ✅ Async event loop
Documentación automática ❌ Requiere Flasgger/Connexion ✅ Incluida (/docs)
Validación de datos ⚠️ Marshmallow, marshmallow-jsonschema ✅ Pydantic built-in
Performance crudo ~1500 req/s (single core) ~4500 req/s (single core)
💡 TL;DR: Usa Flask para webs tradicionales con Jinja2. Usa FastAPI para APIs modernas y LLM apps. Si buscas trabajo como AI Engineer, FastAPI es el estándar.

🚀 Parte B: Tu Primera API con FastAPI

Paso 1: Instalación

pip install fastapi uvicorn python-dotenv requests

Paso 2: Hello World

from fastapi import FastAPI app = FastAPI() @app.get("/") async def root(): return {"message": "¡Hola, FastAPI!"} @app.get("/users/{user_id}") async def get_user(user_id: int): return {"user_id": user_id, "name": f"Usuario {user_id}"} @app.get("/search") async def search(q: str, limit: int = 10): return {"query": q, "limit": limit}

Notar: async def es nativo en FastAPI. FastAPI infiere tipos automáticamente (int, str).

Paso 3: Ejecutar

uvicorn main:app --reload

Ve a http://localhost:8000/docs — ¡Swagger UI automático!

Paso 4: Pydantic Models (Validación)

from pydantic import BaseModel class Message(BaseModel): text: str user_id: int @app.post("/chat") async def chat(msg: Message): return {"user": msg.user_id, "response": f"Echo: {msg.text}"}
🎯 Pydantic vs Marshmallow: Pydantic valida automáticamente en el POST. Si envías {"text": "hola", "user_id": "abc"}, FastAPI rechaza con 422 (validation error). El cliente ve un error claro. ¡Cero boilerplate!

🌊 Parte C: Streaming de Respuestas LLM

Este es el killer feature para integrar OpenAI/Claude. El cliente ve tokens en vivo mientras se generan, no espera a que termine toda la respuesta.

Endpoint de Streaming

from fastapi.responses import StreamingResponse import openai @app.post("/chat/stream") async def chat_stream(msg: Message): async def generate(): response = await openai.ChatCompletion.acreate( model="gpt-3.5-turbo", messages=[{"role": "user", "content": msg.text}], stream=True ) async for chunk in response: token = chunk["choices"][0]["delta"].get("content", "") if token: yield f"data: {token}\n\n" return StreamingResponse(generate(), media_type="text/event-stream")

Cómo probarlo con curl:

curl -X POST http://localhost:8000/chat/stream \ -H "Content-Type: application/json" \ -d '{"text": "Explica qué es la IA", "user_id": 1}'

Verás tokens llegando en tiempo real: data: Explicar\n\ndata: ía\n\n...

💡 En el navegador: Usa fetch() + EventSource o librería como eventsource-parser para consumir el stream en JavaScript.

🔧 Parte D: Middleware y Entorno de Producción

CORS Middleware (necesario para frontend)

from fastapi.middleware.cors import CORSMiddleware app.add_middleware( CORSMiddleware, allow_origins=["*"], # En prod: ["https://mitio.com"] allow_credentials=True, allow_methods=["*"], allow_headers=["*"], )

Health Check (obligatorio en producción)

@app.get("/health") async def health(): return {"status": "ok", "version": "1.0"}

Error Handling Global

from fastapi import HTTPException @app.post("/chat") async def chat(msg: Message): if not msg.text: raise HTTPException(status_code=400, detail="Text cannot be empty") return {"response": f"Echo: {msg.text}"}

Background Tasks (para LLMs lentos)

from fastapi import BackgroundTasks @app.post("/process") async def process(msg: Message, background_tasks: BackgroundTasks): background_tasks.add_task(slow_operation, msg.user_id) return {"status": "processing"} def slow_operation(user_id: int): # Esto se ejecuta en background time.sleep(10) print(f"Procesado user {user_id}")

📋 Parte E: FastAPI vs Flask — Cheat Sheet de Migración

Tarea Flask FastAPI
GET simple @app.route('/') @app.get('/')
Parámetro path <int:id> {id: int}
Query string request.args.get('q') def func(q: str)
POST con body JSON request.json + validar class Model(BaseModel):
Async ⚠️ No nativo, requiere extensiones ✅ async def
Documentación Manual o Flasgger (requiere código extra) Automática en /docs
Validación Marshmallow (librería externa) Pydantic (built-in)
Streaming stream_with_context (complejo) StreamingResponse (simple)
Migración rápida: 1 ruta Flask = 1 ruta FastAPI en 80% de los casos. Lo que cambia es la validación (antes manual, ahora Pydantic) y el async (ahora nativo).

💻 Ejercicios Prácticos

🎯 Ejercicio 1: Code-Along — API de Chat con Streaming

Construye paso a paso una API completa. Tiempo: 30 min

  1. Crea main.py con FastAPI + CORS middleware
  2. Define Pydantic model ChatRequest (text: str, user_id: int)
  3. Endpoint GET /health que devuelve {"status": "ok"}
  4. Endpoint POST /chat que recibe ChatRequest y devuelve echo
  5. Endpoint POST /chat/stream que simula streaming con yield
  6. Ejecuta con uvicorn main:app --reload
  7. Prueba en http://localhost:8000/docs — ¡usa Swagger UI para enviar requests!
# Esqueleto: from fastapi import FastAPI from fastapi.middleware.cors import CORSMiddleware from pydantic import BaseModel from fastapi.responses import StreamingResponse app = FastAPI() app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"]) class ChatRequest(BaseModel): text: str user_id: int @app.get("/health") async def health(): return {"status": "ok"} @app.post("/chat") async def chat(req: ChatRequest): return {"user_id": req.user_id, "response": f"Echo: {req.text}"} @app.post("/chat/stream") async def chat_stream(req: ChatRequest): async def generate(): for word in req.text.split(): yield f"data: {word}\n\n" return StreamingResponse(generate(), media_type="text/event-stream")

🎯 Ejercicio 2: Mini-Challenge — Migra tu Flask Chatbot a FastAPI

Toma el chatbot Flask del Módulo 1 y reescríbelo en FastAPI. Tiempo: 40 min

  1. Copia la lógica de chat de tu código Flask
  2. Crea dos Pydantic models: ChatRequest (mensaje) y ChatResponse (respuesta)
  3. Reescribe el endpoint Flask POST → FastAPI POST
  4. Añade async: async def chat(...)
  5. EXTRA: Añade endpoint /chat/stream que devuelva streaming
  6. Documenta tu API en /docs
  7. Prueba con curl y en el navegador

⭐ Reto Extra: API con Autenticación Básica

Nivel avanzado. Tiempo: 45 min

  1. Crea un middleware que valide un header X-API-Key
  2. Rechaza requests sin key válida con 401
  3. Implementa rate limiting: máximo 10 requests por minuto por key
  4. Guarda logs de cada request (timestamp, key, endpoint, status)
  5. Crea endpoint /stats (solo con API key correcta) que muestra estadísticas
  6. Bonus: Usa slowapi (librería para rate limiting en FastAPI)
