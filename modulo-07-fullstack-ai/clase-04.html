📚 Resumen de Conceptos

1. Principios de UX para Apps de IA

Las mejores apps de IA (ChatGPT, Claude, Perplexity) siguen un patrón de diseño que no es casualidad:

  • Conversacional: Interface de chat bidireccional, como hablar con una persona.
  • Transparente sobre limitaciones: Disclaimers claros: "Puede cometer errores", "No tiene acceso a internet".
  • Feedback visual constante: Loading indicators, typing animations, indicadores de estado.
  • Personalización sutil: Dark mode por defecto, compresión de historial, continuidad entre sesiones.
  • Accesibilidad copy-paste: Cada respuesta es fácil de copiar, código resaltado, botones de acción.

2. El Efecto "Typing" — Por Qué Funciona

Cuando ves que la IA escribe carácter por carácter (streaming), la percepción es que está "pensando". Esto:

  • Reduce la sensación de espera (percibida como más rápida que esperar un bloque completo).
  • Aumenta la confianza: parece menos robótico, más vivo.
  • Permite al usuario empezar a leer mientras la IA sigue escribiendo.
  • Es técnicamente necesario para no sobrecargar al cliente (HTTP streaming).

3. Gestión de Expectativas

La UI debe ser honesta sobre lo que la IA puede y no puede hacer:

  • Disclaimers en lugar clave: Junto a la respuesta, no escondido en FAQ.
  • Indicadores de confianza: Algunos modelos muestran qué fuentes usaron.
  • Limitaciones visibles: "Esta versión tiene acceso a datos de 2024" o "No puede ejecutar código".
  • Ejemplos de errores: Mostrar casos donde la IA falló (aumenta credibilidad paradójicamente).

4. Input Design: El Textarea Inteligente

El campo de entrada es donde todo empieza:

  • Auto-expandible: Crece mientras escribes (no necesitas scroll dentro del input).
  • Placeholder inspirador: "Pregunta cualquier cosa, copia código, haz análisis..." guía al usuario.
  • Sugerencias de prompts: Cards con ejemplos ("Resume esto", "Explica como si fuera un niño").
  • Atajos de teclado: Shift+Enter para salto de línea, Ctrl/Cmd+Enter para enviar.
  • Archivo drag-and-drop: Muchas IAs ahora aceptan imágenes, PDFs, etc.

5. Output Design: Renderizado Markdown + Syntax Highlighting

La respuesta no es solo texto — es contenido estructurado:

  • Markdown rendering: Títulos, listas, negrita, enlace renderizados.
  • Code blocks con lenguaje: JavaScript, Python, SQL... cada uno con su color.
  • Copy button en cada bloque: Una línea para copiar rápido sin seleccionar.
  • Tables: Si la IA genera tabla, aparece bien formateada.
  • Math rendering: Ecuaciones LaTeX si es necesario ($x^2 + y^2$).

6. Historial de Conversaciones: Sidebar + Búsqueda

El historial es el "instinto" de la app — acceso rápido a lo pasado:

  • Sidebar colapsable: Lista todas las conversaciones (con títulos auto-generados).
  • Búsqueda en historial: Buscar por palabra clave dentro de todas las conversations.
  • Persistencia en localStorage/Backend: Los chats se guardan automáticamente.
  • Renombrado de chats: El usuario puede cambiar el título para organizarse.
  • Borrado selectivo: Eliminar un chat o toda la historia con un clic (con confirmación).

7. Responsive Design: Mobile-First

Las AI apps funcionan en móvil porque están optimizadas desde el inicio:

  • Sidebar → Hamburger menu: En pantalla pequeña, el historial se oculta.
  • Textarea y botones grandes: Fácil de tocar en un teléfono.
  • Scroll suave: Las respuestas se desplazan sin saltos.
  • Viewport correcta: No hay zoom accidental, texto legible sin zoom.

8. Dark Mode: Por Qué Es Estándar

Casi todas las AI apps usan dark mode por defecto. Por qué:

  • Menos fatiga visual: Especialmente en sesiones largas.
  • Mejor para el código: Los temas oscuros están optimizados para syntax highlighting.
  • Psicológico: Sensación de "espacio de trabajo profesional".
  • Batería: En OLED (muchos teléfonos modernos), reduce consumo.
  • Toggle fácil: Botón en la esquina para cambiar tema al vuelo.

⚙️ Práctica Guiada: Build a Chat UI Step by Step

Vamos a construir una UI de chat completa con React. Cada paso añade un componente o funcionalidad.

Step 1: Chat Bubble Component

Diferencia visual entre mensajes del usuario y la IA:


// ChatBubble.jsx
export function ChatBubble({ role, content }) {
  const isUser = role === 'user';

  return (
    <div
      style={{
        display: 'flex',
        justifyContent: isUser ? 'flex-end' : 'flex-start',
        marginBottom: '1rem',
        animation: 'fadeIn 0.3s ease-in',
      }}
    >
      <div
        style={{
          maxWidth: '70%',
          padding: '0.75rem 1rem',
          borderRadius: '12px',
          backgroundColor: isUser ? '#00e676' : '#2a2a2a',
          color: isUser ? 'white' : '#e0e0e0',
          wordWrap: 'break-word',
          boxShadow: '0 2px 8px rgba(0,0,0,0.3)',
        }}
      >
        {content}
      </div>
    </div>
  );
}

Step 2: Typing Indicator Animation

El efecto de "escritura" que ves en ChatGPT:


// TypingIndicator.jsx
export function TypingIndicator() {
  return (
    <div style={{ display: 'flex', gap: '4px', padding: '0.75rem' }}>
      {[0, 1, 2].map((i) => (
        <div
          key={i}
          style={{
            width: '8px',
            height: '8px',
            borderRadius: '50%',
            backgroundColor: '#888',
            animation: `bounce 1.4s infinite`,
            animationDelay: `${i * 0.2}s`,
          }}
        />
      ))}
    </div>
  );
}

Step 3: Markdown Rendering con react-markdown

Las respuestas no son texto plano — son Markdown renderizado:


// npm install react-markdown
import ReactMarkdown from 'react-markdown';
import remarkGfm from 'remark-gfm'; // GitHub Flavored Markdown

export function ChatMessage({ content }) {
  return (
    <ReactMarkdown
      remarkPlugins={[remarkGfm]}
      components={{
        h1: ({ node, ...props }) => (
          <h1 style={{ marginTop: '0.8em', color: '#00e676' }} {...props} />
        ),
        h2: ({ node, ...props }) => (
          <h2 style={{ marginTop: '0.6em', color: '#00e676' }} {...props} />
        ),
        ul: ({ node, ...props }) => (
          <ul style={{ marginLeft: '1.5em', lineHeight: '1.8' }} {...props} />
        ),
        code: ({ node, inline, ...props }) => (
          inline ? (
            <code style={{ backgroundColor: '#333', padding: '2px 6px', borderRadius: '4px' }} {...props} />
          ) : (
            <code {...props} />
          )
        ),
      }}
    >
      {content}
    </ReactMarkdown>
  );
}

Step 4: Code Blocks con Syntax Highlighting

Resalta código JavaScript, Python, SQL, etc. con colores:


// npm install react-syntax-highlighter
import SyntaxHighlighter from 'react-syntax-highlighter';
import { atomOneDark } from 'react-syntax-highlighter/dist/esm/styles/hljs';

export function CodeBlock({ language, code }) {
  return (
    <div style={{ margin: '1em 0', borderRadius: '8px', overflow: 'hidden' }}>
      <div style={{
        backgroundColor: '#1a1a1a',
        padding: '0.5em 1em',
        fontSize: '0.85em',
        color: '#b0b0cc',
        borderBottom: '1px solid #333'
      }}>
        {language || 'code'}
      </div>
      <SyntaxHighlighter language={language} style={atomOneDark}>
        {code}
      </SyntaxHighlighter>
    </div>
  );
}

Step 5: Copy to Clipboard Button

Botón invisible hasta hover, copia con un clic:


// CopyButton.jsx
export function CopyButton({ text }) {
  const [copied, setCopied] = React.useState(false);

  const handleCopy = () => {
    navigator.clipboard.writeText(text);
    setCopied(true);
    setTimeout(() => setCopied(false), 2000);
  };

  return (
    <button
      onClick={handleCopy}
      style={{
        padding: '0.4em 0.8em',
        backgroundColor: copied ? '#00e676' : '#333',
        color: copied ? '#000' : '#ccc',
        border: 'none',
        borderRadius: '4px',
        cursor: 'pointer',
        fontSize: '0.85em',
        transition: 'all 0.2s',
      }}
    >
      {copied ? '✓ Copied!' : 'Copy'}
    </button>
  );
}

Step 6: Auto-Scroll a Latest Message

El chat siempre muestra el último mensaje sin que el usuario scroll:


// ChatContainer.jsx
export function ChatContainer({ messages }) {
  const messagesEndRef = React.useRef(null);

  const scrollToBottom = () => {
    messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' });
  };

  React.useEffect(() => {
    scrollToBottom();
  }, [messages]);

  return (
    <div
      style={{
        flex: 1,
        overflowY: 'auto',
        padding: '1em',
        display: 'flex',
        flexDirection: 'column',
      }}
    >
      {messages.map((msg, i) => (
        <ChatBubble key={i} role={msg.role} content={msg.content} />
      ))}
      <div ref={messagesEndRef} />
    </div>
  );
}

Step 7: Responsive Layout con Tailwind CSS

Layout completo que funciona en móvil y desktop:


// App.jsx — Estructura completa
export default function ChatApp() {
  const [messages, setMessages] = React.useState([]);
  const [input, setInput] = React.useState('');
  const [sidebarOpen, setSidebarOpen] = React.useState(true);

  const handleSend = async () => {
    if (!input.trim()) return;

    // Add user message
    setMessages([...messages, { role: 'user', content: input }]);
    setInput('');

    // Simulate AI response (en real, llamarías a tu API)
    setTimeout(() => {
      setMessages(prev => [
        ...prev,
        { role: 'assistant', content: '# Respuesta\n\nEsto es una **respuesta** de ejemplo en Markdown.' }
      ]);
    }, 1000);
  };

  return (
    <div style={{ display: 'flex', height: '100vh', backgroundColor: '#121212' }}>
      {/* SIDEBAR */}
      {sidebarOpen && (
        <aside
          style={{
            width: '250px',
            backgroundColor: '#1a1a1a',
            borderRight: '1px solid #333',
            padding: '1em',
            overflowY: 'auto',
            transition: 'transform 0.3s',
          }}
        >
          <h3 style={{ color: '#00e676', marginBottom: '1em' }}>Historial</h3>
          {/* Lista de chats */}
        </aside>
      )}

      {/* MAIN CHAT */}
      <div style={{ flex: 1, display: 'flex', flexDirection: 'column' }}>
        {/* HEADER */}
        <header style={{
          borderBottom: '1px solid #333',
          padding: '1em',
          display: 'flex',
          alignItems: 'center',
          backgroundColor: '#1a1a1a',
        }}>
          <button
            onClick={() => setSidebarOpen(!sidebarOpen)}
            style={{
              background: 'none',
              border: 'none',
              color: '#ccc',
              cursor: 'pointer',
              fontSize: '1.2em',
            }}
          >
            ☰
          </button>
          <h1 style={{ marginLeft: '1em', color: '#00e676', fontSize: '1.2em' }}>AI Chat</h1>
        </header>

        {/* MESSAGES */}
        <ChatContainer messages={messages} />

        {/* INPUT SECTION */}
        <div style={{
          padding: '1em',
          borderTop: '1px solid #333',
          backgroundColor: '#1a1a1a',
          display: 'flex',
          gap: '0.5em',
        }}>
          <textarea
            value={input}
            onChange={(e) => setInput(e.target.value)}
            onKeyDown={(e) => {
              if (e.key === 'Enter' && !e.shiftKey) {
                e.preventDefault();
                handleSend();
              }
            }}
            placeholder="Escribe tu pregunta aquí... (Shift+Enter para salto de línea)"
            style={{
              flex: 1,
              padding: '0.75em',
              backgroundColor: '#2a2a2a',
              color: '#e0e0e0',
              border: '1px solid #333',
              borderRadius: '8px',
              fontFamily: 'inherit',
              resize: 'none',
              maxHeight: '120px',
            }}
          />
          <button
            onClick={handleSend}
            style={{
              padding: '0.75em 1.5em',
              backgroundColor: '#00e676',
              color: '#0a0a1a',
              border: 'none',
              borderRadius: '8px',
              cursor: 'pointer',
              fontWeight: 'bold',
            }}
          >
            Enviar ↓
          </button>
        </div>
      </div>
    </div>
  );
}

🎯 Mini-Reto: Build Complete Chat UI

Objetivo

Construye una interface de chat funcional con todos estos elementos:

  • ✅ Message bubbles (user vs assistant con estilos diferentes)
  • ✅ Typing indicator (animación de 3 puntos)
  • ✅ Markdown rendering (títulos, listas, negrita)
  • ✅ Code blocks con syntax highlighting
  • ✅ Copy to clipboard button (aparece en hover)
  • ✅ Dark mode toggle (botón en header)
  • ✅ Responsive design (funciona en móvil y desktop)
  • ✅ Auto-scroll a último mensaje
  • ✅ Sidebar colapsable con historial (mínimo 3 chats dummy)
  • ✅ Placeholder inspirador en input

Pasos

  1. Crea componentes separados para: ChatBubble, TypingIndicator, CodeBlock, CopyButton.
  2. Usa react-markdown para renderizar MD en respuestas.
  3. Usa react-syntax-highlighter para color en código.
  4. Implementa useRef + scrollIntoView para auto-scroll.
  5. Añade toggle de dark/light mode con useState (guarda en localStorage).
  6. Haz el sidebar colapsable con media queries CSS.
  7. Test en móvil (DevTools → Responsive Mode).

Bonus

  • 🔥 Llamadas reales a OpenAI API o similar.
  • 🔥 Historial persistente en IndexedDB.
  • 🔥 Búsqueda en historial con Cmd+K (estilo ChatGPT).
  • 🔥 Animación de "scroll to latest" suave.
  • 🔥 Soporte para archivos (drag-drop).
