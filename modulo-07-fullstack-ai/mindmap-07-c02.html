🧠 Vercel AI SDK

Streaming de respuestas en React

▶ ¿Qué es Vercel AI SDK?
Definición: Toolkit TypeScript para construir aplicaciones de IA con múltiples LLMs
Características principales:
  • Interfaz unificada para OpenAI, Anthropic, Google, Ollama
  • Streaming nativo de respuestas
  • Hooks React listos (useChat, useCompletion)
  • Integración perfecta con Next.js
  • TypeScript first
¿Por qué usarlo? No reinventes la rueda — manejo de streams, estado de mensajes, errores... ya está optimizado.
▶ Streaming vs Respuesta Completa
Sin Streaming (Mala UX):
Usuario envía → LLM procesa (lag) → Respuesta completa de golpe → UI actualiza
Sensación: "¿Está colgado?"
Con Streaming (Buena UX):
Usuario envía → Chunks llegan en tiempo real → UI se actualiza mientras llegan → Sensación de fluidez
Ejemplo: ChatGPT, Copilot
Ventajas: Mejor percepción de velocidad, mejor engagement, user experience como ChatGPT
▶ Arquitectura: Flujo Completo
1. Cliente (React): useChat Hook
  • Gestiona input, mensajes, estado loading
  • Envía POST /api/chat
  • Recibe chunks SSE
  • Actualiza UI automáticamente
2. Servidor (Next.js): API Route
  • Recibe { messages } del cliente
  • Llama a streamText()
  • Convierte a SSE con .toDataStreamResponse()
  • Envía chunks al cliente
3. LLM (OpenAI, Anthropic...): Genera respuesta token por token
▶ Funciones Core del AI SDK
streamText() — En el servidor
model: openai('gpt-4o-mini')
messages: [...],
system: 'Eres un asistente...',
temperature: 0.7
→ Retorna stream → .toDataStreamResponse()
generateText() — En el servidor
const { text } = await generateText({...})
→ Respuesta COMPLETA de golpe (no streaming)
useChat() — En el cliente (React)
const { messages, input, handleInputChange,
handleSubmit, isLoading } = useChat({
api: '/api/chat'
})
→ Gestiona TODO automáticamente
useCompletion() — En el cliente
Similar a useChat() pero SIN historial
→ Para autocompletado, sugerencias
▶ Proveedores Soportados
OpenAI:
import { openai } from '@ai-sdk/openai';
model: openai('gpt-4o-mini')
Anthropic (Claude):
import { anthropic } from '@ai-sdk/anthropic';
model: anthropic('claude-sonnet-4-6')
Google (Gemini):
import { google } from '@ai-sdk/google';
model: google('gemini-2.0-flash')
Ollama (Local):
import { ollama } from 'ollama-ai-sdk';
model: ollama('llama2')
Cambiar proveedor es trivial: Solo reemplaza la importación y la línea del modelo
▶ useChat Hook: Qué gestiona automáticamente
Estado de Mensajes: Array completo de user + assistant, persistido en componente
Loading States: isLoading boolean para deshabilitar botones/inputs
Streaming: Recibe chunks SSE y actualiza messages[].content en tiempo real
Error Handling: Campo error para mostrar problemas de conexión/API
Input Management: handleInputChange actualiza el estado del input
Sin useChat, tendrías que:
  • Mantener estado de mensajes (useState)
  • Parsear SSE manualmente (EventSource)
  • Actualizar UI con cada chunk
  • Manejar errores de conexión
  • 100+ líneas de boilerplate
▶ Parámetros Clave en streamText()
system: Define el rol y comportamiento del asistente
system: 'Eres un profesor de Python. Explica con ejemplos.'
temperature (0-1): Controla la creatividad
  • 0 = Determinista (siempre misma respuesta)
  • 0.7 = Equilibrio (por defecto)
  • 1 = Muy creativo (aleatorio)
maxTokens: Limita la longitud máxima de respuesta
maxTokens: 500 // ≈ 1250 palabras
messages: Array de { role: 'user'|'assistant', content: string }
▶ Setup en 5 Pasos
1. Instalar:
npm install ai @ai-sdk/openai
2. Variables de entorno (.env.local):
OPENAI_API_KEY=sk-proj-xxxxx
3. API Route (app/api/chat/route.ts):
POST request → streamText() → .toDataStreamResponse()
4. React Component ('use client'):
const { ... } = useChat({ api: '/api/chat' })
5. Renderizar mensajes y formulario: Listo para chattear
▶ Casos de Uso Comunes
Chatbot de Soporte: Agente que responde preguntas en tiempo real con system prompt específico
Asistente Personal: Toma notas, resume, proporciona ideas basadas en conversación
Generador de Contenido: Blog, redes sociales, emails (useCompletion)
Tutor Interactivo: Sistema prompt como "profesor de Python" o "entrenador"
Análisis de Documentos: Context en mensajes, análisis basado en chunks
Juegos conversacionales: Role-playing, aventuras de texto con IA
▶ Errores Comunes y Soluciones
❌ useChat en componente SIN 'use client': Error de hidratación → Solución: Añade 'use client' al principio
❌ API Key en código frontend: Inseguro, expone credenciales → Solución: Variables de entorno en .env.local
❌ No manejar isLoading: Usuario puede clickear enviar 10 veces → Solución: disabled={isLoading}
❌ Olvidar .toDataStreamResponse(): No hace streaming → Solución: Siempre usa esto en la API Route
❌ Temperature fuera de rango: Valores > 1 dan error → Solución: Usa 0-1
