Opción A: Chatbot IA — Requisitos Mínimos (MVP)

  • ✅ Next.js 14+ con App Router: Backend y frontend en un solo proyecto, deployable sin fricción
  • ✅ Vercel AI SDK con streaming: Las respuestas del IA llegan en tiempo real, no esperamos a que termine de pensar
  • ✅ Autenticación con Auth.js: Al menos un provider OAuth (Google, GitHub, etc.). Los usuarios tienen sesión segura
  • ✅ Rate limiting: Mínimo 20 requests por minuto por usuario. Evita abuso y controlamos costos de API
  • ✅ Chat UI profesional: Soporte markdown, bloques de código formateados, indicador de escritura, historial limpio
  • ✅ System prompt personalizado: El asistente tiene una personalidad única para tu caso de uso elegido
  • ✅ Responsive: Funciona perfectamente en móvil, tablet y desktop
  • ✅ Deployado en Vercel: URL pública, dominio personalizado opcional, actualizaciones automáticas desde GitHub

Bonus (Gana XP Extra)

✍️ Opción B: AI Co-Writer — Editor Inteligente RECOMENDADO

En lugar de un chatbot (que todo el mundo tiene), construye algo que casi nadie tiene en su portfolio: un editor de texto con IA integrada. Seleccionas un párrafo, haces clic en "Reescribir", "Expandir" o "Resumir", y la IA lo transforma en tiempo real. Como Notion AI, pero tuyo.

¿Por qué el Co-Writer es mejor para tu portfolio?

  • Diferenciación: El 95% de portfolios de IA junior tienen un chatbot. Un Co-Writer muestra que entiendes UX, selección de texto, prompts contextuales y streaming — nivel senior.
  • Complejidad real: Combina editor de texto + API IA + selección de texto + streaming parcial + RAG opcional. Recruiters ven esto y piensan "este sabe".
  • Producto útil: Tú mismo lo usarás para escribir README, emails, documentación. No es un demo que nadie usa.

Requisitos del Co-Writer (MVP)

  • ✅ Editor Markdown: Área de texto con preview en tiempo real (usa react-markdown o @uiw/react-md-editor)
  • ✅ Selección + Acciones IA: El usuario selecciona texto → aparece un menú flotante con 4 acciones:
    • 🔄 Reescribir — misma idea, mejor redacción
    • 📝 Expandir — añade detalle y ejemplos
    • ✂️ Resumir — comprime en menos palabras
    • 🌍 Traducir — español ↔ inglés
  • ✅ Streaming parcial: La IA reemplaza el texto seleccionado mientras se genera (no espera a terminar)
  • ✅ Undo/Redo: Ctrl+Z deshace la última acción de IA (guarda el texto original antes de cada transformación)
  • ✅ Prompt completo desde texto: Botón "Escribe sobre..." donde describes un tema y la IA genera un borrador inicial
  • ✅ Auth + Rate Limit: Mismo requisito que la Opción A
  • ✅ Deployado en Vercel: URL pública funcional

Prompts por Acción (lib/prompts.ts)

Cada acción tiene un prompt específico. El truco es que SOLO devuelves el texto transformado, sin explicaciones:

export const ACTION_PROMPTS = {
  rewrite: `Reescribe el siguiente texto manteniendo el mismo significado
pero mejorando la claridad y fluidez. Devuelve SOLO el texto reescrito,
sin explicaciones ni comillas.\n\nTexto:\n{text}`,

  expand: `Expande el siguiente texto añadiendo detalles, ejemplos y
explicaciones. Mantén el tono original. Devuelve SOLO el texto expandido,
sin explicaciones.\n\nTexto:\n{text}`,

  summarize: `Resume el siguiente texto en aproximadamente 1/3 de su
longitud original. Mantén las ideas clave. Devuelve SOLO el resumen,
sin explicaciones.\n\nTexto:\n{text}`,

  translate: `Traduce el siguiente texto. Si está en español, traduce al
inglés. Si está en inglés, traduce al español. Devuelve SOLO la
traducción.\n\nTexto:\n{text}`,

  generate: `Escribe un borrador sobre el siguiente tema. Usa markdown
con headers, listas y párrafos claros. Tono profesional pero accesible.
Extensión: 300-500 palabras.\n\nTema:\n{topic}`
};

Clave Técnica: Selección de Texto + Menú Flotante

Este es el componente más interesante. Usa window.getSelection() para detectar qué ha seleccionado el usuario:

// components/SelectionToolbar.tsx (concepto)
function SelectionToolbar({ onAction }) {
  const [position, setPosition] = useState(null);
  const [selectedText, setSelectedText] = useState('');

  useEffect(() => {
    const handleMouseUp = () => {
      const selection = window.getSelection();
      if (selection && selection.toString().trim().length > 5) {
        const range = selection.getRangeAt(0);
        const rect = range.getBoundingClientRect();
        setPosition({
          top: rect.top - 50 + window.scrollY,
          left: rect.left + rect.width / 2
        });
        setSelectedText(selection.toString());
      } else {
        setPosition(null);
      }
    };
    document.addEventListener('mouseup', handleMouseUp);
    return () => document.removeEventListener('mouseup', handleMouseUp);
  }, []);

  if (!position) return null;

  return (
    <div style={{ position: 'absolute', ...position }}>
      <button onClick={() => onAction('rewrite', selectedText)}>
        🔄 Reescribir
      </button>
      <button onClick={() => onAction('expand', selectedText)}>
        📝 Expandir
      </button>
      {/* ... más botones */}
    </div>
  );
}

Paso 3: Configura Autenticación con Auth.js

Esto es crítico para tracking de usuarios y rate limiting:

  • Crea un OAuth app en Google Cloud Console (o GitHub)
  • Configura src/lib/auth.ts con tus credentials
  • Añade NEXTAUTH_SECRET y NEXTAUTH_URL a .env.local
  • Protege las rutas sensibles (API /chat, /history) con getServerSession()

Tip: Para desarrollo rápido, usa Auth.js en modo "desarrollo" sin verificación de correo.

Paso 4: Implementa el Endpoint del Chat

Crea src/app/api/chat/route.ts — este es el corazón de tu app:

  • Recibe POST con { messages: Message[], model: string }
  • Valida que el usuario está autenticado (getServerSession)
  • Aplica rate limiting (máximo 20 req/min por usuario_id)
  • Construye el system prompt (personalizado para tu caso de uso)
  • Llama a OpenAI/Claude con generateStream() de Vercel AI SDK
  • Devuelve streaming de respuesta (no esperes a terminar)
  • Guarda el mensaje en BD si es bonus (persistencia)

Ejemplo de system prompt: "Eres un asistente experto en magia con 20 años de experiencia. Cuando el usuario pregunta por un truco, sugieres opciones basadas en el nivel de dificultad y propósito del evento. Siempre explicas brevemente el fundamento mágico."

Paso 5: Construye la UI del Chat

Crea componentes React para:

  • ChatUI: Contenedor principal. Maneja estado de mensajes, loading, errores
  • Message: Renderiza cada mensaje. Soporte markdown, código con syntax highlighting
  • InputBox: Campo de entrada con envío de Enter, indicador de carga, contador de caracteres
  • Typing indicator: "el asistente está escribiendo..." mientras llega la respuesta

Tip: Usa useChat del hook ai/react para simplificar el streaming. Automáticamente maneja el historial y actualiza en tiempo real.

Paso 6: Implementa Rate Limiting

Protege tu API de abuso:

  • Usa una librearía como Ratelimit (Vercel Edge) o Redis
  • Checkea antes de procesar cada request: await ratelimit.limit(user_id)
  • Devuelve 429 (Too Many Requests) si se excede
  • Muestra al usuario: "Espera 30 segundos antes de la siguiente pregunta"

Configuración recomendada: 20 requests / minuto por usuario (testing) o 50 / hora (producción).

Paso 7: Testing y Polish

Antes de deployar:

  • Prueba el chat localmente: escribe varias preguntas, verifica que las respuestas fluyen en tiempo real
  • Verifica autenticación: logout y login con OAuth
  • Prueba rate limiting: envía 25 requests rápidamente, confirma que falla en el 21º
  • Test en móvil: respuestas, input box, todo debe ser readable
  • Error handling: ¿qué pasa si la API de OpenAI falla? ¿Si se pierde conexión?
  • Performance: mide speed (FirstContentfulPaint, etc.) con Lighthouse

Paso 8: Deploy en Vercel

Tu app está lista:

  • Haz push a GitHub (rama main)
  • Conecta el repo a Vercel (vercel.com > Import Project > GitHub)
  • Añade variables de entorno en Vercel: OPENAI_API_KEY, NEXTAUTH_SECRET, DATABASE_URL, etc.
  • Deploy se ejecuta automáticamente. En 2 minutos, tu app está live en una URL pública
  • Dominio personalizado (optional): Apunta tu propio dominio a Vercel

Recuerda: Los secrets no se suben a GitHub — solo a Vercel Environment Variables.

  • La app está deployada en Vercel con URL pública y funcional

  • El system prompt está personalizado para mi caso de uso (la respuesta refleja la "personalidad")

💡 Tips y Trampas Comunes

🔴 Trampa 1: Olvidar .gitignore. Si subes .env.local con tu API key a GitHub, está comprometida. Solución: Asegúrate de que .env.local está en .gitignore antes de hacer push. Verifica con git status antes de cada commit.

🔴 Trampa 2: No implementar rate limiting. Sin límites, un usuario malicioso puede drenar tu presupuesto de OpenAI en minutos. Solución: Implementa rate limiting DESDE EL DÍA 1. No es opcional, es crítico.

🔴 Trampa 3: Sistema prompt genérico. Si tu asistente responde como ChatGPT normal, no hay diferencial. Solución: Escribe un system prompt que refleje tu caso de uso. Ejemplo: "Eres un coach de ajedrez que explica movimientos brillantes. Siempre sugiere la siguiente jugada táctica."

🔴 Trampa 4: No testear en móvil. Tu UI se ve bien en desktop pero es inutilizable en phone. Solución: Abre DevTools, modo responsive, prueba en 375px y 768px. Asegúrate de que el input box no desaparece, los mensajes son legibles.

💡 Consejo Pro: Antes de gastar dinero en OpenAI, prueba con Ollama (modelos gratuitos locales como Mistral). Cuando esté listo el MVP, cambia a OpenAI. Ahorras $100s en development.

💡 Consejo Pro 2: Si quieres bonus RAG, integra LangChain + Chroma en el endpoint del chat. Busca documentos relevantes al prompt del usuario y pásalos al sistema como contexto. El LLM responde basándose en TUS fuentes — mucho más personalizado.
