📚 Resumen de conceptos

¿Qué es una AI App Full-Stack?

Una aplicación de IA completa ("full-stack") integra tres capas fundamentales:

  • Frontend: La interfaz de usuario (React, Vue, etc.). Recibe input del usuario.
  • Backend/API: El servidor que valida, procesa y orquesta (Next.js, Flask, Express, etc.).
  • LLM Provider: El modelo de IA externo (OpenAI, Anthropic, etc.) o local.
Flujo básico: Usuario escribe → Frontend envía al Backend → Backend llama al LLM → LLM responde → Backend procesa → Frontend muestra resultado.

Arquitectura típica: Frontend → Backend → LLM

El patrón más común en producción es una arquitectura de tres capas:

Arquitectura

│         FRONTEND (React)                 │

                           │

│  - Logs y auditoría                      │
└──────────────────────────┬───────────────┘

                           │

│  - Procesar prompt                       │
│  - Generar respuesta                     │
│  - Contar tokens                         │
└─────────────────────────────────────────┘

Separación de responsabilidades: ¿Por qué el LLM va en el backend?

Las API keys (tokens de autenticación) NUNCA deben estar en el frontend. Razones:

  • Seguridad: Una API key en el navegador es visible para cualquiera (inspector de Chrome).
  • Control de costos: Sin backend, cualquiera podría abusar de tu API y vaciar tu presupuesto.
  • Rate limiting: El backend controla cuántas llamadas por usuario, por hora, etc.
  • Logs y auditoría: Necesitas saber quién hizo qué y cuándo para cumplir normativa.
  • Validación: El backend valida inputs antes de enviar al LLM (evita prompts inyectados).
Regla de oro: API keys siempre en variables de entorno del servidor. Frontend solo conoce la URL del API endpoint.

Next.js como framework ideal para AI Full-Stack

Next.js es perfecto para apps de IA porque:

  • React + API Routes: Frontend y backend en el mismo repositorio.
  • Serverless: Las API routes se despliegan como funciones sin servidor (escalable).
  • SSR/SSG: Server-side rendering para mejor SEO y performance.
  • TypeScript integrado: Tipado fuerte (evita bugs).
  • Vercel: Deploy automático, variables de entorno seguras, logs.
Alternativas: Flask (Python ligero) + Render, Express (Node.js) + Railway, Django (Python robusto) + PythonAnywhere.

Vamos a construir paso a paso una AI app mínima pero funcional. Seguiremos una arquitectura real de producción.

Paso 1: Crear un proyecto Next.js

Ejecuta en tu terminal:

bash 
npx create-next-app@latest mi-app-ia \
  --typescript \
  --tailwind \
  --eslint \
  --app \
  --no-src-dir

cd mi-app-ia
Flags usados:
  • --typescript: Tipado fuerte (evita bugs)
  • --tailwind: CSS framework (estilos rápido)
  • --eslint: Linter (código limpio)
  • --app: Usa App Router (más moderno)
  • --no-src-dir: Estructura raíz más limpia

Paso 2: Crear API route que llama a OpenAI

Crea el archivo app/api/chat/route.ts (Next.js API Route):

TypeScript 
// app/api/chat/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function POST(request: NextRequest) {
  try {
    // 1. Leer el mensaje del usuario desde el request
    const { message } = await request.json();

    if (!message || message.trim() === '') {
      return NextResponse.json(
        { error: 'El mensaje no puede estar vacío' },
        { status: 400 }
      );
    }

    // 2. Validación básica
    if (message.length > 1000) {
      return NextResponse.json(
        { error: 'El mensaje es demasiado largo (máx 1000 caracteres)' },
        { status: 400 }
      );
    }

    // 3. Obtener API key del backend (nunca del frontend)
    const apiKey = process.env.OPENAI_API_KEY;
    if (!apiKey) {
      return NextResponse.json(
        { error: 'API key no configurada en el servidor' },
        { status: 500 }
      );
    }

    // 4. Llamar a OpenAI API
    const response = await fetch('https://api.openai.com/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${apiKey}`,
      },
      body: JSON.stringify({
        model: 'gpt-4o-mini',
        messages: [
          {
            role: 'user',
            content: message,
          },
        ],
        max_tokens: 500,
        temperature: 0.7,
      }),
    });

    // 5. Procesar respuesta de OpenAI
    if (!response.ok) {
      const error = await response.json();
      return NextResponse.json(
        { error: `Error de OpenAI: ${error.error.message}` },
        { status: response.status }
      );
    }

    const data = await response.json();
    const reply = data.choices[0].message.content;

    // 6. Devolver respuesta al frontend
    return NextResponse.json({
      success: true,
      reply,
      model: data.model,
      usage: data.usage,
    });

  } catch (error) {
    console.error('Error en API /chat:', error);
    return NextResponse.json(
      { error: 'Error interno del servidor' },
      { status: 500 }
    );
  }
}
¿Qué pasa aquí?
  1. El frontend envía un JSON: { "message": "Hola IA" }
  2. El backend valida el mensaje
  3. El backend obtiene la API key de la variable de entorno (NO del código)
  4. El backend llama a OpenAI con el mensaje
  5. El backend procesa la respuesta y la devuelve al frontend

Paso 3: Crear componente React que llama al API

Crea app/components/ChatBox.tsx:

TypeScript 
// app/components/ChatBox.tsx
'use client';

import { useState } from 'react';

interface Message {
  id: string;
  text: string;
  sender: 'user' | 'ai';
}

export default function ChatBox() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [input, setInput] = useState('');
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState('');

  // Función para enviar mensaje
  const handleSendMessage = async (e: React.FormEvent) => {
    e.preventDefault();
    if (!input.trim() || isLoading) return;

    // 1. Agregar mensaje del usuario al UI
    const userMessage: Message = {
      id: Date.now().toString(),
      text: input,
      sender: 'user',
    };
    setMessages((prev) => [...prev, userMessage]);
    setInput('');
    setIsLoading(true);
    setError('');

    try {
      // 2. Llamar al API route del backend
      const response = await fetch('/api/chat', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ message: input }),
      });

      if (!response.ok) {
        const data = await response.json();
        throw new Error(data.error || 'Error desconocido');
      }

      const data = await response.json();

      // 3. Agregar respuesta de la IA al UI
      const aiMessage: Message = {
        id: (Date.now() + 1).toString(),
        text: data.reply,
        sender: 'ai',
      };
      setMessages((prev) => [...prev, aiMessage]);

    } catch (err) {
      setError(err instanceof Error ? err.message : 'Error al conectar con la IA');
      console.error('Error:', err);
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <div className="chat-container">
      <div className="messages">
        {messages.map((msg) => (
          <div key={msg.id} className={`message ${msg.sender}`}>
            <p>{msg.text}</p>
          </div>
        ))}
        {isLoading && <div className="message ai loading">Pensando...</div>}
        {error && <div className="message error">{error}</div>}
      </div>

      <form onSubmit={handleSendMessage} className="input-form">
        <input
          type="text"
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="Escribe tu pregunta..."
          disabled={isLoading}
        />
        <button type="submit" disabled={isLoading}>
          {isLoading ? 'Enviando...' : 'Enviar'}
        </button>
      </form>
    </div>
  );
}
Conceptos clave:
  • 'use client': Este componente es interactivo (debe ejecutarse en el navegador)
  • useState: Gestiona el estado (mensajes, input, loading)
  • fetch('/api/chat'): Llamada al backend (NO a OpenAI directamente)
  • El usuario nunca ve la API key de OpenAI

Paso 5: Configurar variables de entorno

Crea un archivo .env.local en la raíz del proyecto:

bash 
# .env.local
OPENAI_API_KEY=sk-tu-api-key-aqui
⚠️ IMPORTANTE:
  • NUNCA commits .env.local a Git. Está en .gitignore por defecto.
  • En desarrollo local, crea una API key en platform.openai.com.
  • En producción (Vercel), configura las variables en el dashboard de Vercel (Settings → Environment Variables).

Paso 6: Testear end-to-end

bash 
# 1. Instalar dependencias (si no las has instalado)
npm install

# 2. Ejecutar servidor de desarrollo
npm run dev

# 3. Abre http://localhost:3000 en tu navegador
# 4. Escribe un mensaje y dale a "Enviar"
# 5. Deberías ver la respuesta de OpenAI en pocos segundos
¿Qué está pasando?
  1. Escribes "Hola" en el input del frontend
  2. React envía un POST a /api/chat con JSON { "message": "Hola" }
  3. El backend recibe el request, valida, obtiene la API key
  4. El backend llama a OpenAI API con tu mensaje
  5. OpenAI procesa y responde (tarda 1-3 segundos típicamente)
  6. El backend devuelve la respuesta al frontend como JSON
  7. React actualiza el UI y muestra la respuesta

🎯 Mini-reto del día

Construye una AI App Simple

Sigue los pasos 1-6 de la práctica guiada y crea una app funcional que:

  • Tenga un input para escribir mensajes
  • Envíe el mensaje al backend (no directamente a OpenAI)
  • El backend valide el mensaje (no vacío, máx 1000 caracteres)
  • El backend llame a OpenAI y devuelva la respuesta
  • El frontend muestre el diálogo de forma clara
  • Muestre un estado "Pensando..." mientras se procesa
  • Maneje errores de forma elegante

Criterios de éxito:

  • ✅ La app se ejecuta en npm run dev sin errores
  • ✅ Puedes escribir mensajes y recibir respuestas de OpenAI
  • ✅ La API key NO está en el código frontend (inspect de Chrome no la muestra)
  • ✅ Validación básica funciona (mensaje vacío muestra error)
  • ✅ El estado "Pensando..." aparece mientras procesa

Bonus (opcional):

  • Añade CSS personalizado para que se vea profesional
  • Guarda el historial de mensajes en localStorage
  • Añade un botón "Limpiar chat" para resetear la conversación
  • Muestra tokens gastados (en respuesta de OpenAI)
  • Implementa un sistema de "personalidad" del bot (system prompt)
