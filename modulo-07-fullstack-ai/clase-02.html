📚 Resumen de Conceptos

¿Qué es el Vercel AI SDK?

Es un toolkit TypeScript de bajo nivel para construir aplicaciones de IA. Proporciona abstracciones unificadas para trabajar con múltiples proveedores de LLMs (OpenAI, Anthropic, Google, Ollama, etc.) con una interfaz consistente.

Por qué es importante:

  • Una interfaz unificada para múltiples LLMs (cambiar de proveedor es fácil)
  • Soporte nativo para streaming de respuestas
  • Hooks React listos para usar (useChat, useCompletion)
  • Integración perfecta con Next.js App Router y API Routes
  • TypeScript first — excelente DX (developer experience)

Streaming vs Respuesta Completa

Respuesta completa (sin streaming):

Usuario espera a que termine todo 
Usuario envía mensaje
         ↓
    LLM procesa (3-5 segundos)
         ↓
   Recibe TODA la respuesta de golpe
         ↓
   UI se actualiza (lag perceptible)

Con streaming (mejor UX):

El usuario ve la respuesta en tiempo real 
Usuario envía mensaje
         ↓
   LLM empieza a generar
         ↓
  Chunk 1: "Hola,"
  Chunk 2: " ¿cómo"
  Chunk 3: " estás?"
         ↓
   UI actualiza mientras llegan chunks (sensación de fluidez)

Ventaja principal: El usuario percibe que la respuesta es inmediata, aunque el LLM esté procesando. Es la misma experiencia que ChatGPT o Copilot.

El Flujo Completo: User → React → API → LLM → Streaming → UI

Funciones Core del AI SDK

1. streamText() — Genera texto con streaming en el servidor

TypeScript 
import { streamText } from 'ai';
import { openai } from '@ai-sdk/openai';

const result = await streamText({
  model: openai('gpt-4o-mini'),
  messages: [
    { role: 'user', content: 'Explica qué es streaming' }
  ],
  system: 'Eres un asistente de IA útil',
  temperature: 0.7,
});

// result contiene el stream y metadata
return result.toDataStreamResponse();

2. generateText() — Genera texto sin streaming (respuesta completa)

TypeScript 
const result = await generateText({
  model: openai('gpt-4o-mini'),
  prompt: '¿Cuál es la capital de Francia?',
});

console.log(result.text); // "París"

3. useChat() — Hook React para chat con historial

TypeScript (React Client Component) 
'use client';
import { useChat } from 'ai/react';

export default function Chat() {
  const {
    messages,        // Array de mensajes previos
    input,           // Texto del input actual
    handleInputChange, // Para actualizar el input
    handleSubmit,    // Para enviar mensaje
    isLoading,       // Si está esperando respuesta
    error            // Si hay error
  } = useChat({
    api: '/api/chat', // Ruta de la API
  });

  return (
    <>
      <div className="messages">
        {messages.map((m) => (
          <div key={m.id} className={m.role}>
            {m.content}
          </div>
        ))}
      </div>
      <form onSubmit={handleSubmit}>
        <input
          value={input}
          onChange={handleInputChange}
          placeholder="Escribe un mensaje..."
          disabled={isLoading}
        />
        <button type="submit" disabled={isLoading}>
          {isLoading ? 'Pensando...' : 'Enviar'}
        </button>
      </form>
    </>
  );
}

4. useCompletion() — Hook para completación de texto (sin historial de chat)

TypeScript (React) 
const { completion, input, handleInputChange, handleSubmit } = useCompletion({
  api: '/api/completion',
});

// Similar a useChat pero devuelve un string de completación
// Útil para sugerencias, autocompletado, etc.

Proveedores Soportados

El AI SDK funciona con múltiples proveedores. Cambiar es tan simple como cambiar una línea:

TypeScript 
// OpenAI
import { openai } from '@ai-sdk/openai';
const model = openai('gpt-4o-mini');

// Anthropic Claude
import { anthropic } from '@ai-sdk/anthropic';
const model = anthropic('claude-sonnet-4-6');

// Google Gemini
import { google } from '@ai-sdk/google';
const model = google('gemini-2.0-flash');

// Ollama (local)
import { ollama } from 'ollama-ai-sdk';
const model = ollama('llama2');

// Todos se usan de la misma forma en streamText()
const result = await streamText({
  model: model, // ← Intercambia el proveedor aquí
  messages,
});

El Hook useChat — Gestión Automática de Estado

useChat gestiona automáticamente:

  • Historial de mensajes: Mantiene un array de todos los mensajes usuario/asistente
  • Estados de carga: isLoading para deshabilitar botones
  • Streaming en cliente: Los chunks llegan y actualizan la UI automáticamente
  • Manejo de errores: Campo error para mostrar problemas
  • Limpieza: Función setMessages para resetear si necesitas

Sin useChat tendrías que:

  • Mantener tu propio estado de mensajes
  • Hacer fetch manualmente a la API
  • Parsear el stream SSE
  • Actualizar la UI con cada chunk

Con useChat, todo eso ya está hecho.

⚙️ Práctica Guiada

Vamos a construir un chatbot con Vercel AI SDK paso a paso.

Paso 1: Instalar Dependencias

bash 
npm install ai @ai-sdk/openai

Nota: Necesitas una clave API de OpenAI. Crea una en platform.openai.com y guárdala en .env.local:

.env.local 
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxx

Paso 2: Crear la API Route en el Servidor

Crea el archivo app/api/chat/route.ts:

TypeScript — app/api/chat/route.ts 
import { streamText } from 'ai';
import { openai } from '@ai-sdk/openai';

export async function POST(req: Request) {
  // 1. Recibir los mensajes del cliente
  const { messages } = await req.json();

  // 2. Llamar a streamText con el modelo
  const result = streamText({
    model: openai('gpt-4o-mini'),
    messages,
    system: 'Eres un asistente amable y útil. Responde en español.',
    temperature: 0.7,
  });

  // 3. Convertir el stream a Server-Sent Events
  return result.toDataStreamResponse();
}

¿Qué pasa aquí?

  • streamText() inicia la generación del LLM
  • El parámetro system define el comportamiento del asistente
  • .toDataStreamResponse() convierte el stream a respuesta HTTP SSE
  • Los chunks llegan al cliente automáticamente

Paso 3: Crear el Componente React con useChat

Crea el archivo app/components/ChatInterface.tsx:

TypeScript — app/components/ChatInterface.tsx 
'use client'; // ← Necesario para hooks de cliente

import { useChat } from 'ai/react';

export default function ChatInterface() {
  const {
    messages,
    input,
    handleInputChange,
    handleSubmit,
    isLoading,
    error
  } = useChat({
    api: '/api/chat',
  });

  return (
    <div style={{ display: 'flex', flexDirection: 'column', height: '100vh' }}>
      {/* Área de mensajes */}
      <div style={{ flex: 1, overflowY: 'auto', padding: '20px' }}>
        {messages.map((message) => (
          <div
            key={message.id}
            style={{
              marginBottom: '16px',
              textAlign: message.role === 'user' ? 'right' : 'left',
            }}
          >
            <div
              style={{
                display: 'inline-block',
                maxWidth: '70%',
                padding: '12px',
                borderRadius: '8px',
                backgroundColor: message.role === 'user' ? '#0070f3' : '#e5e5e5',
                color: message.role === 'user' ? 'white' : 'black',
              }}
            >
              {message.content}
            </div>
          </div>
        ))}
        {isLoading && <p>Pensando...</p>}
        {error && <p style={{ color: 'red' }}>Error: {error.message}</p>}
      </div>

      {/* Input de mensaje */}
      <form
        onSubmit={handleSubmit}
        style={{
          display: 'flex',
          gap: '8px',
          padding: '20px',
          borderTop: '1px solid rgba(0, 240, 255, 0.15)',
        }}
      >
        <input
          value={input}
          onChange={handleInputChange}
          placeholder="Escribe tu mensaje..."
          disabled={isLoading}
          style={{
            flex: 1,
            padding: '12px',
            borderRadius: '8px',
            border: '1px solid rgba(0, 240, 255, 0.15)',
            fontSize: '16px',
          }}
        />
        <button
          type="submit"
          disabled={isLoading}
          style={{
            padding: '12px 24px',
            backgroundColor: isLoading ? '#ccc' : '#0070f3',
            color: 'white',
            border: 'none',
            borderRadius: '8px',
            cursor: isLoading ? 'not-allowed' : 'pointer',
            fontSize: '16px',
          }}
        >
          {isLoading ? 'Enviando...' : 'Enviar'}
        </button>
      </form>
    </div>
  );
}

Paso 4: Agregar System Prompt y Temperature

Puedes personalizar el comportamiento del modelo con parámetros:

TypeScript 
const result = streamText({
  model: openai('gpt-4o-mini'),
  messages,

  // System prompt: instrucciones del rol del asistente
  system: `Eres un agente de soporte técnico.
  - Sé educado y profesional
  - Responde en español
  - Si no sabes, pregunta para más contexto`,

  // Temperature: controla creatividad (0-1)
  // 0 = determinista, 1 = muy creativo
  temperature: 0.5,

  // Max tokens: limita la longitud de la respuesta
  maxTokens: 500,
});

return result.toDataStreamResponse();

Paso 5: Usar el Componente en tu App

TypeScript — app/page.tsx 
import ChatInterface from '@/app/components/ChatInterface';

export default function Home() {
  return (
    <main>
      <ChatInterface />
    </main>
  );
}

🎯 Mini-Reto

Construye un Chatbot con Rol Específico

Crea una aplicación de chat donde el asistente adopte un rol específico según el parámetro.

Requisitos:

  • Selecciona un rol (ej: "Profesor de Python", "Entrenador de Fitness", "Chef de Cocina")
  • Define un system prompt que capture ese rol
  • Usa useChat con streaming
  • Muestra el rol actual en la UI
  • Permite cambiar de rol sin perder el historial (o limpia el historial al cambiar)

Ejemplo de system prompt para "Profesor de Python":

TypeScript 
const systemPrompts: Record<string, string> = {
  profesor: `Eres un profesor de Python experimentado.
    - Explica conceptos con claridad
    - Usa analogías y ejemplos reales
    - Sugiere ejercicios prácticos
    - Responde siempre en español`,

  entrenador: `Eres un entrenador personal de fitness.
    - Da consejos de ejercicio seguro
    - Personaliza rutinas según el nivel
    - Motiva al usuario
    - Pregunta sobre lesiones previas`,
};

// Después, en tu API Route:
const system = systemPrompts[rol] || systemPrompts.profesor;
const result = streamText({
  model: openai('gpt-4o-mini'),
  messages,
  system, // ← Cambiar según el rol
});

Bonus: Guarda el historial en localStorage y recupera la conversación anterior.
