Cómo Destacar en una Entrevista de Trabajo como Desarrollador

Vídeo de KeepCoding: estrategias, tips y ejemplos reales para brillar en entrevistas tech (77 min).

1. Tipos de Entrevista en Empresas Tech en España

Las entrevistas técnicas tienen estructuras diferentes según el tamaño de la empresa y el rol:

Screening Call (RRHH)

Duración: 15-30 minutos

Enfoque: Motivación, disponibilidad, expectativas salariales, soft skills.

Tip: Prepara un pitch de 1 minuto sobre ti y por qué te interesa la vacante.

Entrevista Técnica

Duración: 45-60 minutos

Enfoque: Preguntas conceptuales + ejercicio de live coding. Evalúan conocimiento y pensamiento en voz alta.

Tip: Piensa en voz alta. El proceso importa más que tener la respuesta perfecta.

Prueba Técnica en Casa

Duración: 2-5 días para completar

Enfoque: Proyecto pequeño (backend, frontend, o IA). Evalúan código limpio, documentación, decisiones arquitectónicas.

Tip: Dedica tiempo a documentar tu código y explicar tus decisiones de diseño.

Entrevista Cultural Fit

Duración: 30-45 minutos

Enfoque: Valores, trabajo en equipo, adaptación a la cultura de la empresa.

Tip: Investiga la misión y valores de la empresa antes de la entrevista.

📊 Número de rondas según empresa:
  • Startups: 2-3 rondas (más ágil)
  • Empresas medianas: 3-4 rondas
  • Grandes empresas: 4-5 rondas (más formal)

2. Banco de Preguntas Frecuentes para AI Developer

Aquí tienes las preguntas más comunes en entrevistas técnicas de IA. Estudia y prepara respuestas para cada una:

Conceptos de IA / LLMs

¿Qué es un LLM y cómo funciona?

Respuesta clave: Large Language Model. Explica: tokens (unidades de texto), transformers (arquitectura), mecanismo de atención (cómo "entiende" el contexto), predicción de siguiente token.

¿Qué diferencia hay entre fine-tuning y RAG?

Respuesta clave: Fine-tuning: reentrenar el modelo con datos específicos (lento, caro). RAG: alimentar al modelo con documentos externos en tiempo real (rápido, flexible). Use fine-tuning para patrones muy específicos; RAG para información actualizada.

¿Qué son los embeddings?

Respuesta clave: Representaciones vectoriales de texto. Convertir palabras/frases en números (vectors) para que los modelos entienda similitud. Ejemplo: "gato" y "felino" tienen vectores parecidos.

¿Qué es la temperatura en un LLM?

Respuesta clave: Parámetro (0-1) que controla creatividad. T=0: determinista (respuestas fijas). T=1: más variedad. Usa T baja para tareas lógicas; T alta para escritura creativa.

¿Explica qué es prompt engineering?

Respuesta clave: Arte de escribir prompts efectivos. Técnicas: few-shot (ejemplos), chain-of-thought (pasos), system prompts (rol del modelo), instrucciones claras. Impacta directamente en calidad de respuestas.

RAG (Retrieval-Augmented Generation)

¿Cómo implementarías un sistema RAG?

Respuesta clave: Paso a paso: (1) Cargar documentos, (2) Dividir en chunks, (3) Generar embeddings, (4) Guardar en vector store, (5) En query: recuperar chunks similares, (6) Pasar al LLM con contexto, (7) Generar respuesta.

¿Qué es un vector store? Nombra algunos.

Respuesta clave: Base de datos para almacenar embeddings. Ejemplos: ChromaDB (local, fácil), FAISS (Facebook, rápido), Pinecone (cloud), Weaviate, Milvus.

¿Cómo mejorarías las respuestas de un sistema RAG?

Respuesta clave: (1) Reranking: ranking adicional de chunks recuperados. (2) Hybrid search: combinar búsqueda vectorial + keyword. (3) Chunking inteligente: mantener contexto en fragmentos. (4) Metadata filtering: filtrar por fecha, tipo, etc.

APIs y Coding

¿Cómo funciona la API de OpenAI?

Respuesta clave: REST API. Endpoint: POST /v1/chat/completions. Estructura: messages array (system, user, assistant), temperature, max_tokens. Respuesta: choices con finish_reason. Streaming disponible.

¿Qué es function calling / tool use?

Respuesta clave: Permite que el LLM ejecute funciones. Defines funciones disponibles, LLM elige cuál llamar, tú ejecutas y devuelves resultado. Ej: LLM decide llamar a "get_weather(city)" y recibe temperatura actual.

¿Cómo protegerías una app de IA contra prompt injection?

Respuesta clave: (1) Validar entrada del usuario. (2) Usar system prompts claros y que no se pueden sobrescribir. (3) Moderate API (contenido). (4) Guardrails: restricciones de temas. (5) Sandboxing de código generado.

Escribe una función que llame a la API de OpenAI con streaming

Respuesta clave (Python):

from openai import OpenAI

client = OpenAI(api_key="tu-key")

def chat_stream(prompt):
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        stream=True
    )
    for chunk in response:
        if chunk.choices[0].delta.content:
            print(chunk.choices[0].delta.content, end="", flush=True)

chat_stream("Explica qué es IA")
Proyectos y Experiencia

Háblame de tu proyecto más desafiante

Respuesta clave (Framework STAR):

  • Situación: Contexto. Ej: "Empresa necesitaba chatbot para soporte"
  • Tarea: Tu responsabilidad. Ej: "Yo era responsable de implementar el RAG"
  • Acción: Qué hiciste. Ej: "Elegí ChromaDB, parseé 500 docs, fine-tuneé embeddings"
  • Resultado: Impacto medible. Ej: "Reducimos latencia 60%, precisión 95%"

¿Qué harías diferente si empezaras tu proyecto RAG de cero?

Respuesta clave: Muestra aprendizaje. Ej: "Empezaría con estrategia de chunking clara, metrics desde día 1, tests de calidad, documentaría decisiones".

¿Cómo escalarías tu chatbot a 1000 usuarios?

Respuesta clave: Arquitectura: (1) Load balancer, (2) Cache de responses frecuentes, (3) Vector store escalable (Pinecone), (4) Queue para requests concurrentes, (5) Logging y monitoring, (6) CDN para assets.

3. Estrategias para Responder Bien

Piensa en Voz Alta

El entrevistador quiere ver tu proceso, no solo la respuesta. Si dices "hmm, déjame pensar en esto paso a paso...", demuestra que razonas bien. Es normal no saber todo.

Si No Sabes Algo

Di: "No lo sé, pero investigaría X" o "He trabajado con Y pero no con Z". Honestidad + iniciativa es mejor que bullshit.

Framework STAR para Experiencias

Situación → Tarea → Acción → Resultado. Cada respuesta sobre tu carrera debe seguir este patrón. Dados concretos (números, porcentajes) = más creíble.

Prepara 3 Preguntas para el Entrevistador

Ej: "¿Cómo es el equipo de IA en el día a día?", "¿Qué tech stack usan para RAG?", "¿Cuál fue tu reto más grande en este rol?". Demuestra interés y curiosidad genuina.

4. Tips para Live Coding

Antes de empezar:
  • Lee el problema completo sin interrupciones
  • Pregunta por edge cases y restricciones
  • Pregunta qué lenguaje prefieres usar
Mientras codeas:
  • Explica tu pensamiento ("primero voy a validar entrada")
  • Comenta tu código mientras escribes
  • Empieza con pseudocódigo, luego implementa
  • No te apures; calidad > velocidad
Plataformas de práctica:
  • LeetCode: Problemas algoritmos, nivel fácil-medio
  • HackerRank: Problemas estructurados por tema
  • Codewars: Gamificado, comunidad activa
  • CodinGame: Retos IA y lógica

5. El Simulacro: Estructura Completa (45 min)

Un simulacro de entrevista técnica real, para practicar bajo presión:

Fase 1: Preguntas Conceptuales (10 min)

El entrevistador elige 3-4 preguntas del banco de arriba y te pide que las expliques en voz alta. Énfasis en pensar en voz alta y comunicación clara.

Fase 2: Live Coding (20 min)

Problema típico: "Escribe una función que conecte con OpenAI API y devuelva respuestas con streaming". Código limpio, manejo de errores, testing.

Fase 3: Preguntas sobre Proyectos (10 min)

Explica 2-3 de tus proyectos usando framework STAR. Enfoque: qué aprendiste, retos, resultados.

Fase 4: Tus Preguntas (5 min)

Haz las 3 preguntas que preparaste. Escucha con atención, toma notas (demuestra interés).

6. Práctica Guiada

Para consolidar, completa estas 5 tareas:

Tarea 1: Banco de Respuestas Escritas

Crea un documento (Google Docs o Word) con respuestas escritas para las 15 preguntas principales del banco. Objetivo: tener contenido de referencia para estudiar.

Tarea 2: Grabación de Respuestas

Coge tu teléfono. Elige 5 preguntas al azar del banco y grábate respondiendo en voz alta (sin leer de papeles). Objetivo: acostumbrarte a explicar con fluidez.

Tarea 3: Ejercicio Live Coding

Implementa en Python/JS una función que:

  1. Conecte con OpenAI API
  2. Envíe un prompt del usuario
  3. Reciba respuesta con streaming
  4. Maneje errores (API down, rate limit, etc.)

Tarea 4: Framework STAR en Acción

Elige 3 proyectos propios (o del curso). Escribe + graba (2 min cada una) tu explicación usando STAR:

  1. S: ¿Cuál era el problema?
  2. T: ¿Cuál fue tu rol?
  3. A: ¿Qué hiciste?
  4. R: ¿Cuál fue el resultado (medible)?

Tarea 5: Preguntas para el Entrevistador

Investiga 3 empresas tech (tamaño pequeño, mediano, grande). Para cada una, prepara 3 preguntas personalizadas basadas en su misión y tech stack.

7. Desafío Final

Simulacro Completo con Compañero/Familiar

Objetivo: Replicar una entrevista real de 45 minutos.

Setup:

  1. Busca un amigo, familiar o compañero
  2. Dale el banco de preguntas + la estructura de simulacro
  3. Que te pregunte aleatoriamente (Fase 1, 2, 3, 4)
  4. Graba todo (audio + vídeo si puedes)

Revisión: Mira la grabación. ¿Dónde no fue fluido? ¿Qué mejorar? ¿Qué pregunta te sorprendió?

Bonus: Repite el simulacro 2-3 veces con diferentes personas. Cada ronda será más confiado.
