📖 Resumen

¿Qué son los entornos virtuales?

Un entorno virtual es una carpeta aislada en tu proyecto que contiene una copia de Python y todos los paquetes que instalas. Es el equivalente en Python a node_modules en JavaScript.

Ventajas:

  • Cada proyecto tiene sus propias dependencias sin conflictos
  • Fácil compartir requerimientos con otros desarrolladores
  • Control total sobre versiones de paquetes
  • Proyectos aislados uno de otro

Crear y activar entornos virtuales

Usa python -m venv para crear un entorno virtual:

# Crear un entorno virtual llamado 'venv'
python -m venv venv

# En Windows (Command Prompt)
venv\Scripts\activate

# En macOS/Linux
source venv/bin/activate

# Ver que está activado (aparece (venv) en la terminal)
(venv) $ python --version

Desactivar entornos virtuales

# En cualquier sistema
deactivate

pip: gestor de paquetes

pip (Package Installer for Python) es la herramienta para instalar y gestionar paquetes.

# Instalar un paquete
pip install requests

# Instalar una versión específica
pip install requests==2.28.0

# Instalar desde un archivo de requerimientos
pip install -r requirements.txt

# Listar paquetes instalados
pip list

# Ver información de un paquete
pip show requests

requirements.txt y freeze

requirements.txt es un archivo que lista todos los paquetes y versiones que tu proyecto necesita. Úsalo para reproducir el mismo entorno en otro lugar. 

# Generar requirements.txt con las versiones actuales
pip freeze > requirements.txt

# Instalar desde requirements.txt (perfecto para clonar un proyecto)
pip install -r requirements.txt

Contenido típico de requirements.txt:

requests==2.28.0
flask==2.2.0
python-dotenv==0.20.0
pytest==7.1.3

Estructura recomendada de proyectos Python

mi_proyecto/
├── venv/                  # Entorno virtual (nunca commitearlo)
├── src/                   # Código fuente
│   ├── __init__.py
│   ├── main.py
│   └── utils.py
├── tests/                 # Tests unitarios
│   ├── __init__.py
│   └── test_utils.py
├── README.md              # Descripción del proyecto
├── requirements.txt       # Dependencias
├── .gitignore            # Archivos a ignorar en Git
└── pyproject.toml        # Configuración del proyecto (moderno)

pyproject.toml (configuración moderna)

pyproject.toml es el estándar moderno para configurar proyectos Python. Reemplaza varias configuraciones antiguas en un archivo TOML. 

[build-system]
requires = ["setuptools>=61.0"]
build-backend = "setuptools.build_meta"

[project]
name = "mi_proyecto"
version = "0.1.0"
description = "Un proyecto Python de ejemplo"
dependencies = [
    "requests>=2.28.0",
    "flask>=2.2.0",
]

[project.optional-dependencies]
dev = ["pytest>=7.1.3", "black>=22.0.0"]

.gitignore para proyectos Python

Qué archivos NO debes commitar a Git:

# Entorno virtual
venv/
env/
ENV/

# Caché de Python
__pycache__/
*.py[cod]
*$py.class
*.so

# Archivos de entorno
.env
.env.local

# IDE
.vscode/

*.swp

# Dependencias del sistema
dist/
build/
*.egg-info/

💻 Práctica Guiada

Objetivo: Crear un proyecto completo desde cero con entorno virtual, instalar paquetes y organizar archivos correctamente.

Paso 1: Crear la estructura base

# Crear carpeta del proyecto
mkdir weather_app
cd weather_app

# Crear entorno virtual
python -m venv venv

# Activar entorno (Windows)
venv\Scripts\activate

# Activar entorno (macOS/Linux)
source venv/bin/activate

Paso 2: Crear estructura de carpetas

# En la carpeta del proyecto
mkdir src tests

# Crear archivos iniciales
type nul > src/__init__.py
type nul > tests/__init__.py
type nul > src/main.py
type nul > README.md

Paso 3: Instalar paquetes necesarios

(venv) $ pip install requests python-dotenv

# Ver qué instalaste
(venv) $ pip list

Paso 4: Crear requirements.txt

(venv) $ pip freeze > requirements.txt

# Ver el contenido
(venv) $ cat requirements.txt

Paso 5: Escribir código de ejemplo

En src/main.py:

import requests
from dotenv import load_dotenv
import os

load_dotenv()

def get_weather(city):
    """Obtiene el clima de una ciudad (ejemplo)"""
    response = requests.get(f"https://api.example.com/weather?city={city}")
    return response.json()

if __name__ == "__main__":
    weather = get_weather("Madrid")
    print(weather)

Paso 6: Crear .gitignore

# Crear archivo .gitignore en la raíz del proyecto
venv/
__pycache__/
*.py[cod]
.env
.env.local
.vscode/
.idea/

Paso 7: Desactivar entorno

(venv) $ deactivate

# Ya no ves (venv) en la terminal
$ python --version

Resultado final

weather_app/
├── venv/                  # Creado con python -m venv venv
├── src/
│   ├── __init__.py
│   └── main.py
├── tests/
│   └── __init__.py
├── README.md
├── requirements.txt       # Generado con pip freeze
└── .gitignore

🧪 Mini-reto

Desafío: Configura un proyecto completo para una "weather app" con estructura profesional.

Requisitos:

  • Crear carpeta weather_app
  • Crear y activar entorno virtual con python -m venv venv
  • Instalar al menos 3 paquetes: requests, python-dotenv, pytest
  • Crear estructura: src/, tests/
  • Generar requirements.txt con pip freeze
  • Crear .gitignore apropiado
  • Crear un archivo src/weather.py con una función que simule obtener el clima
  • Crear tests/test_weather.py con un test simple
  • Escribir un README.md que explique cómo activar el proyecto

Bonus (opcional):

  • Crea un pyproject.toml con la configuración del proyecto
  • Usa python-dotenv para cargar variables de entorno desde un .env
  • Intenta ejecutar los tests con pytest

Cómo validar tu proyecto:

# Activar entorno
source venv/bin/activate  # o venv\Scripts\activate en Windows

# Instalar dependencias desde requirements.txt
pip install -r requirements.txt

# Ejecutar tests
pytest

# Desactivar
deactivate
