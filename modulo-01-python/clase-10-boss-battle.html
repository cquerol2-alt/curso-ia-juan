🗺️ Guía Paso a Paso

1 Preparación: Estructura y Virtualenv

Qué hacer:

  1. Crea una carpeta llamada flask-game-api
  2. Entra en la carpeta: cd flask-game-api
  3. Crea el virtualenv: python -m venv venv
  4. Activa venv:
    • Linux/Mac: source venv/bin/activate
    • Windows: venv\Scripts\activate
  5. Instala Flask: pip install flask
  6. Crea requirements.txt: pip freeze > requirements.txt
Asegúrate de que (venv) aparezca en tu terminal. Si ves eso, el virtualenv está activado.
2 Modelo de Datos: juegos.json

Qué hacer:

Crea un archivo juegos.json en la raíz del proyecto con datos iniciales:

[
  {
    "id": 1,
    "titulo": "The Legend of Zelda",
    "genero": "aventura",
    "anio": 1986,
    "plataforma": "NES",
    "rating": 9.5
  },
  {
    "id": 2,
    "titulo": "Super Mario Bros",
    "genero": "plataforma",
    "anio": 1985,
    "plataforma": "NES",
    "rating": 9.0
  }
]
Este es tu "mini-base de datos". Cada juego tiene id único, título, género, año, plataforma y rating (0-10).
3 Endpoints de Lectura (GET)

Qué hacer:

Implementa dos endpoints GET:

  • GET /api/juegos — Devuelve la lista completa de juegos
  • GET /api/juegos/<id> — Devuelve un juego específico o 404 si no existe

💡 Tip: Carga los datos de juegos.json con json.load(), léelos en memoria, y devuelve el JSON.

4 Endpoints de Escritura (POST, PUT, DELETE)

Qué hacer:

  • POST /api/juegos — Crea un nuevo juego. Valida que el JSON tenga título y género. Devuelve 201 Created.
  • PUT /api/juegos/<id> — Actualiza un juego existente. Devuelve 200 OK o 404.
  • DELETE /api/juegos/<id> — Elimina un juego. Devuelve 204 No Content o 404.

💡 Tip: Después de cada cambio (POST, PUT, DELETE), reescribe juegos.json con json.dump(). Genera IDs nuevos con max([j['id'] for j in juegos]) + 1 (o usa uuid si prefieres).

5 Manejo de Errores y Testing

Qué hacer:

  • Implementa respuestas de error claras: {"error": "Mensaje de error"}
  • Usa abort(404) cuando no exista un recurso
  • Usa abort(400) cuando los datos sean inválidos
  • Testea cada endpoint con curl o Postman

▼ app.py — Aplicación Flask completa 
import json
import os
from flask import Flask, request, jsonify

app = Flask(__name__)

# Ruta al archivo de datos
DATA_FILE = 'juegos.json'

def load_games():
    """Carga los juegos desde JSON"""
    if not os.path.exists(DATA_FILE):
        return []
    with open(DATA_FILE, 'r', encoding='utf-8') as f:
        try:
            return json.load(f)
        except:
            return []

def save_games(games):
    """Guarda los juegos en JSON"""
    with open(DATA_FILE, 'w', encoding='utf-8') as f:
        json.dump(games, f, ensure_ascii=False, indent=2)

def get_next_id(games):
    """Obtiene el siguiente ID disponible"""
    if not games:
        return 1
    return max([g['id'] for g in games]) + 1

# ============= ENDPOINTS =============

@app.route('/api/juegos', methods=['GET'])
def get_all_games():
    """GET /api/juegos — Devuelve todos los juegos"""
    games = load_games()
    return jsonify(games), 200

@app.route('/api/juegos/<int:game_id>', methods=['GET'])
def get_game(game_id):
    """GET /api/juegos/<id> — Devuelve un juego específico"""
    games = load_games()
    game = next((g for g in games if g['id'] == game_id), None)

    if not game:
        return jsonify({'error': f'Juego con id {game_id} no encontrado'}), 404

    return jsonify(game), 200

@app.route('/api/juegos', methods=['POST'])
def create_game():
    """POST /api/juegos — Crea un nuevo juego"""
    data = request.get_json()

    # Validación
    if not data:
        return jsonify({'error': 'No JSON data provided'}), 400

    required_fields = ['titulo', 'genero', 'anio', 'plataforma', 'rating']
    for field in required_fields:
        if field not in data:
            return jsonify({'error': f'Campo requerido: {field}'}), 400

    # Validar tipos y valores
    if not isinstance(data['titulo'], str) or not data['titulo'].strip():
        return jsonify({'error': 'El título debe ser un string no vacío'}), 400

    if not isinstance(data['rating'], (int, float)) or not (0 <= data['rating'] <= 10):
        return jsonify({'error': 'Rating debe estar entre 0 y 10'}), 400

    games = load_games()

    # Crear nuevo juego
    new_game = {
        'id': get_next_id(games),
        'titulo': data['titulo'].strip(),
        'genero': data['genero'].strip(),
        'anio': int(data['anio']),
        'plataforma': data['plataforma'].strip(),
        'rating': float(data['rating'])
    }

    games.append(new_game)
    save_games(games)

    return jsonify(new_game), 201

@app.route('/api/juegos/<int:game_id>', methods=['PUT'])
def update_game(game_id):
    """PUT /api/juegos/<id> — Actualiza un juego"""
    games = load_games()
    game = next((g for g in games if g['id'] == game_id), None)

    if not game:
        return jsonify({'error': f'Juego con id {game_id} no encontrado'}), 404

    data = request.get_json()
    if not data:
        return jsonify({'error': 'No JSON data provided'}), 400

    # Actualizar campos si se proporcionan
    if 'titulo' in data:
        if not data['titulo'].strip():
            return jsonify({'error': 'El título no puede estar vacío'}), 400
        game['titulo'] = data['titulo'].strip()

    if 'genero' in data:
        game['genero'] = data['genero'].strip()

    if 'anio' in data:
        game['anio'] = int(data['anio'])

    if 'plataforma' in data:
        game['plataforma'] = data['plataforma'].strip()

    if 'rating' in data:
        if not (0 <= data['rating'] <= 10):
            return jsonify({'error': 'Rating debe estar entre 0 y 10'}), 400
        game['rating'] = float(data['rating'])

    save_games(games)

    return jsonify(game), 200

@app.route('/api/juegos/<int:game_id>', methods=['DELETE'])
def delete_game(game_id):
    """DELETE /api/juegos/<id> — Elimina un juego"""
    games = load_games()
    game = next((g for g in games if g['id'] == game_id), None)

    if not game:
        return jsonify({'error': f'Juego con id {game_id} no encontrado'}), 404

    games = [g for g in games if g['id'] != game_id]
    save_games(games)

    return jsonify({'message': f'Juego {game_id} eliminado'}), 204

@app.errorhandler(404)
def not_found(error):
    return jsonify({'error': 'Recurso no encontrado'}), 404

@app.errorhandler(500)
def server_error(error):
    return jsonify({'error': 'Error interno del servidor'}), 500

# ============= MAIN =============

if __name__ == '__main__':
    # Crear juegos.json si no existe
    if not os.path.exists(DATA_FILE):
        initial_games = [
            {'id': 1, 'titulo': 'The Legend of Zelda', 'genero': 'aventura', 'anio': 1986, 'plataforma': 'NES', 'rating': 9.5},
            {'id': 2, 'titulo': 'Super Mario Bros', 'genero': 'plataforma', 'anio': 1985, 'plataforma': 'NES', 'rating': 9.0},
            {'id': 3, 'titulo': 'Pac-Man', 'genero': 'arcade', 'anio': 1980, 'plataforma': 'Arcade', 'rating': 8.5},
        ]
        save_games(initial_games)

    app.run(debug=True, port=5000)

# Mini API de Juegos en Flask

API REST completa para gestionar una colección de videojuegos clásicos.

## Instalación y Configuración

### 1. Crear y activar virtualenv

**En Linux/macOS:**
```bash
python -m venv venv
source venv/bin/activate
```

**En Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

### 2. Instalar dependencias

```bash
pip install -r requirements.txt
```

### 3. Ejecutar la API

```bash
python app.py
```

La API estará disponible en `http://localhost:5000`

## Endpoints Disponibles

### GET /api/juegos
Obtiene la lista completa de juegos.

**Respuesta (200):**
```json
[
  {
    "id": 1,
    "titulo": "The Legend of Zelda",
    "genero": "aventura",
    "anio": 1986,
    "plataforma": "NES",
    "rating": 9.5
  }
]
```

### GET /api/juegos/<id>
Obtiene un juego específico.

**Respuesta (200):**
```json
{
  "id": 1,
  "titulo": "The Legend of Zelda",
  "genero": "aventura",
  "anio": 1986,
  "plataforma": "NES",
  "rating": 9.5
}
```

### POST /api/juegos
Crea un nuevo juego.

**Request:**
```json
{
  "titulo": "Nuevo Juego",
  "genero": "aventura",
  "anio": 2024,
  "plataforma": "Nintendo Switch",
  "rating": 8.5
}
```

**Respuesta (201):** El juego creado con ID asignado

### PUT /api/juegos/<id>
Actualiza un juego existente.

**Request:**
```json
{
  "rating": 9.0
}
```

**Respuesta (200):** El juego actualizado

### DELETE /api/juegos/<id>
Elimina un juego.

**Respuesta (204):** Sin contenido (pero juego eliminado)
