📖 Conceptos clave

2.1 El array de messages y los 3 roles

Toda llamada a una API de chat usa un array de mensajes. Cada mensaje tiene un role que define quién habla:

const messages = [
  {
    "role": "system",
    "content": "Eres un experto en Python. Responde de forma clara y concisa."
  },
  {
    "role": "user",
    "content": "¿Cómo funciona una list comprehension?"
  },
  {
    "role": "assistant",
    "content": "Una list comprehension es una forma concisa de crear listas..."
  },
  {
    "role": "user",
    "content": "Dame un ejemplo con números pares"
  }
];
Role Propósito Ejemplo
system Define la personalidad e instrucciones del asistente «Eres un tutor de programación»
user Lo que dice el usuario en esta vuelta «Explica qué es una clase»
assistant Respuestas anteriores (contexto para la siguiente llamada) «Una clase es un plano para crear objetos...»
💡 El order importa: El modelo lee los mensajes de arriba abajo. Un system prompt al inicio da instrucciones globales; los mensajes user/assistant alternados forman la conversación.

2.2 Temperature (0.0 a 2.0)

Controla la creatividad y aleatoriedad de las respuestas.

Temperature = 0.0 → Determinista
Siempre la misma respuesta. Ideal para: código, cálculos, respuestas de hechos. 
// PREGUNTA: ¿Cuánto es 2 + 2?
// Temperature 0.0 → Siempre: "2 + 2 = 4"
Temperature = 0.7 → Balance creativo (DEFAULT)
Buen balance entre consistencia y variación. Ideal para: chatbots, análisis, explicaciones generales. 
// PREGUNTA: ¿Qué es la programación?
// Temperature 0.7 → Respuestas variadas pero coherentes
// Respuesta A: "Es el arte de crear instrucciones..."
// Respuesta B: "Es un proceso de comunicarse con máquinas..."
Temperature = 1.5+ → Muy creativo/aleatorio
Respuestas muy variadas, impredecibles. Ideal para: brainstorming, poesía, historias, ideación. 
// PREGUNTA: Escribe el primer párrafo de una novela de ciencia ficción
// Temperature 1.5 → Respuestas muy diferentes cada vez
// Respuesta A: "La nave plateada atravesaba..."
// Respuesta B: "En el año 3045, los androides descubrieron..."
// Respuesta C: "Las estrellas lloraban esa noche..."

Comparación lado a lado — Mismo prompt, diferentes temperatures: 

// PROMPT: "Completar la frase: 'La IA es...'"

// Temperature 0.0:
// "La IA es un campo de la informática que..."

// Temperature 0.7:
// "La IA es una tecnología que permite a las máquinas aprender patrones..."

// Temperature 1.5:
// "La IA es el pulso del futuro, la sinfonía digital que..."

2.3 max_tokens / max_completion_tokens

Limita la longitud máxima de la respuesta que genera el modelo.

  • ¿Qué es un token? Unidad de texto. Roughly: 1 token ≈ 4 caracteres en inglés, ~3 en español.
  • Relación con coste: Pagas por tokens usados. Más tokens = más caro.
  • Context window: Cada modelo tiene un máximo total. Ejemplo: gpt-4o-mini = 128K tokens.
Cálculo rápido: 
// Si quieres una respuesta corta (párrafo): max_tokens = 100-150
// Si quieres una respuesta media (ensayo): max_tokens = 300-500
// Si quieres una respuesta larga (reporte): max_tokens = 800+

// Coste (aproximado para gpt-4o-mini):
// 1M input tokens = $0.075
// 1M output tokens = $0.30
// Si generas 1000 tokens, cuesta ~$0.0003 (muy barato)
⚠️ Atención: Si pones max_tokens muy bajo, la respuesta se corta a mitad. Siempre deja margen. Para una respuesta "normal", min 100-150.

2.4 Otros parámetros útiles

top_p (nucleus sampling):
Filtra palabras poco probables. Valores bajos (0.1-0.3) = más enfocado; altos (0.9) = más creativo. Default 1.0.
frequency_penalty (0 a 2):
Evita repetir palabras. Más alto = menos repetición. Útil para brainstorming largo.
presence_penalty (0 a 2):
Incentiva mencionar temas nuevos. Más alto = más diversidad temática.
stop sequences:
Lista de strings donde parar de generar. Ejemplo: ["###", "Fin"]
response_format (JSON mode):
Fuerza respuestas en JSON válido. Súper útil para parsear datos automáticamente.

2.5 Modelos disponibles y cuándo usar cada uno

Modelo Capacidad Coste Cuándo usarlo
gpt-4o Máxima calidad $$ Tareas críticas, análisis complejos, producción
gpt-4o-mini Muy bueno + rápido $ 90% de casos: chatbots, análisis, generación
gpt-3.5-turbo Bueno (legacy) ¢ Proyectos antiguos. NO recomendado para nuevos
💡 Recomendación: Empieza con gpt-4o-mini. Es barato, rápido y suficientemente potente. Solo sube a gpt-4o si necesitas máxima calidad.

2.6 Ejemplos de código completo

Ejemplo 1: Chat simple con temperature ajustable

const response = await fetch('https://api.openai.com/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${API_KEY}`
  },
  body: JSON.stringify({
    model: 'gpt-4o-mini',
    messages: [
      {
        role: 'system',
        content: 'Eres un chef italiano experto. Responde siempre en español.'
      },
      {
        role: 'user',
        content: '¿Cuál es el secreto de una buena carbonara?'
      }
    ],
    temperature: 0.7,
    max_tokens: 200
  })
});

const data = await response.json();
console.log(data.choices[0].message.content);

Ejemplo 2: JSON mode para datos estructurados

const response = await fetch('https://api.openai.com/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${API_KEY}`
  },
  body: JSON.stringify({
    model: 'gpt-4o-mini',
    messages: [
      {
        role: 'user',
        content: 'Extrae los datos de este texto: "Juan tiene 25 años, trabaja como ingeniero y vive en Madrid"'
      }
    ],
    temperature: 0,
    max_tokens: 200,
    response_format: { type: 'json_object' }
  })
});

const data = await response.json();
// Respuesta parseada automáticamente en JSON válido
const person = JSON.parse(data.choices[0].message.content);
console.log(person.name, person.age, person.job, person.city);

💻 Práctica guiada

Tres ejercicios para dominar los parámetros. Realiza estos en orden.

Ejercicio 1: Laboratorio de temperature

Objetivo: Ver cómo temperature cambia el comportamiento del modelo.

Pasos:
  1. Usa un tool como OpenAI Playground o escribe un script.
  2. Usa el mismo prompt: "Escribe un párrafo sobre la belleza del atardecer"
  3. Llama 3 veces con temperature 0, 0.7 y 1.5
  4. Compara las 3 respuestas. Anota las diferencias.
Qué esperar:
  • Temperature 0: Respuesta científica, técnica, casi la misma cada vez
  • Temperature 0.7: Respuesta variada pero coherente
  • Temperature 1.5: Respuestas muy diferentes, poéticas, impredecibles

Ejercicio 2: System prompt designer

Objetivo: Crear 3 personalidades distintas de asistentes con system prompts diferentes.

Tarea: Para cada personalidad, diseña un system prompt y pruébalo con el mismo user prompt.
Personalidad 1: Profesor Formal 
system: "Eres un profesor universitario de informática. Responde siempre con rigor académico y referencias a conceptos fundamentales."

user: "¿Qué es un API?"

// Respuesta esperada: Definición formal, referencias a REST, ejemplos académicos
Personalidad 2: Amigo Casual 
system: "Eres un amigo programador casual. Hablas con emojis, bromas técnicas y explicaciones simples."

user: "¿Qué es un API?"

// Respuesta esperada: Explicación casual, emojis, analogía con la vida real
Personalidad 3: Consultor de Negocios 
system: "Eres un consultor de tecnología empresarial. Responde enfocándote en ROI, casos de uso y valor de negocio."

user: "¿Qué es un API?"

// Respuesta esperada: Valor de negocio, casos de uso, impacto económico
Reflexión: ¿Cómo cambia la respuesta según el role? ¿Cuál es más útil para tu caso de uso?

Ejercicio 3: JSON mode

Objetivo: Obtener respuestas estructuradas en JSON que puedas parsear automáticamente.

Tarea: Escribe un prompt que pida respuestas en JSON y usa response_format. 
const response = await fetch('https://api.openai.com/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${API_KEY}`
  },
  body: JSON.stringify({
    model: 'gpt-4o-mini',
    messages: [
      {
        role: 'user',
        content: `Analiza esta película: "Dune (2021) es una epopeya de ciencia ficción dirigida por Denis Villeneuve."

        Responde en JSON con estos campos:
        - titulo (string)
        - genero (string)
        - director (string)
        - año (number)
        - description (string corta)`
      }
    ],
    temperature: 0,
    max_tokens: 200,
    response_format: { type: 'json_object' }
  })
});

const data = await response.json();
const movie = JSON.parse(data.choices[0].message.content);

// Ahora puedes acceder a movie.titulo, movie.genero, etc.
console.log(`Película: ${movie.titulo} (${movie.año}), director: ${movie.director}`);
¿Por qué JSON mode? Porque garantiza que la respuesta es JSON válido. Sin él, el modelo podría devolver "aquí está el JSON:" seguido de un bloque, lo que rompe el parsing.

🎯 Mini-reto del día: El generador de historias

Crea un script que genere historias cortas con parámetros ajustables.

Especificación

Tu script debe aceptar estos parámetros y ajustar temperature y max_tokens:

  • genre: "ciencia ficción", "romance", "thriller"
  • length: "corta" (100 tokens), "media" (250 tokens), "larga" (500 tokens)
  • creativity: "baja" (temp 0.3), "media" (temp 0.7), "alta" (temp 1.5)
async function generateStory(genre, length, creativity) {
  // Mapea los parámetros
  const tokenMap = { corta: 100, media: 250, larga: 500 };
  const tempMap = { baja: 0.3, media: 0.7, alta: 1.5 };

  const maxTokens = tokenMap[length];
  const temperature = tempMap[creativity];

  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${API_KEY}`
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: [
        {
          role: 'system',
          content: 'Eres un escritor de historias cortas. Crea narrativas cautivadoras con personajes vividos.'
        },
        {
          role: 'user',
          content: `Escribe una historia de ${genre} en español. Debe ser una historia completa con inicio, desarrollo y desenlace.`
        }
      ],
      temperature: temperature,
      max_tokens: maxTokens
    })
  });

  const data = await response.json();
  return data.choices[0].message.content;
}

// Uso:
const story = await generateStory('ciencia ficción', 'media', 'alta');
console.log(story);
Bonificación: Añade un parámetro style que cambie el system prompt: "estilo Borges", "estilo Stephen King", "estilo juvenil".
