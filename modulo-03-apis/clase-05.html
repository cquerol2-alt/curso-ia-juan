📖 Parte A: El Problema — Por Qué No Basta con Texto Libre

A.1 — LLMs Devuelven Texto, Pero Necesitas JSON

Cuando llamas a un LLM (ChatGPT, Claude, Llama), obtienes texto sin estructura. Pero las aplicaciones necesitan datos estructurados (JSON, objetos de datos) para procesar automáticamente.

Ejemplo: Quieres extraer información de una factura en texto libre.

Problema 1: JSON Malformado

Prompt: "Extrae los datos de esta factura en JSON:
Invoice #001, Amount: €45.99, Items: 3 widgets"

Respuesta del modelo:
"Aquí está el JSON:
{
  "invoice_number": "001"
  "amount": 45.99,
  "items": 3
}

¡Error de sintaxis! Falta una coma tras "001""

Problema 2: Llaves Inesperadas o Faltantes

Esperas:
{
  "invoice_number": "001",
  "amount": 45.99,
  "items": 3
}

Pero el modelo devuelve:
{
  "factura_numero": "001",       ← Nombre diferente
  "precio": 45.99,               ← Nombre diferente
  "cantidad_articulos": 3,       ← Nombre diferente
  "nota_adicional": "..."        ← Campo no esperado
}

Problema 3: Texto Extra Alrededor del JSON

Respuesta del modelo:
"Perfecto, aquí está la información extraída en JSON:

{
  "invoice_number": "001",
  "amount": 45.99,
  "items": 3
}

Espero que te sea útil."

Tu parser JSON falla porque hay texto antes y después.
⚠️ El Resultado:
  • Tu código se rompe porque el JSON no es válido
  • Tienes que parsear manualmente, filtrar datos, validar tipos
  • Pierdes tiempo en "limpiar" la respuesta del modelo
  • Sin validación, datos incorrectos entran en tu base de datos

A.2 — Casos Reales Donde Falla

  • Extracción de datos: Convertir facturas, contratos, o recibos en JSON estructurado
  • Análisis de sentimientos: Clasificar un review en rating, pros, cons, etc.
  • Extracción de entidades: De un párrafo, extraer nombres, fechas, dinero
  • Búsqueda y filtrado: Un usuario describe lo que quiere, el LLM lo convierte en una consulta SQL
  • Validación de datos: Asegurar que las respuestas cumplen con tipos, rangos, y restricciones

📖 Parte B: Pydantic — Tu Esquema de Datos en Python

B.1 — ¿Qué es Pydantic?

Pydantic es una librería Python que valida datos usando modelos tipados. Define la estructura esperada de tus datos (esquema) y Pydantic se asegura de que los datos cumplen con esa estructura.

La idea es simple: defines una clase, Pydantic valida los datos contra esa clase.

Instalación:

pip install pydantic

B.2 — Definir un Modelo Pydantic

Aquí está lo fundamental: un modelo Pydantic es una clase que hereda de BaseModel.

from pydantic import BaseModel, Field
from typing import List
from datetime import date

# Definir el modelo
class Invoice(BaseModel):
    invoice_number: str
    amount: float
    items: int
    due_date: date = None  # Campo opcional
    description: str = Field(..., description="Descripción de la factura")

# Usar el modelo
invoice_data = {
    "invoice_number": "001",
    "amount": 45.99,
    "items": 3,
    "due_date": "2026-04-30"
}

invoice = Invoice(**invoice_data)
print(invoice.invoice_number)  # Output: 001
print(invoice.amount)           # Output: 45.99
print(invoice.model_dump_json()) # Output: JSON válido

Qué Hace Pydantic Automáticamente:

  • ✅ Valida que invoice_number es string
  • ✅ Valida que amount es float
  • ✅ Valida que items es int
  • ✅ Convierte "2026-04-30" (string) a objeto date
  • ✅ Levanta error si faltan campos obligatorios
  • ✅ Rechaza campos extra inesperados

B.3 — Validación Avanzada: Restricciones y Valores por Defecto

from pydantic import BaseModel, Field
from typing import List, Optional

class MovieReview(BaseModel):
    title: str = Field(..., min_length=1, max_length=100)
    rating: int = Field(..., ge=1, le=10)  # ge=greater/equal, le=less/equal
    sentiment: str = Field(..., pattern="^(positive|neutral|negative)$")
    pros: List[str] = Field(default_factory=list)
    cons: List[str] = Field(default_factory=list)

# Válido
review = MovieReview(
    title="Inception",
    rating=9,
    sentiment="positive",
    pros=["Trama compleja", "Actores geniales"],
    cons=["Confusa al principio"]
)
print(review)

# INVÁLIDO — Pydantic levanta error
try:
    bad_review = MovieReview(
        title="",  # ❌ min_length=1
        rating=11,  # ❌ rating > 10
        sentiment="awesome"  # ❌ no coincide pattern
    )
except Exception as e:
    print(f"Error de validación: {e}")
💡 Field(): Especifica restricciones detalladas: min_length, max_length, ge (>=), le (<=), pattern (regex), description, etc.

B.4 — Convertir a JSON y Leer desde JSON

from pydantic import BaseModel
import json

class Invoice(BaseModel):
    invoice_number: str
    amount: float

# Crear objeto
invoice = Invoice(invoice_number="001", amount=45.99)

# Convertir a JSON
json_string = invoice.model_dump_json()
print(json_string)
# Output: {"invoice_number":"001","amount":45.99}

# Convertir a diccionario
dict_data = invoice.model_dump()
print(dict_data)
# Output: {'invoice_number': '001', 'amount': 45.99}

# Leer desde JSON
json_input = '{"invoice_number":"002","amount":99.99}'
invoice2 = Invoice.model_validate_json(json_input)
print(invoice2.invoice_number)  # Output: 002

📖 Parte C: Structured Outputs con OpenAI

C.1 — JSON Schema: El Esquema Que Controla el Modelo

OpenAI permite pasar un JSON Schema al modelo. El modelo es FORZADO a devolver datos que coincidan exactamente con ese esquema.

El flujo es:

  1. Defines un modelo Pydantic
  2. Extraes su JSON Schema
  3. Se lo pasas a la API con response_format
  4. El modelo responde con JSON válido que coincide
  5. Parseamos y usamos directamente (sin validación manual)

C.2 — Código Completo: Structured Outputs en OpenAI

from pydantic import BaseModel, Field
from openai import OpenAI
import json

# 1. Definir modelo Pydantic
class Invoice(BaseModel):
    invoice_number: str = Field(..., description="Número de factura")
    amount: float = Field(..., description="Monto total en EUR")
    items: int = Field(..., description="Cantidad de artículos")
    due_date: str = Field(..., description="Fecha de vencimiento")

# 2. Crear cliente OpenAI
client = OpenAI(api_key="sk-...")

# 3. Extraer schema de Pydantic
schema = Invoice.model_json_schema()

# 4. Llamar a OpenAI con response_format
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {
            "role": "user",
            "content": """Extrae los datos de esta factura en JSON:

Factura #ABC123
Monto: €199.99
Artículos: 5
Vence: 30 de junio de 2026"""
        }
    ],
    response_format={
        "type": "json_schema",
        "json_schema": {
            "name": "Invoice",
            "schema": schema
        }
    }
)

# 5. Parsear respuesta
json_str = response.choices[0].message.content
invoice = Invoice.model_validate_json(json_str)

print(f"Número: {invoice.invoice_number}")
print(f"Monto: {invoice.amount}")
print(f"Fecha vencimiento: {invoice.due_date}")
⚠️ Importante:
  • response_format es un parámetro de OpenAI que FUERZA al modelo a devolver JSON válido
  • El modelo NO puede devolver texto libre — SOLO JSON que coincide con el schema
  • Esto elimina 90% de los errores de parsing

C.3 — Diferencias: OpenAI vs Anthropic (Claude)

OpenAI tiene response_format con JSON schema. Anthropic no tiene este parámetro nativo, pero podemos lograr lo mismo con Instructor.

Aspecto OpenAI Anthropic
Structured Output response_format (nativo) Instructor (librería)
JSON Schema Obligatorio Sí, fuerza el modelo Sí, con Instructor
Validación Automática en API Automática con Instructor

📖 Parte D: Instructor — Simplifica Todo

D.1 — ¿Qué es Instructor?

Instructor es una librería que envuelve cualquier API de LLM (OpenAI, Anthropic, Llama, etc.) y la combina con Pydantic para devolver objetos validados directamente.

En vez de parsear JSON manualmente, Instructor se encarga de TODO.

Instalación:

pip install instructor

D.2 — Instructor + OpenAI

import instructor
from openai import OpenAI
from pydantic import BaseModel

# Envolver el cliente OpenAI con Instructor
client = instructor.from_openai(OpenAI())

class Invoice(BaseModel):
    invoice_number: str
    amount: float
    items: int

# Hacer llamada — Instructor valida automáticamente
invoice = client.chat.completions.create(
    model="gpt-4o-mini",
    response_model=Invoice,  # ← Especificar modelo Pydantic
    messages=[
        {
            "role": "user",
            "content": """Extrae los datos en JSON:
Factura #ABC123, Monto: €199.99, Artículos: 5"""
        }
    ]
)

# ¡LISTO! invoice es un objeto Invoice válido
print(f"Número: {invoice.invoice_number}")
print(f"Monto: {invoice.amount}")
print(type(invoice))  # Output:
🎯 Lo Mágico: Instructor convierte automáticamente el modelo Pydantic a JSON schema, se lo pasa a OpenAI, valida la respuesta, y te devuelve un objeto Python listo para usar.

D.3 — Instructor + Claude (Anthropic)

import instructor
from anthropic import Anthropic
from pydantic import BaseModel

# Envolver el cliente Anthropic con Instructor
client = instructor.from_anthropic(Anthropic())

class MovieReview(BaseModel):

Extrae: título, rating (1-10), sentimiento (positive/neutral/negative),
pros y cons."""
        }

# review es un objeto MovieReview completamente validado
print(f"Película: {review.title}")
print(f"Rating: {review.rating}")
print(f"Pros: {review.pros}")

D.4 — Instructor vs Llamadas Manuales

Con Instructor, el código es mucho más limpio:

SIN Instructor (manual, más código):

import json
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Extrae JSON..."}]
)

# Tienes que:
# 1. Extraer el texto
text = response.choices[0].message.content

# 2. Limpiar (si tiene prefijo/sufijo)
json_str = text.split("```json")[1].split("```")[0]  # Hack

# 3. Parsear
data = json.loads(json_str)

# 4. Validar manualmente cada campo
if not isinstance(data.get("amount"), (int, float)):
    raise ValueError("amount debe ser número")

# 5. Convertir a objeto
invoice = Invoice(**data)

CON Instructor (una línea):

import instructor
from openai import OpenAI
from pydantic import BaseModel

client = instructor.from_openai(OpenAI())

class Invoice(BaseModel):
    amount: float

# ¡Todo hecho automáticamente!
invoice = client.chat.completions.create(
    model="gpt-4o-mini",
    response_model=Invoice,
    messages=[{"role": "user", "content": "Extrae..."}]
)

Exercise 1: Construir un Invoice Parser (Code-Along)

Objetivo: Desde cero, definir un modelo Pydantic, validar datos, y extraer una factura de texto natural.

Pasos:

  1. Crea una carpeta clase-05-practica/
  2. Crea ex1_invoice_parser.py:
from pydantic import BaseModel, Field
from datetime import date

# Paso 1: Definir modelo
class Invoice(BaseModel):
    invoice_number: str = Field(..., description="Número de factura")
    amount: float = Field(..., gt=0, description="Monto en EUR")
    items: int = Field(..., ge=1, description="Cantidad de artículos")
    due_date: date = Field(..., description="Fecha de vencimiento")

# Paso 2: Crear datos de prueba (como si viniera del LLM)
raw_data = {
    "invoice_number": "FAC-2026-001",
    "amount": 250.50,
    "items": 5,
    "due_date": "2026-05-15"
}

# Paso 3: Validar con Pydantic
try:
    invoice = Invoice(**raw_data)
    print("✅ Factura válida:")
    print(f"  Número: {invoice.invoice_number}")
    print(f"  Monto: €{invoice.amount}")
    print(f"  Items: {invoice.items}")
    print(f"  Vence: {invoice.due_date}")
    print(f"\nJSON: {invoice.model_dump_json()}")
except Exception as e:
    print(f"❌ Error: {e}")

# Paso 4: Intentar datos inválidos
print("\n--- Intentando con datos inválidos ---")
bad_data = {
    "invoice_number": "FAC-001",
    "amount": -50,  # ❌ Negativo
    "items": 0,     # ❌ Menor que 1
    "due_date": "15/05/2026"  # ❌ Formato incorrecto
}

try:
    bad_invoice = Invoice(**bad_data)
except Exception as e:
    print(f"❌ Validación falla (como esperado): {type(e).__name__}")

Ejecuta: python ex1_invoice_parser.py

Verás cómo Pydantic rechaza datos inválidos automáticamente.

import instructor

class Invoice(BaseModel):

Exercise 3: Instructor + Claude — Lo Mismo, Pero Mejor

Objetivo: Usar Instructor con Claude en lugar de OpenAI.

Crea ex3_instructor_claude.py:

import instructor
from anthropic import Anthropic
from pydantic import BaseModel, Field
from typing import List

# Definir modelo más complejo
class MovieReview(BaseModel):
    title: str = Field(..., description="Título de la película", min_length=1)
    rating: int = Field(..., description="Rating 1-10", ge=1, le=10)
    sentiment: str = Field(..., description="positive/neutral/negative")
    pros: List[str] = Field(default_factory=list, description="Puntos positivos")
    cons: List[str] = Field(default_factory=list, description="Puntos negativos")

# Envolver Anthropic con Instructor
client = instructor.from_anthropic(Anthropic(api_key="sk-ant-..."))

# Reseña de ejemplo
review_text = """
'Dune: Part Two' es una continuación épica. La cinematografía es
impresionante y la dirección de Denis Villeneuve es magistral.
Los actores (Timothée Chalamet, Zendaya) entregan actuaciones sólidas.
Sin embargo, la trama puede sentirse lenta en algunos momentos,
y algunos fans del libro podrían sentir que se saltan detalles importantes.
"""

# Extraer datos
review = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=512,
    response_model=MovieReview,
    messages=[
        {
            "role": "user",
            "content": f"Analiza esta reseña:\n\n{review_text}\n\nExtrae: título, rating (1-10), sentimiento, pros y cons."
        }
    ]
)

# Imprimir resultados
print(f"Película: {review.title}")
print(f"Rating: {review.rating}/10")
print(f"Sentimiento: {review.sentiment}")
print(f"Pros:")
for pro in review.pros:
    print(f"  - {pro}")
print(f"Cons:")
for con in review.cons:
    print(f"  - {con}")

🎯 Mini-Reto del Día: Multi-Item Product Extractor

Challenge: Extrae Todos los Productos

Objetivo: Dado un párrafo con múltiples productos mencionados, extrae TODOS en una lista estructurada.

Especificación:

Cada producto debe tener:

  • name (nombre del producto)
  • price (precio en EUR, puede ser None si no se menciona)
  • quantity (cantidad, default 1)
  • category (categoría: electrónica, ropa, hogar, etc.)

Entrada de ejemplo:

"Compré un laptop por €899, 3 camisetas a €15 cada una,
y una lámpara LED sin especificar precio. También necesito
2 auriculares inalámbricos a €45."

Salida esperada: Una lista con 5 productos (laptop, camiseta x3, lámpara, auriculares x2) con estructura validada por Pydantic.

Entregable: Fichero reto_product_extractor.py que:

  1. Define un modelo Pydantic para Product
  2. Define un modelo para ProductList (contiene lista de productos)
  3. Usa Instructor + Claude (o OpenAI) para extraer
  4. Imprime los resultados en formato legible
💡 Pista de estructura: 
from pydantic import BaseModel
from typing import List, Optional

class Product(BaseModel):
    name: str
    price: Optional[float] = None
    quantity: int = 1
    category: str

class ProductList(BaseModel):
    products: List[Product]
