Part A: Streaming — Respuestas Palabra a Palabra

¿Por qué importa el Streaming?

Imagina que haces una pregunta a una IA y esperas 10 segundos mirando un spinner giratorio. Mala experiencia de usuario. Con streaming, las palabras empiezan a llegar inmediatamente—la respuesta se construye en tiempo real, como si un humano estuviera escribiendo.

En aplicaciones web, streaming es la diferencia entre una interfaz "muerta" y una que se siente viva.

¿Cómo Funciona el Streaming?

El servidor envía la respuesta en chunks (fragmentos) en lugar de todo de golpe:

  • Server-Sent Events (SSE): El servidor abre una conexión y envía datos incrementalmente al cliente.
  • Tokens/Deltas: Cada chunk es un delta—un cambio incremental que se suma al anterior.
  • Connection abierta: La conexión se mantiene abierta hasta que se recibe el último token.

OpenAI Streaming: Código en Python

Para activar streaming en OpenAI, solo hay que pasar stream=True:

from openai import OpenAI client = OpenAI() response = client.chat.completions.create( model="gpt-4o-mini", messages=[{"role": "user", "content": "Explica qué es un token en 50 palabras"}], stream=True # ← Activa streaming ) # Itera sobre los chunks for chunk in response: if chunk.choices[0].delta.content: print(chunk.choices[0].delta.content, end="", flush=True) print() # Nueva línea al final

Streaming con Anthropic (Claude)

Claude también soporta streaming, con una API muy similar:

from anthropic import Anthropic client = Anthropic() with client.messages.stream( model="claude-sonnet-4-6", max_tokens=100, messages=[{"role": "user", "content": "¿Qué es un token?"}], ) as stream: for text in stream.text_stream: print(text, end="", flush=True) print()

Streaming en FastAPI (Intro Rápida)

Para exponer un endpoint streaming en tu API web:

from fastapi import FastAPI from fastapi.responses import StreamingResponse from openai import OpenAI app = FastAPI() client = OpenAI() @app.post("/stream-chat") async def stream_chat(prompt: str): def generate(): response = client.chat.completions.create( model="gpt-4o-mini", messages=[{"role": "user", "content": prompt}], stream=True ) for chunk in response: if chunk.choices[0].delta.content: yield chunk.choices[0].delta.content + "\n" return StreamingResponse(generate(), media_type="text/event-stream")
Tip: En producción, usa librerías como server-sent-events o WebSockets para mayor robustez. Por ahora, esto es suficiente para prototipado.

Part B: Tokens — La Moneda de los LLMs

¿Qué es un Token?

Un token es una unidad mínima de texto que el modelo procesa. Aproximadamente:

  • 1 token ≈ 4 caracteres
  • 1 token ≈ 0.75 palabras en inglés
  • En español: un poco menos (~0.6-0.7 palabras por token, porque las palabras son más largas)

Ejemplo: "Hola, ¿cómo estás?" = ~5-6 tokens (no 4 palabras).

Herramienta interactiva: Prueba cuántos tokens tiene tu texto en OpenAI Tokenizer. Copia tu prompt y mira exactamente cuántos tokens son.

Input Tokens vs Output Tokens

Ambos se cobran por separado:

  • Input tokens: Lo que envías al modelo (tu prompt + contexto).
  • Output tokens: Lo que el modelo genera (la respuesta).

Generalmente, los output tokens son más caros (el modelo "trabaja más" generándolos).

📥 Input

Tu pregunta + contexto

📤 Output

La respuesta generada

Context Window: El Límite de Memoria

El context window es la cantidad máxima de tokens que el modelo puede procesar de golpe:

  • GPT-4o: 128.000 tokens (~100.000 palabras)
  • Claude 3.5 Sonnet: 200.000 tokens (~150.000 palabras)
  • GPT-4o-mini: 128.000 tokens

Si intentas pasar más tokens de los que caben en el context window, el modelo rechazará la solicitud.

⚠️ Atención: Un fichero de 100 páginas ≈ 25.000-30.000 tokens. Si pasas 5 ficheros grandes + tu pregunta, fácilmente superarás el límite.

Contar Tokens con tiktoken (Python)

Antes de enviar un request, puedes estimar tokens:

import tiktoken # Carga el codificador para GPT-4o enc = tiktoken.encoding_for_model("gpt-4o") text = "Este es un ejemplo de texto para contar tokens." tokens = enc.encode(text) print(f"Texto: {text}") print(f"Tokens: {len(tokens)}") # Ej: 10 tokens # Útil para validar antes de hacer el request: if len(tokens) > 100000: print("❌ Demasiados tokens, el request fallará") else: print("✅ OK, puedes enviar el request")

Fórmula: Calcular el Coste de un Request

Coste Total = (Input_tokens / 1,000,000 × Input_price) + (Output_tokens / 1,000,000 × Output_price)

Ejemplo:

  • Prompt: "Analiza este código" = 500 tokens (input)
  • Respuesta: 200 tokens (output)
  • Modelo: GPT-4o-mini ($0.15 input, $0.60 output)
Coste = (500 / 1,000,000 × 0.15) + (200 / 1,000,000 × 0.60) = 0.000075 + 0.00012 = $0.000195 ≈ 0.02 centavos

Conclusión: Una llamada típica a GPT-4o-mini cuesta centavos. Pero si haces 10.000 llamadas al día, suman rápido.

La Regla de Oro: No Todos los Modelos Para Todo

✅ GPT-4o para tareas complejas (análisis de código, reasoning, creatividad).

✅ GPT-4o-mini para tareas simples (clasificación, extracción, resúmenes cortos).

✅ Claude Haiku para tareas ultra-rápidas (clasificación binaria, búsqueda).

❌ No hagas esto: Usar GPT-4o para clasificar un texto entre 3 categorías. Pagas 30× más de lo que deberías.

Part D: Optimización Práctica para Ahorrar Dinero

1. Prompts Cortos = Menos Tokens = Menos Dinero

Antes (ineficiente):

"Eres un experto en programación con 20 años de experiencia. Tu trabajo es analizar código Python y encontrar bugs. Por favor, analiza el siguiente código y dime qué está mal..."

Después (eficiente):

"Analiza este código Python y encuentra bugs."

Resultado: prompts cortos sin "fluff" ahorran tokens. El modelo entiende igual sin la preambuladita.

2. Caching: No Preguntar Dos Veces lo Mismo

Si tienes requests idénticos, cachéalos (guarda la respuesta) para no enviar a la API:

import hashlib from functools import lru_cache @lru_cache(maxsize=1000) def ask_gpt(prompt: str) -> str: # La respuesta se cachea automáticamente # Si llamas dos veces con el mismo prompt, # se devuelve del cache sin tocar la API response = client.chat.completions.create(...) return response.choices[0].message.content ask_gpt("¿Qué es un token?") # API call ask_gpt("¿Qué es un token?") # Desde cache ✅

3. Model Routing: La Inteligencia de Elegir el Modelo Correcto

Analiza la complejidad de la pregunta y elige el modelo más barato que la resuelve:

def smart_route(user_query: str) -> str: # Palabras clave que indican complejidad complex_keywords = ["analiza", "explain", "why", "debug", "algorithm"] is_complex = any(kw in user_query.lower() for kw in complex_keywords) if is_complex: model = "gpt-4o" # Para reasoning, análisis else: model = "gpt-4o-mini" # Para lo demás response = client.chat.completions.create( model=model, messages=[{"role": "user", "content": user_query}] ) return response.choices[0].message.content

Resultado: 90% de preguntas van a mini (barato), 10% a GPT-4o (más preciso).

4. Monitoreo con Dashboards de Uso

Revisa regularmente cuánto estás gastando:

  • OpenAI: Billing → Usage (actualiza cada hora)
  • Anthropic: Console → Usage

Busca spikes anormales. Si ves un pico de gasto, investiga qué lo causó (bug en un loop, alguien abusando de la API, etc.).

Ejercicio 1: Chatbot Streaming con Contador de Tokens (Code-Along)

Objetivo: Construir un chatbot terminal que hace streaming de respuestas y muestra el número total de tokens usados.

Pasos:

  1. Instala las librerías necesarias:
    pip install openai tiktoken
  2. Crea un archivo `chatbot_streaming.py`:
    import sys from openai import OpenAI import tiktoken client = OpenAI() enc = tiktoken.encoding_for_model("gpt-4o-mini") messages = [] def count_tokens(text: str) -> int: return len(enc.encode(text)) def chat_stream(user_input: str): messages.append({"role": "user", "content": user_input}) # Contar tokens de input input_tokens = sum( count_tokens(msg.get("content", "")) for msg in messages ) print("\n🤖 Asistente: ", end="", flush=True) full_response = "" response = client.chat.completions.create( model="gpt-4o-mini", messages=messages, stream=True ) for chunk in response: if chunk.choices[0].delta.content: text = chunk.choices[0].delta.content print(text, end="", flush=True) full_response += text print() # Nueva línea # Contar tokens de output output_tokens = count_tokens(full_response) # Mostrar resumen print(f"\n📊 Tokens - Input: {input_tokens}, Output: {output_tokens}, Total: {input_tokens + output_tokens}") # Guardar en historial messages.append({"role": "assistant", "content": full_response}) # Loop principal print("🚀 Chatbot Streaming | Escribe 'salir' para terminar\n") while True: user = input("👤 Tú: ").strip() if user.lower() in ["salir", "exit"]: print("Hasta luego!") break if user: chat_stream(user)
  3. Corre el script:
    python chatbot_streaming.py
  4. Prueba con varias preguntas y observa cómo:
    • La respuesta aparece palabra por palabra (streaming)
    • Se cuenta el total de tokens después de cada respuesta
Tip: Prueba preguntas de diferente longitud y mira cómo varían los tokens. Una pregunta de 2 palabras vs una de 20 costará muy diferente.

Ejercicio 3: Reto Extra — Smart Model Router

Objetivo: Crear un router que elige automáticamente entre GPT-4o-mini y GPT-4o basado en la complejidad de la pregunta.

Requisitos:

  1. Función `classify_query(query: str) -> str` que devuelve "simple" o "complex"
    • Busca palabras clave: "why", "how", "explain", "analyze", "debug", "design"
    • Si hay 2+ palabras clave, es "complex"
    • Si el prompt es > 100 caracteres, es "complex"
    • Sino, es "simple"
  2. Función `route_and_respond(query: str) -> dict` que:
    • Clasifica la query
    • Elige model ("gpt-4o-mini" para simple, "gpt-4o" para complex)
    • Hace el request (sin streaming para simplificar)
    • Retorna: respuesta, modelo usado, y coste estimado
  3. Prueba con 5 queries distintas y calcula:
    • Coste total si usaras GPT-4o para todas
    • Coste total con el router
    • Ahorro ($)

Ejemplo de salida:

Query 1: "Hola" Clasificación: simple Modelo: gpt-4o-mini Coste: $0.00000075 Query 2: "Why is quantum computing relevant for cryptography?" Clasificación: complex Modelo: gpt-4o Coste: $0.001234 --- Coste si todo fuera GPT-4o: $0.00987 Coste con router: $0.00345 Ahorro: $0.00642 (65% más barato) ✅
