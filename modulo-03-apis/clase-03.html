📖 Conceptos clave

a) El problema: los LLMs no tienen memoria

Los modelos de lenguaje no tienen estado persistente. Cada llamada a la API es completamente independiente.

⚠️ Importante: Si no le envías el historial completo a OpenAI, el modelo no recuerda nada de lo que hablaste antes.

Analogía JavaScript: Es como si cada request HTTP no tuviera cookies ni session storage. El servidor no sabe quién eres a menos que le lo digas en cada petición.

b) La solución: mantener un array de messages

La clave es guardar todo el historial y enviarlo completo en cada llamada:

  • conversation_history = [] (array que crece con cada turno)
  • Usuario escribe → Añadir mensaje a historial
  • Enviar TODO el historial a la API
  • Recibir respuesta → Añadir a historial
  • Repetir

Código completo: Chatbot básico con memoria

from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

# Historial de conversación
conversation = [
    {"role": "system", "content": "Eres un asistente amable y conciso."}
]

def chat(user_message):
    """Envía mensaje y mantiene memoria de conversación."""
    # Agregar mensaje del usuario al historial
    conversation.append({"role": "user", "content": user_message})

    # Enviar TODO el historial
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=conversation
    )

    # Extraer respuesta y guardarla
    assistant_message = response.choices[0].message.content
    conversation.append({"role": "assistant", "content": assistant_message})

    return assistant_message

# Loop principal
print("🤖 Chatbot listo. Escribe 'salir' para terminar.\n")
while True:
    user_input = input("Tú: ")
    if user_input.lower() == 'salir':
        break
    response = chat(user_input)
    print(f"\n🤖 Bot: {response}\n")

c) El problema: context window overflow

El historial crece indefinidamente. Eventualmente excede el context window del modelo y obtienes error.

Contexto window de GPT-4o Mini: ~128K tokens. Parece infinito, pero una conversación larga puede alcanzarlo.

Soluciones comunes:

  • Truncar mensajes antiguos (sliding window)
  • Resumir la conversación periódicamente
  • Usar modelos más pequeños con ventanas mayores

Sliding Window: Mantener solo los últimos N mensajes

MAX_MESSAGES = 20  # Guardar últimos 20 mensajes + system prompt

def chat_with_limit(user_message):
    """Chat con límite de historial."""
    conversation.append({"role": "user", "content": user_message})

    # Enviar: system prompt + últimos N mensajes
    messages_to_send = [conversation[0]] + conversation[-MAX_MESSAGES:]

    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages_to_send
    )

    assistant_message = response.choices[0].message.content
    conversation.append({"role": "assistant", "content": assistant_message})

    return assistant_message

d) Streaming: Respuestas en tiempo real

Por defecto, OpenAI espera a generar la respuesta completa antes de devolverla. Con stream=True, obtienes tokens conforme se generan (como ChatGPT).

def chat_streaming(user_message):
    """Chat con streaming de respuesta."""
    conversation.append({"role": "user", "content": user_message})

    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=conversation,
        stream=True  # ← Activa streaming
    )

    assistant_message = ""
    print("\n🤖 Bot: ", end="", flush=True)

    # Iterar sobre chunks y imprimir conforme llegan
    for chunk in response:
        if chunk.choices[0].delta.content:
            text = chunk.choices[0].delta.content
            print(text, end="", flush=True)
            assistant_message += text

    print("\n")
    conversation.append({"role": "assistant", "content": assistant_message})
    return assistant_message

e) Manejo de errores

La API de OpenAI puede fallar por rate limits, timeouts, claves inválidas, etc. Siempre envuelve en try/except:

from openai import OpenAIError, RateLimitError

def chat_safe(user_message):
    """Chat con manejo robusto de errores."""
    try:
        conversation.append({"role": "user", "content": user_message})

        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=conversation
        )

        assistant_message = response.choices[0].message.content
        conversation.append({"role": "assistant", "content": assistant_message})
        return assistant_message

    except RateLimitError:
        return "⏳ Has alcanzado el límite de requests. Espera un momento."
    except OpenAIError as e:
        return f"❌ Error de API: {str(e)}"
    except Exception as e:
        return f"❌ Error inesperado: {str(e)}"

💻 Práctica guiada: 3 ejercicios

Ejercicio 1: Chatbot básico con memoria

🎯 Objetivo: Implementa el loop completo con historial funcionando.

Pasos:

  1. Copia el código completo de "Chatbot básico" (sección b arriba)
  2. Crea un archivo chatbot_simple.py en tu proyecto
  3. Ejecuta: python chatbot_simple.py
  4. Mantén una conversación de al menos 5 turnos. Verifica que el bot recuerda lo anterior.
  5. Comprobación: Pregunta "¿Qué dijiste hace poco?" y el bot debería ser capaz de responder.

Ejercicio 2: Añadir streaming

🎯 Objetivo: Modifica el chatbot para que las respuestas aparezcan palabra a palabra.

Pasos:

  1. Usa el código de "Streaming" (sección d)
  2. Reemplaza la función chat() por chat_streaming() en el loop principal
  3. Ejecuta y observa cómo la respuesta aparece en tiempo real en lugar de de golpe
  4. Desafío extra: Agrega una barra de "escribiendo..." mientras el modelo genera la respuesta

Ejercicio 3: Chatbot con personalidad + sliding window

🎯 Objetivo: Combina system prompt creativo + límite de historial.

Pasos:

  1. Crea un archivo chef_bot.py
  2. Usa un system prompt específico: "Eres un chef experto en cocina española. Hablas con pasión sobre recetas, ingredientes y técnicas. Siempre das recomendaciones prácticas."
  3. Incorpora el sliding window (MAX_MESSAGES = 20)
  4. Mantén una conversación sobre recetas españolas
  5. Comprobación: Genera una conversación de 30+ turnos. Debería funcionar sin error, y al final, el bot no debería recor dar los primeros mensajes (solo los últimos 20)

🎯 Mini-reto del día: "El asistente de cocina"

Construye un chatbot que:

  • Recuerda las preferencias alimentarias del usuario a lo largo de la conversación
  • Sugiere recetas basándose en conversaciones previas (dietario, alergias, sabores favoritos)
  • Mantiene un tono amable y profesional
  • Usa sliding window para evitar overflow

Bonus: Guarda la conversación en un archivo JSON al finalizar para poder "recordar" al usuario en futuras sesiones.
