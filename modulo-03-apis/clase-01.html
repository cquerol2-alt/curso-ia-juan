📖 Resumen de conceptos

a) ¿Qué es una API y por qué importa?

API = Application Programming Interface. Es la forma en que dos programas se comunican entre sí. Piensa en una API como un "menú de restaurante":

  • El restaurante (servidor) ofrece opciones (endpoints)
  • Tú (cliente) haces un pedido (request)
  • El camarero trae la comida (response)

REST API: Es el estándar actual. Usa HTTP (el protocolo web) para comunicarse. Cada acción se mapea a verbos HTTP:

  • GET: Obtener datos (lectura)
  • POST: Crear datos o enviar información (lo usaremos en OpenAI)
  • PUT: Actualizar datos
  • DELETE: Borrar datos

JSON: Formato de datos universal. Ejemplo:

{
  "model": "gpt-4o-mini",
  "messages": [
    {"role": "user", "content": "¿Hola, quién eres?"}
  ]
}

b) OpenAI Platform: Crear cuenta, API key y billing

Paso 1: Crear cuenta en OpenAI

  1. Ve a https://platform.openai.com/
  2. Haz clic en "Sign up"
  3. Usa tu email personal (no corporativo recomendado para desarrollo)
  4. Confirma tu email

Paso 2: Generar API Key

  1. Una vez logueado, ve a API Keys
  2. Haz clic en "+ Create new secret key"
  3. Copia y guarda en un lugar seguro (nunca la compartas)
  4. Verás algo como: sk-proj-xxxxxxxxxxxxx

Paso 3: Configurar billing y límite de gasto

  1. Ve a Billing > Overview
  2. Añade un método de pago (tarjeta de crédito)
  3. Ve a "Usage limits" (Límites de uso)
  4. Configura un hard limit (ej: 5€/mes) para evitar sorpresas
⚠️ Seguridad crítica: Tu API key es como una contraseña. Quien la tenga puede hacer llamadas a la API (y gastar dinero). NUNCA la hardcodees en tu código. NUNCA la publiques en GitHub.

c) Instalación: Python, pip y librerías

Requisitos previos:

  • Python 3.8+ instalado
  • pip (gestor de paquetes, viene con Python)
  • Un editor de texto (VS Code recomendado)

Instalar librerías necesarias:

pip install openai python-dotenv

¿Qué hace cada una?

  • openai: Cliente oficial de OpenAI. Simplifica las llamadas HTTP a la API
  • python-dotenv: Lee variables de entorno desde un archivo .env

d) Anatomía de una llamada a ChatCompletions

Estructura básica en Python:

from openai import OpenAI
from dotenv import load_dotenv
import os

# Cargar variables de entorno
load_dotenv()

# Crear cliente (lee OPENAI_API_KEY automáticamente del .env)
client = OpenAI()

# Hacer una llamada
response = client.chat.completions.create(
    model="gpt-4o-mini",  # Modelo económico
    messages=[
        {"role": "system", "content": "Eres un asistente útil."},
        {"role": "user", "content": "¿Qué es una API?"}
    ]
)

# Obtener la respuesta
print(response.choices[0].message.content)

Parámetros importantes:

  • model: Qué modelo usar. "gpt-4o-mini" es rápido y barato. "gpt-4o" es más potente pero más caro
  • messages: Array de mensajes con roles (system, user, assistant)
  • temperature (opcional): 0-2. 0 = determinista, 1 = equilibrio, 2 = muy creativo
  • max_tokens (opcional): Limita la longitud de la respuesta

e) Estructura de la respuesta

La respuesta es un objeto con esta estructura:

response = {
    "id": "chatcmpl-8bfMqxxx",
    "object": "chat.completion",
    "created": 1699564800,
    "model": "gpt-4o-mini",
    "choices": [
        {
            "index": 0,
            "message": {
                "role": "assistant",
                "content": "Una API es..."
            },
            "finish_reason": "stop"
        }
    ],
    "usage": {
        "prompt_tokens": 15,
        "completion_tokens": 120,
        "total_tokens": 135
    }
}

# Para obtener el texto:
texto = response.choices[0].message.content  # "Una API es..."

# Para ver uso de tokens (importante para el billing):
print(response.usage.total_tokens)  # 135 tokens
💡 Tokens = dinero. Las llamadas se facturam por tokens (aproximadamente 1 token = 4 caracteres). Cuanto más larga la respuesta, más caro.

f) Variables de entorno: .env y python-dotenv

Nunca hagas esto (INSEGURO):

# ❌ MAL: Hardcodeada, si subes a GitHub todos pueden verla
api_key = "sk-proj-xxxxxxxxxxxxx"
client = OpenAI(api_key=api_key)

Haz esto (SEGURO):

1. Crea un archivo .env en la raíz del proyecto:

# .env
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxx
OPENAI_MODEL=gpt-4o-mini

2. Configura .gitignore para que NO suba a GitHub:

# .gitignore
.env
*.pyc
__pycache__/

3. En tu código Python, carga las variables:

from dotenv import load_dotenv
import os

# Cargar desde .env
load_dotenv()

# Acceder a las variables
api_key = os.getenv("OPENAI_API_KEY")
model = os.getenv("OPENAI_MODEL", "gpt-4o-mini")  # Valor por defecto si no existe

# Crear cliente
client = OpenAI(api_key=api_key)
✓ Ventajas:
  • API key nunca en el repositorio
  • Fácil cambiar entre desarrollo y producción
  • Compatible con variables de entorno del servidor

💻 Práctica guiada

Ejercicio 1: Setup en OpenAI Platform (15 min)

Objetivo: Tener tu API key lista y el billing configurado.

  1. Crear cuenta en OpenAI Platform
    • Ve a https://platform.openai.com/
    • Haz clic en "Sign up" (si no tienes cuenta)
    • Completa el registro con tu email y contraseña
    • Confirma tu email
  2. Generar tu API key
    • Logueate en platform.openai.com
    • Ve a "API keys" en el menú lateral izquierdo
    • Haz clic en "+ Create new secret key"
    • Dale un nombre (ej: "desarrollo-juan")
    • Cópiala inmediatamente — no podrás verla de nuevo
    • Pégala en un archivo .env (lo haremos después)
  3. Configurar billing
    • Ve a Billing > Overview
    • Haz clic en "Add to a new card" o selecciona una existente
    • Añade una tarjeta de crédito/débito (se te cargará según uso, no hay suscripción)
    • Ve a "Usage limits" en el menú izquierdo
    • Configura un "Hard limit" (ej: 5€) para protegerte
    • Comprueba que el hard limit está activo
  4. Verificación: Captura pantalla de tu API key creada y tu límite de gasto configurado. Guarda como evidencia.

from openai import OpenAI
from dotenv import load_dotenv

# Cargar .env
load_dotenv()

# Crear cliente
client = OpenAI()

# Hacer una llamada
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {
            "role": "system",
            "content": "Eres un asistente útil que responde en español."
        },
        {

            "content": "¿Qué es una API REST?"

Ejercicio 3: Input interactivo (15 min)

Objetivo: Hacer un script que permita escribir preguntas y recibir respuestas.

Crea un archivo chat.py:

from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

# Guardar historial de conversación
messages = [
    {
        "role": "system",
        "content": "Eres un asistente útil. Responde siempre en español."
    }
]

print("=== Chat con OpenAI ===")
print("Escribe 'salir' para terminar\n")

while True:
    # Entrada del usuario
    user_input = input("Tú: ").strip()

    if user_input.lower() == "salir":
        print("¡Hasta luego!")
        break

    if not user_input:
        continue

    # Añadir mensaje del usuario al historial
    messages.append({"role": "user", "content": user_input})

    # Llamar a la API
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages
    )

    # Obtener respuesta
    assistant_message = response.choices[0].message.content

    # Guardar respuesta en historial
    messages.append({"role": "assistant", "content": assistant_message})

    # Mostrar respuesta
    print(f"ChatGPT: {assistant_message}\n")

Ejecutar:

python chat.py

Ahora puedes tener una conversación real. Nota cómo el historial se mantiene de un mensaje a otro.

🎯 Mini-reto del día: "El traductor instantáneo"

Objetivo: Crear un traductor automático entre idiomas usando la API.

Requisitos:

  • El script debe pedir un texto y un idioma destino
  • Usar la API de OpenAI para traducir
  • Mostrar tanto el texto original como la traducción

Esquema:

from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

# Pedir entrada
texto = input("Escribe el texto a traducir: ").strip()
idioma = input("¿A qué idioma quieres traducir? (inglés, francés, alemán, japonés...): ").strip()

# Llamada a API con instrucción clara
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {
            "role": "system",
            "content": f"Eres un traductor profesional. Traduce el siguiente texto SOLO al {idioma}. No añadas explicaciones, solo el texto traducido."
        },
        {
            "role": "user",
            "content": texto
        }
    ]
)

traduccion = response.choices[0].message.content

print(f"\n📝 Original: {texto}")
print(f"🌍 {idioma.capitalize()}: {traduccion}")

Bonus: Añade un contador de caracteres y palabras antes y después de la traducción. ¿Cambia mucho el tamaño según el idioma?

Consejo: Experimenta con diferentes idiomas. Algunos (chino, árabe) cambian mucho el número de caracteres. ¿Por qué crees que pasa?
