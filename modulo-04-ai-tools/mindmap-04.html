🔴🟢 TDD con IA ▼
Test-Driven Development para generación de código
Ciclo Red-Green-Refactor:

1. RED: Escribes un test que falla (la función no existe)

2. GREEN: Generas el código mínimo para que pase

3. REFACTOR: Mejoras manteniendo los tests en verde

Por qué TDD + IA es poderoso:

• El test es la especificación

• La IA sabe exactamente qué construir

• Menos refactoring manual después

• Confianza: todos los tests pasan = funciona

Ejemplo Python:
# RED: Test que falla def test_is_palindrome(): assert is_palindrome("aba") == True assert is_palindrome("abc") == False # GREEN: Copilot/Cursor completa def is_palindrome(s): return s == s[::-1] # REFACTOR: Optimizar # Cmd+K: "Mejora esto"
Ventajas:

✓ Menos bugs (validados por tests)

✓ Documentación viva (tests = casos de uso)

✓ Refactoring seguro (tests te avisan si rompiste algo)

Elementos clave:

• Input/Output: qué recibe y qué devuelve

• Tipos: use type hints (Python, TypeScript)

• Edge cases: vacío, null, negativos, duplicados

• Contexto: performance crítica, seguridad, etc.

// Normaliza números de teléfono a formato +34 // Entrada: mix de formatos (+34 123, 123456789, etc.) // Salida: array de strings en formato +34XXXXXXXXX // Ignora números inválidos sin error def normalize_phones(numbers):
Ejemplo 3:
// PERFORMANCE CRÍTICA: debe procesar 1M registros en < 100ms // Preferir arrays sobre objetos, evitar loops anidados // Resultado: lista de IDs sin duplicados, ordenada def filter_unique_ids(records):
Tips:

✓ Sé específico, no vago

✓ Menciona tecnología (Python, JavaScript, SQL, etc.)

✓ Si es crítico (seguridad/performance), avisa

🔨 Refactoring con IA ▼
Mejora código automáticamente con Copilot
Flujo estándar:

1. Selecciona código

2. Presiona Cmd+K (Mac) o Ctrl+K (Windows)

3. Escribe qué quieres: "Refactoriza para legibilidad"

4. Aplica cambios ✓

Patrones comunes:

Extract Function: separa lógica en funciones

Simplify Conditionals: if/if/if → switch/guard clauses

Add Error Handling: null checks, try/catch

Performance: O(n²) → O(n)

Comandos especiales:

/fix — Copilot detecta y arregla bugs

/doc — Genera docstrings y comentarios

/test — Genera tests unitarios

Ejemplo:
// Antes (desordenado) if (x > 0) { if (y > 0) { if (z > 0) { /* hacer algo */ } } } // Cmd+K: "Simplifica anidación" // Después (limpio) if (x <= 0 || y <= 0 || z <= 0) return; // hacer algo
Ventajas:

✓ Mantiene tests en verde

✓ Aplica patrones automáticamente

✓ Educa (ves cómo debería hacerse)

🐛 Debugging con IA ▼
Encuentra y arregla bugs rápidamente
Flujo estándar:

1. Copia el mensaje de error completo

2. Pégalo en Chat (Copilot, Claude, etc.)

3. Pregunta: "¿Qué está mal y cómo lo arreglo?"

4. La IA explica + solución

Con contexto (mejor):
"Aquí está el código: [código] Aquí el error: [error message] ¿Qué está mal?"
Variantes:

@terminal — Copilot analiza errores de terminal

@workspace — Copilot lee tu contexto de proyecto

Error profundo: "Tengo un loop infinito. ¿Dónde está?"

Ejemplo:
Error: TypeError undefined is not a function at processData (file.js:45) // Chat: "Pego error + código → IA:" // "En línea 45 llamas a .map() en 'data', pero // data no es un array. Verifica que sea array."
Tips:

✓ Copia el error completo (no resumen)

✓ Incluye dónde y cuándo ocurre

✓ Muestra el código relevante

👀 Code Review con IA ▼
Revisa código antes de hacer commit
Self-Review automático:

1. Selecciona tu código

2. Abre Chat de IA

3. Pide análisis estructura:

"Revisa este código para: • Seguridad (inyecciones, validaciones) • Rendimiento (loops, queries, O(n)) • Legibilidad (nombres, estructura) • Mantenibilidad (edge cases, errores)"
Elementos a revisar:

Seguridad: validar inputs, SQL injection, XSS

Performance: loops anidados, queries N+1

Legibilidad: nombres claros, funciones pequeñas

Mantenibilidad: DRY, SOLID, patrones

Flujo:

1. Code review antes de push

2. IA te da problemas detectados

3. Arreglas con Cmd+K

4. Haces commit limpio ✓

Ejemplo:
// IA detecta: ❌ "Loop O(n²) en línea 23" ❌ "No validas input en getUserData()" ❌ "Variable i no es descriptiva" // Arreglas: ✓ Optimizas con índice/hash ✓ Añades validación ✓ Renombras a itemIndex
Ventajas:

✓ Encuentra bugs antes de PR

✓ Educativo (aprendes mejores prácticas)

✓ PRs más limpios = reviews más rápidos

📄 Archivos de Contexto ▼
Enseña a la IA sobre tu proyecto
¿Qué son?

Archivos que dan contexto a agentes IA:

• .cursorrules → Cursor IDE

• CLAUDE.md → Claude Code

• AGENTS.md → Estándar genérico (nuevo)

Contenido típico:

Convenciones: snake_case, camelCase, tipos

Stack: Python 3.11, FastAPI, PostgreSQL

Estructura: src/, tests/, docs/

Estándares: tests antes de commit, type hints, docstrings

Ejemplo AGENTS.md:
# AGENTS.md - Contexto del Proyecto ## Convenciones - Variables: snake_case - Funciones: camelCase - Clases: PascalCase - Max 80 caracteres/línea ## Stack - Python 3.11 - FastAPI, SQLAlchemy - PostgreSQL ## Estándares - Type hints obligatorio - Docstrings para públicas - Tests antes de commit - No subir .env
Ventajas:

✓ La IA sigue tus convenciones automáticamente

✓ Código consistente en todo el equipo

✓ Menos ajustes manuales después

✓ Documentación viva de estándares

Dónde ponerlo:

Raíz del proyecto. La IA lo lee automáticamente.
