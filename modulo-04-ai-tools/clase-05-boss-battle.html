🎯 Boss Battle: Reconstruye tu Proyecto con AI Tools

📅 Semana 5 — Viernes | 💻 2 horas de proyecto | 🏆 50 XP

⚔️ El Reto

Has aprendido las 4 herramientas principales para codificar con IA: Claude Code, Cursor, GitHub Copilot y TDD con IA. Ahora es el momento de aplicar TODO ESTO a un proyecto real.

El objetivo: reconstruir un proyecto que ya hayas hecho (o crear uno nuevo) usando todas las herramientas y técnicas del módulo. El resultado debe ser:

  • Funcional: la app debe funcionar correctamente
  • Documentado: proceso de construcción con IA registrado
  • Con comparativa de tiempos: "Sin IA habría tardado X, con IA tardé Y"
  • En GitHub: repositorio público con código, README, y documentación
  • Profesional: código limpio, bien organizado, y listo para portfolio

⭐ Este proyecto va directo a tu CV y LinkedIn. Es una demostración de que puedes usar IA para acelerar desarrollo sin sacrificar calidad.

💡 Opciones de Proyecto

Elige UNA de estas opciones. El proyecto debe ser completo pero simple — mejor una app pequeña bien hecha que una grande a medias.

🌐 Red Social de Ideas (Recomendado)

Tu proyecto original del DAW: reconstruirlo con IA desde cero.

  • Usuarios pueden publicar "ideas" (título + descripción)
  • Feed de ideas con scroll infinito
  • Sistema de likes anónimo
  • Filtros por categoría (opcional)

Por qué: Es tu proyecto, lo conoces bien, y es lo suficientemente simple para 2 horas.

📝 Gestor de Tareas + Notas

Una app minimalista para organizar tareas diarias.

  • Crear, editar, eliminar tareas
  • Marcar como completadas
  • Persistencia en localStorage
  • Búsqueda y filtros básicos

Perfecto si prefieres algo sin backend.

🛒 Mini E-commerce

Un carrito de compra funcional (sin pagos reales).

  • Catálogo de 5-10 productos
  • Agregar/quitar del carrito
  • Cálculo de total y impuestos
  • Simulación de checkout

Frontend puro con localStorage.

🎬 Dashboard de Películas

Explora y guarda películas favoritas.

  • Integración con API pública (OMDb o TMDb)
  • Búsqueda de películas
  • Lista de favoritos (localStorage)
  • Detalles de película (rating, sinopsis, etc.)

Te enseña a integrar APIs externas con IA.

Consejo: No intentes hacer algo demasiado complejo. Una app sencilla bien hecha con IA vale más que una app compleja a medias. Apunta a 2 horas máximo.

📋 Plan de Ataque (Timeline de 2 horas)

Sigue este cronograma. Cada paso usa una herramienta diferente.

Paso Herramienta Tarea Tiempo
1 Claude Code Planificación + especificación del proyecto 20 min
2 Cursor Composer Generar estructura base + archivos principales 30 min
3 GitHub Copilot Completar funcionalidades, lógica, estilos 20 min
4 TDD + IA Tests con Cursor + validaciones 15 min
5 Claude Code Refactoring + documentación + README 20 min
6 Git + GitHub Push a GitHub, verificar que funciona en público 15 min
⚡ Regla de oro: Si una herramienta no funciona bien para algo, cambia a otra. Parte del aprendizaje es saber cuándo usar cada una.

🔨 Paso a Paso Detallado

✅ Paso 1: Planificación (20 min — Claude Code)

Objetivo: definir exactamente qué vas a construir.

Prompt para Claude Code:
"Voy a reconstruir [TU PROYECTO] con IA. Necesito:
1. Estructura de carpetas recomendada
2. Lista de librerías/dependencias
3. Descripción de 5-10 features principales
4. Plan paso a paso de desarrollo
5. Consideraciones de testing"

Entregable: Un plan de proyecto en texto que puedas seguir paso a paso.

✅ Paso 2: Estructura Base (30 min — Cursor Composer)

Objetivo: generar el esqueleto del proyecto.

Cómo hacerlo:
1. Abre Cursor
2. Crea una carpeta vacía para el proyecto
3. Abre Composer (Cmd+Shift+I en Mac, Ctrl+Shift+I en Windows)
4. Describe: "Crea la estructura de un [TU PROYECTO]. Incluye: [lista de features]"
5. Deja que Composer genere todos los archivos

Entregable: Carpeta con estructura HTML/CSS/JS (o Python) lista para editar.

✅ Paso 3: Completar Funcionalidades (20 min — GitHub Copilot)

Objetivo: llenar los huecos con lógica y estilos.

Cómo hacerlo:
1. Abre los archivos en VS Code (con Copilot activado)
2. Escribe el nombre de una función: function addItem() {
3. Copilot sugerirá. Presiona Tab para aceptar
4. Usa Cmd+K (Mac) o Ctrl+K (Windows) para refactorizar
5. Completa todos los métodos principales

Entregable: App funcional pero sin tests (yet).

✅ Paso 4: Testing (15 min — TDD + Cursor)

Objetivo: asegurar que funciona correctamente.

Cómo hacerlo:
1. Crea archivo test.js o test.py
2. Escribe en Cursor: "Genérame tests para la función addItem()"
3. Copila y ejecuta: npm test o pytest
4. Arregla cualquier fallo

Entregable: Test suite pasando al 100%.

✅ Paso 5: Refactoring + Documentación (20 min — Claude Code)

Objetivo: código limpio y bien documentado.

Cómo hacerlo:
1. Abre el código en Claude Code
2. Pide: "Refactoriza esto para que sea más limpio"
3. Pide: "Añade comentarios y docstrings"
4. Pide: "Crea un README.md profesional"

Entregable: README.md + código limpio con comentarios.

✅ Paso 6: GitHub + Deploy (15 min)

Objetivo: código en público, funcional, con URL.

Cómo hacerlo:
1. git init && git add . && git commit -m "Initial commit"
2. Crea repo en GitHub
3. Push: git push origin main
4. Si es frontend puro: usa GitHub Pages
5. Si es backend: despliega en Render/Railway

Entregable: URL pública funcional en README.

💡 Tips Finales

🎯 Alcance realista

No intentes hacer algo demasiado complejo. Una app sencilla bien hecha con IA vale más que una app compleja a medias. En 2 horas debes tener algo funcional y desplegado.

🔄 Cambiar de herramienta cuando sea necesario

Si Cursor no genera bien HTML, usa Claude Code. Si Copilot no entiende tu lógica, pide ayuda a Cursor. Saber cuándo cambiar es clave.

📝 Commits frecuentes

git commit -m "feat: agregar feature X" después de cada paso importante. Facilita revisar tu trabajo y muestra buen flujo de trabajo.

🧪 Tests desde el principio

No dejes los tests para el final. Escribe un test, haz que IA complete la función, repite. Es más rápido y confías en el código.

📱 Responsivo = bonus points

Si tu app funciona en mobile tanto como en desktop, suma puntos mentales. La mayoría de usuarios están en mobile.

🌟 "Portfolio piece" mindset

Piensa en esto como algo que mostrarás a futuros clientes o empleadores. ¿Parecería profesional? ¿Funciona sin errores? ¿El código es legible? Si la respuesta es no a cualquiera, arréglalo antes de hacer push.

🌟 Bonus: Características Avanzadas

Si terminas rápido y quieres más, implementa uno de estos:

🔐 Autenticación básica

Login/logout con localStorage. No es seguro, pero demuestra que entiendes sessions.

💾 Base de datos

Cambia localStorage por una BD real: MongoDB, PostgreSQL, etc. Usa IA para la integración.

⚡ Modo oscuro/claro

Toggle de tema con persistencia. Añade 10 líneas, ve cómo mejora UX.

📊 Analytics básicas

Cuenta clics, vistas, acciones. Visualízalos en una tabla. Enseña el valor del data.

🔍 Búsqueda avanzada

Filtros por múltiples criterios, ordenamiento, paginación.

📧 Email o notificaciones

Integra SendGrid o Twilio. Demuestra que puedes usar APIs externas.
