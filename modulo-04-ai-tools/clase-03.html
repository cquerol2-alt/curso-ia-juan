Clase 03: Claude Code y Windsurf — Agentes de Código

📅 Semana 5 — Miércoles | 🎬 30 min vídeo + 💻 65 min práctica + 🧪 15 min test

🅰️ Claude Code — Tu Pair Programmer en Terminal

📖 Resumen: ¿Qué es Claude Code?

Claude Code es la herramienta agentica de Anthropic para programadores. Se ejecuta en tu terminal y actúa como un senior developer pair-programming contigo. Lee tu codebase completa, edita archivos, ejecuta comandos, gestiona git, y todo desde línea de comandos con lenguaje natural.

🚀 Instalación

npm install -g @anthropic-ai/claude-code
cd tu-proyecto
claude

Requisitos:

  • Node.js 18 o superior
  • Anthropic API key o cuenta de Claude

⚙️ Cómo Funciona

  1. Escribes una orden en lenguaje natural (ej: "refactoriza este componente para que sea reutilizable")
  2. Claude Code lee los archivos relevantes de tu proyecto
  3. Propone cambios y te pregunta antes de aplicarlos
  4. Ejecuta tests, compila, hace commit si todo va bien
  5. Si hay errores, intenta arreglarlos automáticamente

🎯 Comandos Clave

  • Lenguaje natural: Solo escribe qué quieres. Claude Code entiende el contexto.
  • /help: Muestra los comandos disponibles
  • /clear: Reinicia la conversación
  • CLAUDE.md: Claude Code lee tu archivo de instrucciones del proyecto para entender contexto, convenciones, estructura

🔍 Cuándo Usar Cada Herramienta

Herramienta Mejor Para Ejemplo
GitHub Copilot Autocompletado mientras escribes Completa líneas, funciones simples
Cursor Edición visual multi-archivo (Composer) Refactorizar 3-4 archivos con control visual
Claude Code Tareas complejas desde terminal, entender codebase grande Refactorizar todo el proyecto, migrar versiones
Windsurf Agéntico + IDE visual (Cascade) Cambios complejos con ejecución automática
Pro Tip: Claude Code brilla cuando necesitas entender y cambiar una codebase entera. Lee CLAUDE.md para saber convenciones, estructura, y puede aplicar cambios consistentes en 50 archivos en un comando.

🅱️ Windsurf — IDE Agéntico Nativo

📖 Resumen: ¿Qué es Windsurf?

Windsurf es un IDE completo de Codeium basado en VS Code, pero con IA agentica nativa. Su estrella es Cascade, un agente que combina sugerencias inline (copilot) con ejecución automática de tareas complejas (agent).

🚀 Instalación

Descarga desde windsurf.ai (disponible para Windows, macOS, Linux).

🤖 Cascade: El Agente de Windsurf

  • Modo Copilot: Sugerencias inline mientras escribes (como Copilot clásico)
  • Modo Agente: Ejecuta tareas complejas paso a paso, propone cambios, los aplica si apruebas
  • Flows: Windsurf combina ambos modos automáticamente según lo que necesites

⚡ Cascade vs Cursor Composer

Aspecto Cursor Composer Windsurf Cascade
Interfaz Panel lateral con cambios Panel lateral con agente conversacional
Control Tú apruebas/rechazas cambios visualmente IA propone, ejecuta, itera si hay errores
Agentico No, manual Sí, automático por defecto
Mejor Para Cambios donde quieres ver cada paso Tareas complejas donde IA itera sola
Pro Tip: Windsurf Cascade es más "agéntico" por defecto. Si le pides "implementa un login con JWT", Cascade escribirá el código, ejecutará tests, e incluso intentará arreglarlo si hay errores. Cursor te da más control manual paso a paso.

💻 Práctica: 50 minutos

Ejercicio 1: Claude Code Setup (15 min)

  1. Instala Claude Code globalmente: npm install -g @anthropic-ai/claude-code
  2. Navega a un proyecto existente: cd tu-proyecto
  3. Inicia Claude Code: claude
  4. Pide a Claude Code que explique la estructura del proyecto: 
    "Explica la estructura de carpetas de este proyecto y qué hace cada módulo principal"
  5. Observa cómo lee los archivos, entiende el contexto, y te da un resumen completo.

Ejercicio 2: Claude Code - Feature Simple (15 min)

  1. En el mismo proyecto, pide a Claude Code que añada una feature: 
    "Añade validación de formularios al archivo login.html.
Debe validar: email (formato correcto), password (mínimo 8 caracteres).
Si hay error, muestra mensaje rojo debajo del input."
  2. Claude Code leerá el HTML, propondrá el código (probablemente JavaScript puro o usando una librería).
  3. Revisa su propuesta y aprueba (y) o rechaza (n).
  4. Anota: ¿entendió bien el contexto? ¿la solución es buena? ¿necesitó correcciones?

Ejercicio 3: Codebase-to-Course — Entiende cualquier repo con IA (15 min)

¿Qué es? Codebase-to-Course es un skill de Claude Code que convierte cualquier repositorio de código en un curso interactivo en HTML. Le pasas un repo y genera automáticamente un archivo con explicaciones línea a línea, diagramas de arquitectura animados, quizzes y un glosario de términos técnicos. Todo en un solo HTML que funciona offline.

¿Para qué sirve? Para aprender a leer y entender código ajeno — una de las habilidades más importantes como desarrollador. En vez de leer un repo a ciegas intentando descifrar qué hace cada archivo, esta herramienta te genera una guía visual que te lo explica paso a paso.

  1. Clona tu API Flask del Módulo 1 (o cualquier repo pequeño que te interese): 
    git clone https://github.com/tu-usuario/tu-proyecto-flask.git
cd tu-proyecto-flask
  2. Abre Claude Code en ese directorio y pide que use el skill: 
    claude
"Usa codebase-to-course para generar un curso interactivo de este proyecto"
  3. Claude Code analizará todo el código y generará un HTML interactivo con:
    • Explicaciones del código en lenguaje natural (línea a línea)
    • Diagramas de cómo se conectan los módulos
    • Un quiz para comprobar que has entendido la estructura
    • Glosario con tooltips de los términos técnicos
  4. Abre el HTML generado en el navegador y recórrelo
  5. Reflexión: ¿Entiendes mejor el código después de leer el curso generado? ¿Hay algo que no sabías que hacía tu propio proyecto?
Pro Tip: Esta herramienta es brutal para entender repos open source. En los Módulos 5 y 6 la puedes usar para entender por dentro librerías como LangChain o CrewAI antes de usarlas. Leer código ajeno es tan importante como escribir código propio.

Ejercicio 4: Windsurf Setup (10 min)

  1. Descarga e instala Windsurf desde windsurf.ai
  2. Abre tu proyecto en Windsurf (File > Open Folder)
  3. Abre el panel de Cascade (debería estar en la barra lateral)
  4. Pide algo simple: "¿Qué es el archivo package.json y qué contiene?"
  5. Observa la diferencia de UI con Claude Code (Windsurf es visual, Claude Code es terminal)

Ejercicio 5: El Mismo Cambio, 3 Herramientas (10 min)

  1. Elige un cambio concreto (ej: "añade un footer con copyright y año actual a index.html")
  2. Hazlo con Cursor Composer: Abre Cursor, llama a Composer, pide el cambio, aprueba visualmente
  3. Hazlo con Claude Code: Vuelve a terminal, claude, pide el cambio, aprueba
  4. Hazlo con Windsurf Cascade: Abre Windsurf, pide el cambio en Cascade, observa cómo lo ejecuta
  5. Anota en papel:
    • ¿Cuál fue más rápido?
    • ¿Cuál dio mejor resultado?
    • ¿Cuál preferiste y por qué?
    • ¿En qué situación usarías cada uno?

🎯 Mini-Reto: Dark Mode en 3 IDEs

Misión: Implementa dark mode en tu proyecto con las 3 herramientas y compara resultados.

Qué hacer:

  1. Cursor Composer: Pide "Implementa dark mode usando CSS variables. Añade toggle en navbar."
  2. Claude Code: Mismo comando en terminal
  3. Windsurf Cascade: Mismo comando en panel
  4. Análisis:
    • ¿Cuál escribió el código más limpio?
    • ¿Cuál ejecutó tests automáticamente?
    • ¿Cuál necesitó más iteraciones?
    • ¿Cuál es tu favorito y por qué?
