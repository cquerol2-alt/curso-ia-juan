Clase 04: Técnicas — TDD con IA, Refactoring y Prompts

📅 Semana 5 — Jueves | 🎬 30 min vídeo + 💻 50 min práctica + 🧪 15 min test

📖 Resumen de la clase

a) TDD con IA (Test-Driven Development)

Ciclo Red-Green-Refactor:

  • RED: Tú escribes un test que falla (porque la función no existe aún)
  • GREEN: Escribes el mínimo código para que el test pase (o dejas que la IA lo genere)
  • REFACTOR: Mejoras el código manteniendo los tests en verde

Por qué TDD + IA es una combinación poderosa:

  • El test es la especificación — no necesitas explicar qué debe hacer, el test lo dice
  • La IA sabe exactamente qué construir porque tiene un objetivo claro
  • Reduces refactoring manual: dejas que la IA genere y luego validas con los tests
  • Confianza: si todos los tests pasan, el código funciona como esperado

Ejemplo en Python:

# 1. RED: Tú escribes el test
def test_is_palindrome():
    assert is_palindrome("aba") == True
    assert is_palindrome("abc") == False
    assert is_palindrome("") == True
    assert is_palindrome("A") == True

# 2. GREEN: Copilot/Cursor genera la función
# Escribe en la siguiente línea: "def is_palindrome(s):"
# Opción 1: Déjalo que autocomplete
# Opción 2: Con Cmd+K: "Genera una función que verifique si s es un palíndromo (ignorando mayúsculas)"

# 3. REFACTOR: Pide a la IA que optimice
# Con Cmd+K: "Refactoriza esto para usar un one-liner"

b) Prompts en Comentarios

Regla de oro: Escribe comentarios detallados ANTES del código. Cuanto más específico, mejor la sugerencia.

Ejemplos:

// Función que recibe un array de objetos {nombre, edad, ciudad}
// y devuelve solo los mayores de 18 ordenados alfabéticamente por nombre
// Si no hay resultados, devuelve un array vacío

// Recibe una lista de strings con formatos de teléfono mixtos
// (+34 123 456 789, 123456789, (123) 456-7890)
// Normaliza a formato internacional +34 y devuelve array
// Ignora números inválidos sin lanzar error

// Esta función es performance-critical:
// debe procesar 1M de registros en menos de 100ms
// Prefiere arrays sobre objetos, evita loops anidados

Tips:

  • Usa anotaciones de tipo (type hints en Python/TypeScript)
  • Sé específico sobre edge cases (vacío, null, valores negativos)
  • Menciona el idioma/framework si es poco obvio
  • Si el código es crítico (seguridad, performance), dilo en el comentario

c) Refactoring con IA

Flujo estándar:

  • Selecciona el código → Presiona Cmd+K (Mac) o Ctrl+K (Windows) en Copilot/Cursor
  • Escribe: "Refactoriza esto para que sea más legible" o "Extrae una función para..." o "Simplifica estos condicionales"

Patrones comunes:

  • Extract Function: "Extrae la lógica de validación en una función separada"
  • Simplify Conditionals: "Reemplaza estos 5 ifs anidados con un switch o guard clauses"
  • Add Error Handling: "Añade manejo de errores para null checks y excepciones"
  • Performance: "Optimiza esto para O(n) en lugar de O(n²)"
  • /fix: Comando especial para arreglar bugs automáticamente
  • /doc: Genera documentación/docstrings

d) Debugging con IA

Flujo:

  1. Copia el mensaje de error completo → Pégalo en el Chat de Copilot/Claude
  2. Pregunta: "Explícame este error y cómo solucionarlo"
  3. La IA te da: qué está mal, por qué ocurre, y la solución paso a paso

Variantes:

  • @terminal: En Copilot, escribe @terminal para pedirle que analice errores de terminal
  • Con contexto: "Aquí está el código: [código]. Aquí está el error: [error]. ¿Qué está mal?"
  • Debugging profundo: "¿Por qué mi loop infinito no termina? Aquí está el código..."

e) Code Review con IA

Cómo hacer un self-review antes de commit:

// En el Chat de la IA, pega el código y escribe:
"Revisa este código y señala problemas de:
1. Seguridad (inyecciones, validaciones)
2. Rendimiento (loops, queries, complejidad)
3. Legibilidad (nombres, estructura, patrones)
4. Mantenibilidad (errores potenciales, edge cases)"

La IA te dará un análisis estructurado y sugerencias de mejora antes de que lo suban.

f) AGENTS.md y CLAUDE.md

¿Qué son? Archivos que dan contexto a la IA sobre tu proyecto:

  • .cursorrules: Reglas para Cursor IDE (formato estándar de Cursor)
  • CLAUDE.md: Instrucciones para Claude Code (formato usado por Anthropic)
  • AGENTS.md: Nuevo estándar, más genérico para cualquier agente IA

Ejemplo AGENTS.md:

# AGENTS.md - Contexto del proyecto

## Convenciones de código
- Variables en snake_case
- Funciones en camelCase
- Clases en PascalCase
- Máximo 80 caracteres por línea

## Estructura de carpetas
- src/ → código fuente
- tests/ → tests unitarios
- docs/ → documentación
- scripts/ → scripts de utilidad

## Estándares
- Todos los tests deben pasar antes de commit
- Usar type hints en todas las funciones
- Documentar funciones públicas con docstrings
- No subir .env o archivos con secretos

💻 Prácticas (50 minutos)

Ejercicio 1: TDD con Copilot (15 min)

Objetivo: Escribir tests primero, dejar que Copilot genere la implementación.

  1. Crea un archivo shopping_cart.py
  2. Escribe 5 test cases para una clase ShoppingCart:
    • Añadir un producto (aumenta total)
    • Remover un producto (disminuye total)
    • Aplicar descuento (reduce precio final)
    • Carrito vacío devuelve 0
    • Cantidad negativa no se permite (excepción)
  3. Escribe solo los tests. Ejecuta: pytest shopping_cart.py — todos fallan (RED)
  4. Abre Copilot/Cursor y escribe en comentario: "Clase que gestiona un carrito de compra con métodos add, remove, apply_discount"
  5. Deja que la IA genere la clase (GREEN)
  6. Ejecuta tests nuevamente — todos pasan
  7. Pide a la IA que refactorice: "Extrae la lógica de cálculo de total en una función private"

Ejercicio 2: Refactoring Challenge (15 min)

Código legible vs. Código legible:

Aquí hay código desordenado. Tu trabajo: refactorizar usando Cmd+K

def process_data(data):
    result = []
    for i in range(len(data)):
        if data[i]['status'] == 'active' and data[i]['age'] > 18:
            if data[i]['verified'] == True:
                temp = data[i]['salary'] * 1.1
                if temp > 50000:
                    result.append({
                        'name': data[i]['name'],
                        'new_salary': temp,
                        'bonus': temp * 0.15
                    })
    return result
  1. Copia este código en tu IDE
  2. Selecciona todo → Cmd+K → "Refactoriza: extrae funciones, simplifica condicionales, mejora nombres"
  3. Compara resultado con el original — ¿es más claro?
  4. Pide refactoring adicional: "Usa type hints y docstring"

Ejercicio 3: Debug con IA (10 min)

Introduce un bug deliberadamente:

def calculate_discount(price, discount_percent):
    return price * discount_percent  # BUG: debería ser (1 - discount_percent/100)

# Resultado: calculate_discount(100, 20) = 2000 en lugar de 80
  1. Copia el código con el bug
  2. Abre Claude/Copilot Chat y pega: "Aquí está el código, ejecuto calculate_discount(100, 20) y obtengo 2000 en lugar de 80. ¿Qué está mal?"
  3. La IA te explicará el bug y cómo solucionarlo
  4. Aplica la corrección y verifica que funcione

Ejercicio 4: Crear AGENTS.md (10 min)

Documenta tu proyecto para que la IA lo entienda:

  1. En la raíz de tu proyecto, crea AGENTS.md
  2. Incluye:
    • Nombre y descripción del proyecto
    • Convenciones de código (naming, estructura)
    • Stack tecnológico
    • Estructura de carpetas
    • Estándares de testing
    • Cualquier regla especial (no usar ciertos métodos, logging requerido, etc.)
  3. Sube el archivo a tu repo
  4. De ahora en adelante, cuando trabajes con IA, ésta tendrá contexto de tu proyecto

🎯 Mini-reto: TDD Speed Run (Opcional, 15 min)

¿Puedes hacer esto en 15 minutos?

  1. Escribe tests para 5 funciones:
    • validate_password(pwd) — mínimo 8 caracteres, 1 mayúscula, 1 número
    • calculate_discount(price, customer_type) — premium 20%, silver 10%, regular 0%
    • format_currency(amount) — $1,234.56 format
    • parse_date(date_str) — "2024-01-15" → datetime object
    • sort_by_multiple_criteria(list, criteria) — sort por edad DESC, luego nombre ASC
  2. Usa SOLO IA para generar las implementaciones
  3. Ejecuta los tests — ¿todos pasan?
  4. Si falla alguno, pide a la IA que lo arregle con /fix
  5. Tiempo final: ¿cuánto te tardó?
