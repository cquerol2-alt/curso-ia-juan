Clase 02: Cursor IDE — El Editor que Programa por Ti

📅 Semana 5 — Martes | 🎬 30 min vídeo + 💻 50 min práctica + 🧪 15 min test

🎬 Vídeo de la lección

Descubre Cursor IDE, un fork de VS Code con IA integrada nativa que puede generar código, refactorizar y crear features completas solo con descripciones en lenguaje natural.

📚 Documentación oficial: Consulta la guía de Cursor para profundizar en todas las características.

📖 Conceptos clave

2.1 ¿Qué es Cursor?

Cursor es un fork de VS Code con IA nativa. Mantiene todos los atajos, extensiones y temas de VS Code, pero añade herramientas de IA integradas de serie.

Aspecto VS Code + Copilot Cursor IDE
Editor base VS Code estándar Fork de VS Code
IA integrada Extensión (requiere suscripción) Nativa, de serie
Modelos disponibles Solo GPT-4, GPT-3.5 Claude (Sonnet), GPT-4, GPT-4o
Funciones IA Autocompletado + Chat Tab + Cmd+K + Chat + Composer
Free tier Limitado 2000 completions + 50 slow premium/mes
Precio pro $20/mes $20/mes (ilimitado)
💡 Ventaja clave: Cursor no es una extensión, es un editor completo. Eso significa la IA está optimizada en cada rincón del editor.

2.2 Instalación y configuración inicial

Paso 1: Descargar Cursor

Ve a cursor.com, descarga el instalador para tu SO (Windows, macOS, Linux).

Paso 2: Importar configuración de VS Code

Al abrir Cursor por primera vez, te pregunta si quieres importar la configuración de VS Code. Di que sí. Esto importará:

  • Tema oscuro/claro
  • Extensiones instaladas (algunas)
  • Atajos de teclado personalizados
  • Ajustes generales (font size, word wrap, etc.)

Paso 3: Configurar el modelo de IA

Abre Cursor Settings (Cmd+Shift+P en Mac, Ctrl+Shift+P en Windows/Linux, busca "cursor settings").

// En Settings, busca "model" y verás opciones como:
// - Claude 3.5 Sonnet (recomendado, por defecto)
// - GPT-4o
// - GPT-4
// - Claude 3 Opus

// Recomendación: Usa Claude Sonnet (mejor balance calidad/velocidad)
🔑 Autenticación: Cursor te pedirá una API key de OpenAI (si quieres usar GPT) o cuenta de Anthropic (para Claude). Free tier = sin credenciales necesarias.

2.3 Los 4 pilares de Cursor: Tab, Cmd+K, Chat y Composer

1. Tab (Autocompletado inteligente)

Pulsa Tab para aceptar el siguiente bloque de código que predice la IA.

  • ¿Cómo funciona? La IA ve el contexto (función actual, archivo, proyecto) y predice el siguiente cambio completo, no solo la siguiente línea.
  • Caso de uso: Escribir boilerplate rápido, completar bucles, generar tests.
  • Diferencia con Copilot: Cursor predice cambios mayores; Copilot solo predice la siguiente línea.
// Escribes:
function calculateTotal(items) {

// Cursor predice y ofrece como Tab suggestion:
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0);
}
2. Cmd+K / Ctrl+K (Edición inline)

Selecciona código y pulsa Cmd+K (Mac) o Ctrl+K (Windows/Linux). Se abre un input donde puedes describir cambios en lenguaje natural.

  • ¿Cómo funciona? La IA reescribe el código seleccionado según tu descripción.
  • Caso de uso: Refactorizar funciones, cambiar lógica, optimizar código.
  • Ejemplo: Selecciona una función y escribe "convierte esto a async/await".
// Seleccionas esta función:
function getData(url) {
  let result = null;
  fetch(url)
    .then(res => res.json())
    .then(data => { result = data; });
  return result;
}

// Abres Cmd+K y escribes: "convierte a async/await"
// La IA reescribe:
async function getData(url) {
  const res = await fetch(url);
  return await res.json();
}
3. Cmd+L / Ctrl+L (Chat)

Abre un panel de chat lateral donde conversar con la IA sobre tu código.

  • ¿Cómo funciona? Puedes preguntar sobre el código, pedir explicaciones, o pedir cambios sin seleccionar nada.
  • Caso de uso: Entender código legado, obtener ideas de refactorización, debugging colaborativo.
  • Bonus: Puedes usar @symbols para dar contexto (ver sección 2.4).
// En el chat escribes:
"@file Explícame cómo funciona esta función"
// La IA ve el archivo actual y lo explica

// O:
"@codebase ¿Dónde se llama a validateUser()?"
// Busca en todo el proyecto
4. Composer (Cmd+I / Ctrl+I)

Abre un editor grande donde describe qué quieres crear. La IA genera el código en múltiples archivos.

  • ¿Cómo funciona? Describes una feature o componente, y Composer crea todo el código necesario (puede tocar N archivos).
  • Caso de uso: Crear features completas, scaffolding de proyectos, agregar módulos grandes.
  • Modes: Agent Mode (IA decide cambios), Ask Mode (solo propone, tú revisas), Manual Mode (tú dirigeso).
// En Composer (Cmd+I) escribes:
"Crea una API REST con 3 endpoints:
- GET /users (lista todos)
- POST /users (crea nuevo)
- DELETE /users/:id (borra por ID)
Usa Node.js + Express. Devuelve JSON. Incluye error handling."

// Composer genera:
// - server.js (toda la lógica)
// - package.json (dependencias)
// - .env.example (variables)
// Revisa cada cambio antes de aceptar

2.4 @Symbols: Cómo dar contexto a la IA

En Chat y Composer, puedes usar @symbols para indicar a la IA dónde buscar contexto:

Symbol Qué hace Ejemplo
@file Referencia el archivo actual "@file Refactoriza esta función"
@folder Referencia toda una carpeta "@folder /src ¿Dónde se usa React?"
@codebase Indexa todo el proyecto (lento la primera vez) "@codebase ¿Qué es la función main()?"
@web Busca en internet (para documentación, API docs) "@web Integra la API de Stripe"
@docs Busca en documentación oficial (si está indexada) "@docs ¿Cómo hacer pagination en React?"
💡 Pro tip: Combina @symbols. "Crea un hook que use @codebase para entender el estado global del proyecto" → la IA ve el proyecto completo antes de generar.

2.5 Composer Modes: Agent, Ask y Manual

En Composer (Cmd+I), puedes elegir cómo la IA maneja los cambios:

Agent Mode
La IA decide y aplica cambios automáticamente. Rápido, pero menos control.
  • ✓ Rápido para tareas simples
  • ✗ Puedes no revisar cambios importantes
  • Cuándo: Scaffolding rápido, boilerplate, refactoring pequeño
Ask Mode (RECOMENDADO)
La IA propone cambios, tú revisas y aceptas cada uno antes de aplicar.
  • ✓ Control total, aprendes qué hace la IA
  • ✓ Seguro para código importante
  • Cuándo: Siempre, especialmente proyectos reales
Manual Mode
Tú escribes los cambios paso a paso; la IA ayuda pero no toca código.
  • ✓ Control máximo
  • ✗ Más lento
  • Cuándo: Código crítico, seguridad

2.6 Flujo completo: De idea a API en 5 minutos

Escenario: Crear una API REST simple que maneje tareas.

Paso 1: Abre Composer (Cmd+I) 
Crea una API REST en Node.js con Express para gestionar tareas.
Necesito:
- GET /tasks (devuelve todas las tareas)
- GET /tasks/:id (devuelve una tarea por ID)
- POST /tasks (crea una nueva tarea con titulo y descripcion)
- PUT /tasks/:id (actualiza una tarea)
- DELETE /tasks/:id (borra una tarea)
- Usa un array en memoria (no BD)
- Devuelve JSON
- Incluye manejo de errores basico
Paso 2: Selecciona Ask Mode y revisa los cambios

Composer propone crear server.js y package.json. Reviúsa el código, verifica que sea correcto.

Paso 3: Acepta los cambios

Click en "Accept" para cada archivo. El código se escribe automáticamente.

Paso 4: Prueba 
npm install
node server.js
# Ahora tu API corre en http://localhost:3000
⏱️ Tiempo total: ~5 minutos. Sin Cursor, escribir esto desde cero sería 20+ minutos.

💻 Práctica guiada

Cuatro ejercicios para dominar Cursor. Hazlos en orden.

Ejercicio 1: Instalación y primeros pasos (10 min)

Objetivo: Instalar Cursor, importar configuración de VS Code, cambiar el modelo a Claude.

Pasos:
  1. Descargar Cursor desde cursor.com según tu SO.
  2. Abrirlo. Importar configuración de VS Code.
  3. Abre Cursor Settings (Cmd+Shift+P → "cursor settings").
  4. Busca "model" y selecciona Claude 3.5 Sonnet.
  5. Abre un proyecto existente (o crea una carpeta vacía) y verifica que Tab te sugiera código.
✓ Checklist de completitud:
  • Cursor instalado y abierto
  • Configuración importada de VS Code
  • Modelo = Claude Sonnet
  • Tab funciona (escribe un comentario, presiona Tab)

Ejercicio 2: Tab y Cmd+K (15 min)

Objetivo: Practicar autocompletado con Tab y edición inline con Cmd+K.

Tarea A: Usa Tab para autocompletar
  1. Crea un archivo app.js vacío.
  2. Escribe: function sumArray(arr) {
  3. Presiona Tab. Cursor debería completar la función (sumar elementos).
  4. Acepta si parece correcto.
  5. Escribe otro: function reverseString(str) { y prueba Tab.
Tarea B: Usa Cmd+K para refactorizar
  1. Escribe una función (puede ser código mediocre a propósito):
    2. function calculatePrice(price, quantity, tax) {
  var result = price * quantity;
  result = result + (result * tax / 100);
  return result;
}
  2. Selecciona la función (Ctrl+A si es el único contenido).
  3. Presiona Cmd+K (Mac) o Ctrl+K (Windows/Linux).
  4. Escribe: "Refactoriza: usa const, simplifica cálculos, usa una sola línea de retorno".
  5. Revisa el resultado.
✓ Qué deberías ver:
  • Tab genera código funcional (probablemente correcto)
  • Cmd+K reescribe el código según tu descripción

Ejercicio 3: Chat con @symbols (15 min)

Objetivo: Usar Cmd+L para conversar con la IA sobre tu código.

Tarea:
  1. Crea un archivo con una función algo compleja (ej: función recursiva, closure, etc.).
  2. Abre Chat (Cmd+L o Ctrl+L).
  3. En el chat, escribe: "@file Explícame esta función paso a paso".
  4. Revisa la explicación de la IA.
  5. Luego escribe: "@file Sugiere mejoras".
  6. La IA debería analizar el archivo y dar recomendaciones.
✓ Pro tip: Prueba @codebase si tienes varios archivos. Ej: "@codebase ¿Dónde se importa la función helper?"

Ejercicio 4: Composer — Crear una feature completa (10 min)

Objetivo: Usar Composer (Cmd+I) para generar código multi-archivo.

Tarea:
  1. Crea una carpeta vacía llamada todo-app.
  2. Abre Composer (Cmd+I o Ctrl+I).
  3. Escribe este prompt:
    4. Crea una aplicación de tareas simple con HTML, CSS y JavaScript.
- Permite agregar tareas nuevas
- Mostrar lista de tareas con botón eliminar
- Marcar como completada (tachado)
- Guardar en localStorage
- Interfaz limpia y moderna
Necesito: index.html, style.css, script.js
  4. Selecciona Ask Mode.
  5. Revisa cada archivo que propone la IA.
  6. Haz click en "Accept" para aceptar.
  7. Abre index.html en el navegador y prueba la app.

🎯 Mini-reto del día: De idea a código en 5 minutos

Reto: Usa Composer para crear una mini app escribiendo solo la descripción en español. Cronométrate.

Especificación del reto

Elige UNO de estos retos (todos deben completarse en ≤5 minutos):

Opción A: Generador de contraseñas

Input para seleccionar longitud y tipo (números, mayúsculas, símbolos). Botón generar. Mostrar contraseña. Botón copiar al portapapeles.

Opción B: Conversor de unidades

Convierte entre: metros/pies, kg/libras, celsius/fahrenheit. Select para elegir conversión. Inputs para valores. Resultado en tiempo real.

Opción C: Pomodoro Timer

25 minutos trabajo, 5 minutos descanso. Timer con progreso visual. Botones start/pausa/reset. Sonido al completar.

Reglas del reto:
  • Abre Composer (Cmd+I)
  • Describe SOLO en español (sin escribir código)
  • Usa Ask Mode y revisa los cambios
  • Acepta y prueba en el navegador
  • ¿Cuánto tardaste realmente?
🏆 Bonificación: Si logras menos de 5 minutos, intenta otro reto. Si logras todos tres en ≤15 minutos total, eres un Cursor master.
