Este es el plan que me he montado después de ver másteres y bootcamps y darme cuenta de que no cubrían las cosas que más ganas tenía yo de aprender. Aquí está todo lo que voy a hacer de aquí a julio.

02Módulos

9 módulos que van de Python como base hasta agentes autónomos y preparar entrevistas.

MÓDULO 01 · Semanas 1-2 · 20 h

Python para desarrolladores web

COMPLETADO
Qué quería conseguir: manejar Python lo suficiente como para empezar a construir con IA, aprovechando que ya me muevo bien en JavaScript. En dos semanas quería pasar de "hola mundo" a un servidor web funcionando. Lo conseguí.
  • Variables, tipos, strings, f-strings
  • Listas, diccionarios y tuplas
  • Condicionales, bucles y comprensiones
  • Funciones, módulos e imports
  • Ficheros, JSON y APIs con requests
  • Clases y OOP básica
  • Entornos virtuales y pip
  • Web scraping con BeautifulSoup
  • Flask: servidor web desde cero
Boss battle: mini API REST en Flask que devuelve datos en JSON
MÓDULO 02 · Semana 3 · 10 h

Fundamentos de IA y LLMs

EN CURSO
Qué quiero conseguir: entender qué está pasando por dentro cuando hablo con un LLM, y la diferencia entre los cerrados (GPT, Claude) y los open source. Algo de teoría, la necesaria, sin meterme en mates que no voy a usar.
  • ML, Deep Learning y LLMs (los conceptos que hacen falta)
  • Tokens, contexto y temperatura
  • Prompt engineering avanzado: CoT, few-shot, system prompts
  • Modelos cerrados: GPT, Claude, Gemini
  • Modelos open source: Llama, Mistral, Ollama en local
  • Hugging Face Hub como ecosistema
Boss battle: resolver 5 problemas reales solo con prompts, uno con modelo local vía Ollama
Certificaciones: Elements of AI (Helsinki), Google AI Essentials, DeepLearning.AI Prompt Engineering
MÓDULO 03 · Semana 4 · 10 h

APIs de IA — mi primer chatbot

PRÓXIMO
Qué quiero conseguir: que mi propio código hable con una IA. Acabar con un chatbot desplegado en internet, con su seguridad básica cubierta desde el primer proyecto.
  • OpenAI API: setup, primera llamada, parámetros
  • Temperatura, max tokens, roles (system / user / assistant)
  • Chatbot con memoria de conversación
  • Anthropic API (Claude) como alternativa
  • AI Safety: prompt injection, bias, moderación
  • Deploy en Render con gestión de secretos
Boss battle: chatbot web con Flask + OpenAI/Claude desplegado en Render, con protección básica contra prompt injection (Proyecto de portfolio #1)
MÓDULO 04 · Semana 5 · 10 h

AI Coding Tools

PRÓXIMO
Qué quiero conseguir: programar con IA al lado, no detrás. Saber pedir bien, corregir lo que me sugiere y decidir qué acepto y qué no. Es medio magia y tengo mucha curiosidad.
  • GitHub Copilot: autocompletado y chat en VS Code
  • Cursor IDE: Composer y edición asistida por IA
  • Claude Code y Windsurf: agentes de código en terminal
  • Prompts en comentarios, TDD con IA, refactoring asistido
  • Análisis de código ajeno con IA (ejercicio codebase-to-course)
Boss battle: reconstruir mi proyecto de fin de ciclo (red social de ideas) con AI tools y comparar tiempo y calidad vs. original
MÓDULO 05 · Semanas 6-7 · 20 h

Context, Search y RAG — conectar la IA con tus propios datos

PRÓXIMO · EL QUE MÁS GANAS TENGO
Qué quiero conseguir: que una IA pueda responderme sobre datos que no conoce de entrenamiento (mis apuntes, por ejemplo). Hay tres formas de hacerlo (context stuffing, agentic search, RAG clásico) y quiero entender cuándo usar cada una, no solo picar código.
  • Concepto clave: context engineering > prompt engineering
  • Context stuffing: ventanas de 200K-1M, costes y límites reales
  • Agentic search con tool use: el modelo decide qué buscar
  • Embeddings y vector stores desde cero (ChromaDB)
  • RAG clásico: chunking, retrieval, reranking
  • Evals: Ragas, LLM-as-judge, métricas básicas
  • Frontend: Next.js + Vercel AI SDK con streaming
  • Observability: Langfuse para trazar llamadas
Boss battle: "Pregúntale a mis apuntes" — implementar las 3 variantes sobre el mismo corpus con informe comparativo de coste, latencia y calidad (Proyecto de portfolio #2)
MÓDULO 06 · Semana 8 · 10 h

Agentes — del tool use al sistema autónomo

PRÓXIMO
Qué quiero conseguir: entender qué es un agente en realidad (modelo + herramientas + bucle + memoria, no es magia) y montar uno desde cero sin depender de frameworks que añaden ruido.
  • Anatomía de un agente: modelo + tools + loop + memoria
  • Claude Agent SDK / OpenAI Agents SDK (nativos)
  • Tool use avanzado: schemas JSON, llamadas paralelas, retries
  • MCP (Model Context Protocol) como estándar 2026
  • Patrones multi-agente: orquestador, peer-review, sequential
  • Mención a CrewAI y LangGraph (contexto, no uso)
Boss battle: agente que busca en web, razona y escribe informe, construido con SDK nativo y al menos una tool vía MCP
Certificación: IBM AI Fundamentals
MÓDULO 07 · Semana 9 · 10 h

Full-stack AI app

PRÓXIMO
Qué quiero conseguir: juntar todo lo aprendido en una app completa, de las de verdad. Con login, streaming, RAG y todo. Algo que pueda usar yo mismo o dejar a alguien para que pruebe.
  • Arquitectura de una AI app: frontend + backend + LLM
  • Vercel AI SDK: streaming de respuestas en React
  • Autenticación y rate limiting
  • UX para apps de IA: patrones de ChatGPT, Perplexity, Claude
  • Observabilidad y costes en producción
Boss battle: app completa desplegada (asistente IA para un caso de uso real con RAG) (Proyecto de portfolio #3)
MÓDULO 08 · Semana 10 · 10 h

Portfolio, CV y perfil público

PRÓXIMO
Qué quiero conseguir: que lo que he hecho se pueda ver y se pueda entender en 30 segundos. Un proyecto que nadie encuentra es como si no existiera.
  • Pulir los 3 proyectos con README profesional y demos
  • GitHub profile: bio, pinned repos, contribuciones
  • CV actualizado con sección IA y keywords ATS
  • LinkedIn: headline, about, featured, certificaciones
  • Vídeo corto de presentación (2 min)
Boss battle: portfolio web publicado con los 3 proyectos + vídeo de presentación

Empezar a buscar curro

PRÓXIMO
Qué quiero conseguir: saber dónde mirar ofertas, cómo preparar entrevistas técnicas de IA sin morir y cómo contar lo que he hecho para que se entienda.
  • Plataformas: LinkedIn Jobs, InfoJobs Tech, Wellfound, RemoteOK, Turing

Boss battle final: plan de 30 días con 5 empresas target, CV adaptado a cada una, 3 mensajes fríos a recruiters y 1 candidatura enviada de verdad
